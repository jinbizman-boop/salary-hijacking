# Store Screenshot Plan

This plan defines the minimum store screenshot set for the Salary Hijacking mobile app. Images must be captured from real app UI using masked sample data. They must not imply profit guarantees, loan approval, health improvement, event rewards, or unsupported financial outcomes.

## Required Files

- `01_home_salary.png`: salary home, protected paycheck amount, fixed expense summary.
- `02_daily_budget.png`: daily budget, spent amount, remaining daily budget.
- `03_plan_setting.png`: payroll plan, fixed expenses, fixed savings, living budget.
- `04_notifications.png`: reminders for payday, fixed expenses, savings, and LV UP.
- `05_level_up.png`: LV UP hub for reading, news, English, and health routines.
- `feature_graphic_google_play.png`: Google Play feature graphic with brand copy and a real mobile UI composition.

## Safety Rules

- Use real app UI only.
- Use masked sample data only.
- Raw salary, expense, savings, account, card, loan, phone, email, token, and device identifiers must not appear.
- Community screenshots must not expose real user content, employer names, insults, harassment, or private identity hints.
- Advertising or partner content must be clearly labeled if visible.
- Copy must avoid profit guarantees, loan promises, health guarantees, exaggerated savings claims, and unsupported event claims.

## Review Rules

- The visible screens must match implemented app behavior.
- Financial amounts must be sample data and must not represent a real user.
- Korean copy must be readable at store preview size.
- Screenshots must be exported as PNG before store upload.
- Public release remains blocked until every required image file exists and passes the release readiness check.
