# 급여납치 플랫폼 파일 네이밍 규칙 최종본

## 공통 원칙

- 소스 파일은 영문 소문자와 하이픈 또는 카멜케이스를 사용한다.
- 문서 파일은 한국어 제목을 허용하되, 순번과 `_최종본` 표기를 유지한다.
- 도메인 폴더는 `salary`, `plan`, `budget`, `expense`, `level`, `community`, `profile`, `notifications`, `admin` 기준으로 통일한다.
- API 라우트 파일은 `*.routes.ts`, 서비스 파일은 `*.service.ts`, 저장소 파일은 `*.repository.ts`, 스키마 파일은 `*.schema.ts`로 끝낸다.
- 테스트 파일은 `*.test.ts`, E2E 파일은 `*.e2e.ts`, 성능 테스트 파일은 `*.perf.ts`로 끝낸다.

## 브랜치 네이밍

| 유형     | 규칙                      | 예시                         |
| -------- | ------------------------- | ---------------------------- |
| 기능     | `feature/<도메인>-<설명>` | `feature/salary-plan-save`   |
| 버그     | `fix/<도메인>-<설명>`     | `fix/budget-negative-amount` |
| 문서     | `docs/<문서명>`           | `docs/api-contract-update`   |
| 리팩터링 | `refactor/<영역>`         | `refactor/community-service` |
| 핫픽스   | `hotfix/<버전>-<문제>`    | `hotfix/v1.0.1-login-token`  |
| 릴리즈   | `release/v<버전>`         | `release/v1.0.0`             |

## 커밋 네이밍

Conventional Commits 기준을 따른다.

```text
feat: 급여 계획 저장 API 추가
fix: 일일 예산 초과 표시 오류 수정
docs: API 명세서 링크 갱신
refactor: 커뮤니티 게시글 서비스 분리
test: 납치금액 계산 테스트 추가
chore: 패키지 설정 정리
```
