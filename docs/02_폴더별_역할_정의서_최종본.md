# 급여납치 플랫폼 폴더별 역할 정의서 최종본

| 폴더                      | 책임                                   | 주 담당          | 변경 승인     |
| ------------------------- | -------------------------------------- | ---------------- | ------------- |
| `.github/`                | 협업, 이슈, PR, CI/CD, CODEOWNERS      | 플랫폼/개발 리드 | 개발 리드     |
| `apps/mobile/`            | 사용자용 모바일 앱 화면과 기능         | 모바일 개발      | 모바일 리드   |
| `apps/admin/`             | 운영자용 관리자 콘솔                   | 프론트엔드/운영  | 운영 리드     |
| `services/api/`           | REST API, 인증, 권한, 비즈니스 로직    | 백엔드           | 백엔드 리드   |
| `services/scheduler/`     | 자동 알림, 월별 정산, 데이터 정리      | 백엔드/플랫폼    | 백엔드 리드   |
| `services/notifications/` | 푸시 발송, 토큰, 재시도 큐             | 백엔드/플랫폼    | 백엔드 리드   |
| `packages/types/`         | 도메인 타입과 상태값                   | 전체 개발        | 아키텍처 리드 |
| `packages/api-contract/`  | API 요청/응답 계약                     | 프론트+백엔드    | 백엔드 리드   |
| `packages/db/`            | DB 스키마와 쿼리 계층                  | 백엔드           | DB 담당       |
| `packages/ui/`            | 디자인 시스템 기반 공통 UI             | 디자인/프론트    | 디자인 리드   |
| `packages/security/`      | 암호화, 마스킹, 권한, 검증 유틸        | 보안/백엔드      | 보안 담당     |
| `database/`               | Neon DB 마이그레이션, 시드, 정책       | 백엔드/DB        | DB 담당       |
| `infra/`                  | Cloudflare, Neon, GitHub, 도메인, 환경 | 플랫폼           | 플랫폼 리드   |
| `docs/`                   | 최종 문서 전체                         | 기획/PM          | PM            |
| `qa/`                     | 테스트 실행 자산                       | QA               | QA 리드       |
| `ops/`                    | 운영, CS, 장애, 모더레이션             | 운영             | 운영 리드     |
| `release/`                | 스토어 출시, 버전, 롤백                | 출시 담당        | PM/개발 리드  |
| `assets/`                 | 브랜드, 디자인, 스크린샷, 프로토타입   | 디자인           | 디자인 리드   |
| `scripts/`                | 개발/DB/품질/배포 자동화               | 개발             | 개발 리드     |
| `tools/`                  | 내부 점검 도구                         | 플랫폼           | 플랫폼 리드   |
| `security/`               | 위협 모델, 보안 체크, 취약점 대응      | 보안             | 보안 담당     |
| `observability/`          | 로그, 메트릭, 알림, 대시보드           | 플랫폼/운영      | 플랫폼 리드   |
