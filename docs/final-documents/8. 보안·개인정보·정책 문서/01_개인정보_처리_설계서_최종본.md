# 01. 개인정보 처리 설계서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼이 수집·이용·저장·파기하는 개인정보의 전체 처리 구조를 정의한다. 급여납치는 급여관리, 가계부, 자기계발 레벨업, 커뮤니티, 알림, 광고/제휴 기능을 포함하므로 개인정보 흐름을 기능별로 분리하고, 특히 급여·대출·저축·소비 내역은 서비스 내부에서 고위험 재무성 개인정보로 보호한다.

| 원칙             | 확정 기준                                                                                  |
| ---------------- | ------------------------------------------------------------------------------------------ |

## 4. 개인정보 처리 흐름

```text
사용자 입력/앱 이벤트
  → 모바일 앱 입력값 검증
  → API 인증/인가 검증
  → 서비스 계층 목적별 처리
  → Neon DB 저장 또는 Cloudflare R2 파일 저장
  → 앱 화면 표시/알림/집계
  → 보존기간 경과 또는 사용자 삭제 요청 시 삭제/익명화
```

## 8. 개인정보 보호 구현 기준

| 영역       | 기준                                                                  |
| ---------- | --------------------------------------------------------------------- |
| 전송       | 모든 API 통신 HTTPS/TLS 1.2 이상, 권장 TLS 1.3                        |
| 저장       | DB 저장 암호화, 재무성 주요 필드 애플리케이션 레벨 AES-256-GCM 암호화 |

| 세션       | 기기별 세션 관리, 강제 로그아웃 지원                                  |
| 관리자     | MFA 필수, IP allowlist, 감사 로그 필수                                |
| 로그       | 비밀번호, 토큰, 원문 재무 데이터, 주민번호 등 로그 저장 금지          |
| 위탁/제3자 | 위탁/제3자 제공 시 개인정보처리방침 및 동의 화면에 고지               |

## 10. 최종 완료 선언

본 개인정보 처리 설계서는 급여납치 플랫폼의 수집 항목, 이용 목적, 보관 기간, 삭제 방식, 접근권한, 암호화, 사용자 권리, 광고 분리 기준을 완결한다. 따라서 본 문서는 문서상·이론상 더 이상 수행할 작업이 없는 최종본으로 확정한다.
