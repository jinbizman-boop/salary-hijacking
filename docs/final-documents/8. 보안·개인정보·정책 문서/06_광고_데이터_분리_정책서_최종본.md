# 06. 광고 데이터 분리 정책서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 광고/제휴 배너 운영 시 급여·대출·저축·소비 원천 데이터와 광고 타겟팅 데이터를 분리하는 기준을 정의한다. 급여납치는 사용자의 재무 정보를 다루므로 광고 수익화보다 신뢰성과 개인정보 보호를 우선한다.

## 2. 핵심 원칙

| 원칙                            | 확정 기준                                                                    |
| ------------------------------- | ---------------------------------------------------------------------------- |
| 재무 원천 데이터 직접 활용 금지 | 급여액, 대출액, 저축액, 소비 항목 원문을 광고 타겟팅에 직접 사용하지 않는다. |
| 기본 비개인화 광고              | 기본값은 문맥형/위치형 배너이며 개인화 광고는 선택 동의 후에만 활성화한다.   |
| 광고 데이터 별도 저장           | ad\_\* 테이블은 payroll/expense/savings 원천 테이블과 직접 JOIN하지 않는다.  |
| 광고주 제공 제한                | 광고주에게 개인 식별자, 급여액, 대출액, 지출 내역을 제공하지 않는다.         |
| 투명 고지                       | 광고 영역은 `광고` 또는 `제휴`로 명확히 표시한다.                            |
| 철회권 보장                     | 사용자는 맞춤형 광고 동의를 언제든지 철회할 수 있다.                         |

## 3. 데이터 분리 구조

```text
[재무 데이터 영역]
users / payroll_plans / fixed_expenses / savings_plans / variable_expenses
  └─ 광고 시스템 직접 접근 금지

[광고 운영 영역]
ad_campaigns / ad_creatives / ad_placements / ad_impressions / ad_clicks
  └─ user_ad_profile_id 사용

[연결 제한]
user_id → one-way hash → user_ad_profile_id
재무 원천 데이터 → 광고 타겟팅 직접 입력 금지
```

| 기기/OS/앱버전      |                        허용 | 광고 호환성, 오류 분석 목적                       |

## 5. 광고 유형별 정책

| 광고 유형               | 설명                                 | 동의 필요      | 허용 기준                    |
| ----------------------- | ------------------------------------ | -------------- | ---------------------------- |
| 문맥형 광고             | 현재 화면/콘텐츠 문맥 기준 광고      | 불필요         | 재무 원천 데이터 미사용      |
| 비개인화 제휴 배너      | 전체 사용자 또는 화면 위치 기준 배너 | 불필요         | 광고 표시 명확화             |
| 맞춤형 콘텐츠 기반 광고 | LV UP 카테고리·광고 반응 기반        | 선택 동의 필요 | 재무 데이터 제외             |
| 재무 데이터 기반 광고   | 급여/대출/저축/소비 기반             | 금지           | 구현 금지                    |
| 리타겟팅 광고           | 외부 광고 식별자 기반 추적           | 기본 금지      | 별도 명시 동의 없이는 미사용 |

## 6. 광고 테이블 접근 정책

| 테이블            | 접근 주체        | 포함 데이터                                              | 제한                     |
| ----------------- | ---------------- | -------------------------------------------------------- | ------------------------ |
| ad_campaigns      | 운영 관리자      | 캠페인명, 기간, 랜딩 URL                                 | 개인정보 없음            |
| ad_creatives      | 운영 관리자      | 이미지, 문구, 소재 상태                                  | 심사 후 노출             |
| ad_placements     | 운영 관리자      | 화면 위치, 노출 조건                                     | 재무 조건 금지           |
| ad_impressions    | 시스템/광고 운영 | campaign_id, placement_id, user_ad_profile_id, timestamp | user_id 직접 저장 금지   |
| ad_clicks         | 시스템/광고 운영 | click_id, campaign_id, landing_url                       | 개인 재무 정보 포함 금지 |
| ad_frequency_caps | 시스템           | 사용자별 노출 제한                                       | 해시 식별자 사용         |

## 7. 광고주 제공 데이터 기준

| 제공 가능          | 제공 금지              |
| ------------------ | ---------------------- |
| 캠페인별 총 노출수 | 사용자 이메일          |
| 캠페인별 총 클릭수 | user_id 원문           |
| 날짜별 집계 클릭률 | 급여액/납치금액        |
| placement별 성과   | 대출/저축/소비 내역    |
| 비식별 집계 리포트 | 게시글 원문, 신고 내역 |

## 8. 광고 화면 표시 기준

| 위치            | 표시 기준                                                            |
| --------------- | -------------------------------------------------------------------- |
| 급여 홈 배너    | 배너 좌상단 또는 하단에 `광고`/`제휴` 표시                           |
| LV UP 배너      | 콘텐츠와 광고 구분선 표시                                            |
| 마이페이지 배너 | 클릭 시 외부 이동 안내                                               |
| 알림 광고       | 광고성 알림은 마케팅 수신 동의자에게만 발송                          |
| 랜딩 이동       | 외부 브라우저 또는 인앱 브라우저에서 `외부 사이트로 이동합니다` 안내 |

## 9. 맞춤형 광고 철회 처리

| 단계        | 처리                                      |
| ----------- | ----------------------------------------- |
| 사용자 철회 | personalized_ads=false 저장               |
| 즉시 반영   | 광고 요청 시 개인화 파라미터 제거         |
| 기존 로그   | 법정/운영 보존기간까지 보존 후 집계화     |
| 광고주 제공 | 철회 이후 개인화 세그먼트 제외            |
| 재동의      | 사용자가 설정에서 직접 켜는 경우에만 가능 |

## 10. 최종 완료 선언

본 광고 데이터 분리 정책서는 급여·대출·저축·소비 데이터와 광고 타겟팅 데이터의 분리, 활용 금지/허용 기준, 광고주 제공 제한, 동의·철회, 화면 표시 기준을 모두 포함한다. 따라서 본 문서는 문서상·이론상 더 이상 수행할 작업이 없는 최종본으로 확정한다.
