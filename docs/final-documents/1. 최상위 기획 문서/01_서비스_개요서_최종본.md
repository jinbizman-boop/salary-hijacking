# 서비스 개요서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 존재 이유, 서비스 목적, 핵심 사용자, 해결하려는 문제, 제공 가치, 서비스 범위를 최종 확정하기 위한 최상위 기준 문서다. 본 문서를 기준으로 이후 화면 설계, 기능 설계, 데이터 설계, API 설계, 운영 정책, 출시 문서는 동일한 방향성을 유지한다.

본 문서는 문서상·이론상 최종본으로 확정한다. 본 문서 안에서 정의된 서비스 방향, 가치, 핵심 기능, 사용자 범위, 적용 기준은 추가 보완 없이 개발 착수 기준으로 사용할 수 있다.

## 2. 서비스 한 줄 정의

급여납치는 월급 수령 직후 사용자가 지켜야 할 돈을 먼저 분리하고, 고정지출·저축·생활비·일일 소비를 관리하며, 남은 금액을 “납치금액”으로 시각화해 소비 통제와 자기계발 루틴을 동시에 지원하는 모바일 급여관리·가계부 플랫폼이다.

## 3. 서비스 핵심 개념

급여납치의 핵심은 “월급이 사라진 뒤 후회하는 가계부”가 아니라 “월급이 사라지기 전에 먼저 붙잡는 자기관리 시스템”이다. 사용자는 급여일, 수령 예정 급여, 고정지출, 고정저축, 생활비, 변동지출을 입력하고, 앱은 사용 가능한 금액과 지켜낸 금액을 즉시 계산한다.

서비스는 단순 지출 기록을 넘어서 다음 4가지 축으로 구성된다.

| 축        | 설명                                             | 대표 기능                                             |
| --------- | ------------------------------------------------ | ----------------------------------------------------- |
| 급여 관리 | 월급 수령 후 돈의 흐름을 선점 관리               | 급여 등록, 급여일 관리, 지출 예정 금액, 납치금액 계산 |
| 예산 통제 | 월간·일간 소비 한도를 설정하고 초과를 방지       | 고정지출, 고정저축, 생활비, 변동지출, 일일 예산       |
| 자기계발  | 소비관리와 함께 매일 성장 루틴을 제공            | 독서, 뉴스, 영어, 건강 미션, 레벨업                   |
| 커뮤니티  | 직장인·아르바이트생이 소비·성장·생활 경험을 공유 | 자유게시판, 레벨업 인증, 취미 게시판, 인기글          |

## 4. 서비스 목적

급여납치의 목적은 다음과 같이 최종 확정한다.

1. 직장인과 아르바이트생이 급여를 받은 직후 월급 흐름을 명확히 파악하게 한다.
2. 고정지출, 고정저축, 생활비, 변동지출을 분리해 과소비를 줄인다.
3. 사용자가 매일 사용할 수 있는 금액을 직관적으로 확인하도록 한다.
4. 지출 후 남은 돈이 아니라, 사용자가 지켜낸 돈을 “납치금액”으로 성과화한다.
5. 금융관리와 자기계발을 결합해 앱 재방문과 생활 습관 형성을 유도한다.
6. 커뮤니티와 인증 기능을 통해 동기부여와 지속 사용을 강화한다.
7. 광고·제휴·콘텐츠 확장을 통해 지속 가능한 플랫폼 사업 구조를 만든다.

## 10. 핵심 계산 개념

급여납치의 서비스 정체성은 계산 구조로 완성된다. 계산식은 다음과 같이 최종 확정한다.

```text
예상 납치금액 = 수령 예정 급여 - 지출 예정 금액
오늘 남은금액 = 일일 설정금액 - 오늘 사용금액
월별 확정 납치금액 = 실제 수령금액 - 실제 지출금액
누적 납치금액 = 기존 누적 납치금액 + 월별 확정 납치금액
목표 달성률 = 총 누적 납치금액 / 목표 납치금액 × 100
```

금액은 원 단위 정수로 처리한다. 음수 금액은 입력할 수 없다. 지출이 수령금액을 초과하는 경우 납치금액은 0원으로 표시하고 초과 상태를 별도 경고한다.

## 12. 운영 원칙

1. 급여·대출·저축·소비 데이터는 민감 데이터로 취급한다.
2. 광고 추천 데이터와 민감 금융 데이터는 분리한다.
3. 사용자의 핵심 금액 계산은 항상 투명하게 보여준다.
4. 커뮤니티에는 신고, 숨김, 차단, 운영자 검수 정책을 둔다.
5. 사용자가 알림을 직접 켜고 끌 수 있어야 한다.
6. 레벨업 콘텐츠는 투자·건강·금융 조언으로 오인되지 않도록 안내한다.
7. 초기 버전은 급여 관리 핵심 루프를 최우선으로 한다.
