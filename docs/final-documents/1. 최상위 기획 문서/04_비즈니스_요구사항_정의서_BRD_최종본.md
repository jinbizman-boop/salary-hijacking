# 비즈니스 요구사항 정의서 BRD 최종본

## 1. 문서 목적

본 BRD는 급여납치 플랫폼의 사업 관점 요구사항, 운영 목표, 수익화 목표, 제휴·광고 구조, 관리자 요구사항, 이해관계자 요구사항을 최종 확정하는 문서다. 본 문서는 사업기획, 제품기획, 개발, 운영, 제휴, 마케팅, 고객지원이 동일한 기준으로 움직이기 위한 상위 비즈니스 기준이다.

본 문서는 문서상·이론상 최종본이며, 사업 운영과 수익화 관점에서 추가 요구사항 공백이 없도록 작성되었다.

## 8. 비즈니스 프로세스

### 8.1 사용자 가입 프로세스

1. 사용자가 앱 설치 후 실행한다.
2. 스플래시 화면을 본다.
3. 일반 로그인 또는 소셜 로그인을 선택한다.
4. 약관과 개인정보 동의를 완료한다.
5. 급여계획 입력으로 이동한다.
6. 최초 급여 계획을 저장한다.
7. 홈 화면에서 납치금액을 확인한다.

### 8.2 급여관리 프로세스

1. 사용자가 급여일과 수령 예정 급여를 입력한다.
2. 고정지출과 고정저축을 등록한다.
3. 생활비와 일일 예산을 설정한다.
4. 변동지출을 기록한다.
5. 앱은 사용금액, 남은금액, 납치금액을 계산한다.
6. 목표 달성률과 누적 납치금액을 업데이트한다.
7. 월 마감 시 월별 확정 납치금액을 누적한다.

### 8.3 광고 운영 프로세스

1. 운영자가 광고 소재를 등록한다.
2. 광고 위치, 기간, 랜딩 URL, 상태를 설정한다.
3. 광고 심사 기준을 확인한다.
4. 광고가 설정된 화면에 노출된다.
5. 노출과 클릭 로그가 저장된다.
6. 운영자는 광고 성과를 확인한다.
7. 만료되거나 중지된 광고는 노출되지 않는다.

### 8.4 커뮤니티 운영 프로세스

1. 사용자가 게시글을 작성한다.
2. 게시글은 게시판 유형에 맞게 등록된다.
3. 사용자는 조회, 댓글, 좋아요, 공유를 수행한다.
4. 부적절한 게시글은 신고된다.
5. 신고 누적 또는 운영자 판단으로 숨김 처리한다.
6. 운영자는 삭제, 복구, 제재를 결정한다.
7. 처리 이력은 관리자 로그에 남는다.

## 11. 규정 및 준수 요구사항

1. 개인정보 수집과 이용 목적을 명확히 고지한다.
2. 필수 동의와 선택 동의를 분리한다.
3. 광고성 정보 수신 동의는 별도로 받는다.
4. 사용자는 자신의 개인정보 삭제를 요청할 수 있어야 한다.
5. 급여·소비 정보는 목적 외 사용을 금지한다.
6. 커뮤니티 게시글은 명예훼손, 개인정보 침해, 불법 정보에 대응해야 한다.
7. 금융성 콘텐츠는 투자 판단의 책임이 사용자에게 있음을 고지한다.
8. 건강 콘텐츠는 의료 진단이나 치료 목적이 아님을 고지한다.
