# MVP 범위 정의서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 1차 출시 범위, 필수 기능, 제외 기능, 후순위 기능, 성공 기준, 완료 기준을 최종 확정하기 위한 MVP 기준 문서다. 본 문서는 기능 범위 과다로 인한 개발 지연을 방지하고, 급여납치의 핵심 가치를 가장 빠르게 검증하기 위한 기준으로 사용된다.

본 문서는 문서상·이론상 최종본이며, MVP 범위에 대한 추가 판단 작업 없이 개발 착수와 QA 기준으로 사용할 수 있다.

## 2. MVP 정의

급여납치 MVP는 사용자가 회원가입 후 급여계획을 등록하고, 고정지출·고정저축·생활비·변동지출을 입력하며, 홈 화면에서 납치금액과 오늘 남은금액을 확인할 수 있는 최소 완성 제품이다.

MVP의 핵심은 “급여 입력 → 지출 계획 → 일일 예산 → 변동지출 기록 → 납치금액 확인”의 1차 사용 루프를 완성하는 것이다.

## 3. MVP 핵심 목표

| 목표           | 설명                                          | 성공 조건                       |
| -------------- | --------------------------------------------- | ------------------------------- |

| 핵심 가치 체험 | 납치금액을 확인한다                           | 홈 화면에서 계산값 노출         |

| 반복 사용      | 알림과 홈 확인을 통해 재방문한다              | 알림 목록과 기본 푸시 구조 제공 |
| 확장 준비      | 레벨업, 커뮤니티, 광고의 최소 구조를 마련한다 | 탭과 기본 화면 제공             |


## 10. MVP 계산 범위

MVP에서 반드시 구현할 계산은 다음과 같다.

```text
예상 납치금액 = 수령 예정 급여 - 지출 예정 금액
오늘 남은금액 = 일일 설정금액 - 오늘 사용금액
총 지출금액 = 고정지출 + 변동지출
누적 납치금액 = 기존 누적 납치금액 + 월별 확정 납치금액
목표 달성률 = 누적 납치금액 / 목표 납치금액 × 100
```

계산 결과는 홈 화면, 계획 화면, 마이페이지에서 동일한 기준으로 표시되어야 한다.

## 13. MVP 출시 승인 기준

MVP는 다음 조건을 모두 충족해야 출시 가능하다.

1. P1 기능이 모두 구현되어야 한다.
2. 급여 홈, 계획, 지출 기록, 납치금액 계산이 정상 동작해야 한다.
3. 사용자는 최초 로그인 후 급여 계획을 완료할 수 있어야 한다.
4. 변동지출 추가 후 금액이 즉시 재계산되어야 한다.
5. 사용자 데이터는 사용자별로 분리 저장되어야 한다.
6. 필수 입력값 검증이 적용되어야 한다.
7. 커뮤니티 글쓰기의 필수값 검증이 적용되어야 한다.
8. 개인정보와 약관 동의 흐름이 제공되어야 한다.
9. 주요 오류 메시지가 사용자에게 명확히 표시되어야 한다.
10. 핵심 QA 테스트를 통과해야 한다.

## 15. 최종 결론

급여납치 MVP는 급여계획, 지출관리, 일일예산, 납치금액 계산, 홈 확인의 핵심 루프를 중심으로 최종 확정되었다. 본 문서는 MVP 범위를 문서상·이론상 완결하며, 개발팀은 본 범위를 벗어나지 않고 1차 출시 가능한 제품을 구현한다.
