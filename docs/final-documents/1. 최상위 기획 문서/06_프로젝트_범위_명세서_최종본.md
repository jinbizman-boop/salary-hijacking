# 프로젝트 범위 명세서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼 프로젝트의 포함 범위, 제외 범위, 산출물, 기술 범위, 변경 승인 기준, 완료 기준을 최종 확정하는 범위 관리 문서다. 본 문서는 개발 중 임의 확장, 기능 추가, 디자인 변경, 정책 변경으로 인한 범위 이탈을 방지하기 위한 기준이다.

본 문서는 문서상·이론상 최종본이며, 프로젝트 범위와 변경 통제 기준에 대한 추가 작업이 필요 없도록 완결한다.

## 2. 프로젝트 정의

급여납치 프로젝트는 국내 직장인과 아르바이트생을 대상으로 하는 모바일 급여관리·가계부·자기계발·커뮤니티 플랫폼을 구축하는 프로젝트다. 프로젝트는 모바일 애플리케이션 중심으로 진행하며, GitHub, Cloudflare, Neon DB를 주요 배포·운영 환경으로 고려한다.

## 3. 프로젝트 목표

## 6. 기능 우선순위 범위

| 우선순위 | 기준                                    | 기능                                                |
| -------- | --------------------------------------- | --------------------------------------------------- |
| P1       | 핵심 서비스가 성립하기 위해 반드시 필요 | 로그인, 급여계획, 지출관리, 일일예산, 납치금액 계산 |
| P2       | 사용 지속성과 운영을 위해 필요          | 알림, 레벨업, 커뮤니티, 마이페이지, 관리자          |

P1 기능은 출시 전 반드시 완료되어야 한다. P2 기능은 MVP 또는 2차 범위에서 최소 기능 이상을 포함한다. P3 기능은 수익화 준비 구조를 우선 두고 고도화는 후속 단계로 처리한다.

다음 항목은 변경 요청서와 승인 절차 없이는 변경할 수 없다.

2. 핵심 사용자 변경
3. 급여관리 핵심 루프 변경
4. 납치금액 계산식 변경
5. P1 기능 제거 또는 후순위 이동
6. 주요 화면 구조 변경
7. 개인정보 수집 범위 변경
8. 광고 데이터 활용 방식 변경
9. 배포 환경 변경
10. 데이터베이스 구조의 핵심 변경

## 12. 변경 요청 절차

```text
변경 요청 발생
→ 변경 요청서 작성
→ 영향 범위 분석
→ 일정·비용·품질 영향 검토
→ 승인 또는 반려
→ 승인 시 문서 버전 갱신
→ 개발/QA 반영
→ 변경 이력 기록
```

승인되지 않은 변경은 프로젝트 범위에 포함하지 않는다.

## 14. 완료 기준

프로젝트 범위 완료는 다음 조건을 모두 충족해야 한다.

1. 포함 범위의 P1 기능이 모두 완료되어야 한다.
2. 주요 화면이 모바일 환경에서 정상 표시되어야 한다.
3. 급여계획, 지출관리, 납치금액 계산이 정상 작동해야 한다.
4. 사용자별 데이터가 분리 저장되어야 한다.
5. 입력값 검증과 오류 메시지가 적용되어야 한다.
6. 커뮤니티 기본 작성과 조회가 가능해야 한다.
7. 관리자 기본 기능이 작동해야 한다.
8. 광고 배너 기본 노출과 클릭이 작동해야 한다.
9. QA 핵심 테스트가 통과되어야 한다.
10. 출시 체크리스트가 완료되어야 한다.

## 16. 최종 결론

본 프로젝트 범위 명세서는 급여납치 플랫폼의 구현 범위, 제외 범위, 산출물, 기술 범위, 변경 통제 기준을 문서상·이론상 최종 확정한다. 본 문서에 명시되지 않은 신규 기능과 임의 변경은 프로젝트 범위에 포함하지 않으며, 승인된 변경 절차를 통해서만 반영한다.
