# 제품 요구사항 정의서 PRD 최종본

## 1. 문서 목적

본 PRD는 급여납치 모바일 애플리케이션의 제품 목표, 기능 범위, 화면 범위, 사용자 시나리오, 우선순위, 정책, 예외 케이스, 수락 기준을 최종 확정하는 제품 개발 기준 문서다. 본 문서는 개발, 디자인, QA, 운영, 출시 판단의 단일 기준으로 사용된다.

본 문서는 문서상·이론상 최종본이며, 본 문서에 명시된 요구사항은 추가 기획 없이 구현 가능한 수준으로 확정한다.

## 2. 제품 개요

급여납치는 사용자가 월급을 받은 직후 돈의 목적을 먼저 나누고, 고정지출·고정저축·생활비·변동지출을 관리하며, 지켜낸 금액을 납치금액으로 시각화하는 모바일 급여관리 플랫폼이다. 앱은 급여관리 기능을 중심으로 알림, 레벨업 콘텐츠, 커뮤니티, 마이페이지, 광고·제휴 기능을 제공한다.

## 8. 사용자 시나리오

### 8.1 신규 사용자 첫 사용 시나리오

1. 사용자는 앱을 실행한다.
2. 스플래시 화면을 본다.
3. 로그인 또는 회원가입을 진행한다.
4. 급여일과 수령 예정 급여를 입력한다.
5. 고정지출과 고정저축을 입력한다.
6. 일일 생활비를 설정한다.
7. 홈에서 예상 납치금액과 오늘 남은금액을 확인한다.
8. 변동지출을 추가한다.
9. 남은금액과 납치금액이 즉시 변경된다.
10. 알림 또는 LV UP 기능을 통해 다음 행동을 수행한다.

### 8.2 기존 사용자 일일 사용 시나리오

1. 사용자는 앱을 실행한다.
2. 홈에서 오늘 남은금액을 확인한다.
3. 소비 발생 시 변동지출을 입력한다.
4. 앱은 오늘 사용금액과 남은금액을 재계산한다.
5. 예산 초과 시 경고 알림을 표시한다.
6. 사용자는 독서, 뉴스, 영어, 건강 중 하나의 미션을 완료한다.
7. 경험치가 증가하고 마이페이지에 성과가 반영된다.

### 8.3 커뮤니티 사용 시나리오

1. 사용자는 커뮤니티 탭으로 이동한다.
2. 전체, 자유, 레벨업 인증, 취미 게시판을 선택한다.
3. 게시글을 조회한다.
4. 좋아요, 댓글, 공유를 수행한다.
5. 글쓰기 버튼을 누른다.
6. 제목, 본문, 게시판, 익명 여부를 입력한다.
7. 등록 완료 후 게시글 목록에 반영된다.

## 9. 계산 요구사항

| 계산명             | 산식                                    | 조건                                  |
| ------------------ | --------------------------------------- | ------------------------------------- |
| 예상 납치금액      | 수령 예정 급여 - 지출 예정 금액         | 0원 미만이면 0원 표시, 초과 경고      |
| 오늘 남은금액      | 일일 설정금액 - 오늘 사용금액           | 음수 가능하되 초과 상태 표시          |
| 실제 지출금액      | 고정지출 + 변동지출                     | 같은 월 기준 합산                     |
| 월별 확정 납치금액 | 실제 수령금액 - 실제 지출금액           | 월 마감 시 확정                       |
| 누적 납치금액      | 기존 누적 납치금액 + 월별 확정 납치금액 | 사용자별 누적                         |
| 목표 달성률        | 누적 납치금액 / 목표금액 × 100          | 100% 초과 시 별도 초과 달성 표시 가능 |

## 14. 정책 요구사항

1. 사용자는 자신의 급여·지출 데이터를 직접 수정·삭제할 수 있어야 한다.
2. 회원 탈퇴 시 개인정보와 민감 데이터 삭제 정책을 적용한다.
3. 광고성 알림은 선택 동의가 필요하다.
4. 커뮤니티 게시글은 신고·숨김·차단 정책을 적용한다.
5. 익명 게시글은 일반 사용자에게 작성자 정보를 노출하지 않는다.
6. 운영자는 법적·정책 위반 게시글을 숨김 또는 삭제할 수 있다.
7. 레벨업 콘텐츠는 정보 제공 목적이며 전문 자문이 아님을 표시한다.
8. 계산 결과는 금액 기준과 산식을 내부적으로 일관되게 적용한다.
