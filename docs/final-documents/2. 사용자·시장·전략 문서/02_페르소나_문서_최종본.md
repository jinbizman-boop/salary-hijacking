# 페르소나 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 화면, 기능, 문구, 알림, 커뮤니티, KPI 판단 시 기준으로 삼을 대표 사용자를 정의한다. 페르소나는 실제 개인정보가 아닌 제품 설계를 위한 대표 모델이며, 각 페르소나의 급여 패턴, 소비 습관, 통장 관리 방식, 앱 사용 동기를 기준으로 UX 의사결정을 수행한다.

## 2. 페르소나 구성 원칙

급여납치의 페르소나는 다음 5개 축을 기준으로 구성한다.

1. 급여 수령 방식: 월급, 주급, 격주급, 불규칙 소득
2. 소비 통제 수준: 계획형, 충동형, 방치형, 회복형
3. 고정지출 복잡도: 낮음, 보통, 높음
4. 자기계발 관심도: 낮음, 보통, 높음
5. 커뮤니티 참여도: 관찰형, 참여형, 인증형

## 4. P-01 김민수: 사회초년생 월급관리 입문자

| 항목           | 내용                                             |
| -------------- | ------------------------------------------------ |
| 나이           | 27세                                             |
| 직업           | 중소기업 마케팅 사원                             |
| 월 수령액      | 약 270만 원                                      |
| 급여일         | 매월 25일                                        |
| 주요 지출      | 월세, 교통비, 식비, 배달, 커피, OTT 구독         |
| 통장 관리 방식 | 급여통장 1개와 체크카드 중심, 저축통장 분리 미흡 |
| 소비 습관      | 월급 직후 소비가 많고 월말에 급격히 절약         |
| 앱 사용 동기   | 첫 월급부터 돈 관리 루틴을 잡고 싶음             |

### 4.1 핵심 생각

“월급은 받는데 월말만 되면 왜 항상 돈이 부족한지 모르겠다. 복잡한 자산관리보다 이번 달에 얼마를 써도 되는지 바로 알고 싶다.”

### 4.2 불편 사항

1. 월급을 받으면 먼저 써야 할 돈과 쓰면 안 되는 돈이 섞인다.
2. 구독료와 카드 결제일을 자주 잊는다.
3. 가계부 앱은 입력이 귀찮고 오래 못 쓴다.
4. 남은 돈을 보면 이미 늦은 경우가 많다.

### 4.3 필수 요구 기능

- 급여일과 수령금액 입력
- 고정지출 템플릿 추천
- 하루 생활비 자동 계산
- 오늘 남은 금액 표시
- 월별 납치금액과 목표 달성률 표시

### 4.4 UX 적용 기준

- 첫 사용 3분 안에 급여 계획을 완성해야 한다.
- 숫자는 “수령금액 / 지출금액 / 납치금액” 3개로 단순화한다.
- 금액 입력창은 자동 콤마, 원 단위, 음수 방지 처리를 제공한다.
- 첫 성공 경험은 “오늘 쓸 수 있는 돈 확인”이어야 한다.

## 5. P-02 이서연: 소비통제 필요 직장인

| 항목           | 내용                                         |
| -------------- | -------------------------------------------- |
| 나이           | 31세                                         |
| 직업           | 스타트업 서비스기획자                        |
| 월 수령액      | 약 340만 원                                  |
| 급여일         | 매월 10일                                    |
| 주요 지출      | 커피, 택시, 배달, 쇼핑, 구독 서비스, 모임비  |
| 통장 관리 방식 | 신용카드 사용 비중 높음, 월말 카드값 확인    |
| 소비 습관      | 작은 금액을 자주 쓰며 체감 지출이 낮음       |
| 앱 사용 동기   | 하루 기준으로 소비를 멈출 타이밍을 알고 싶음 |

### 5.1 핵심 생각

“큰돈을 쓴 것 같지는 않은데 월말 카드값이 항상 생각보다 크다. 오늘 얼마나 썼는지 바로 보여주는 앱이 필요하다.”

### 5.2 불편 사항

1. 소액 지출이 누적되는 구조를 체감하지 못한다.
2. 카드 결제 전까지 실제 지출감을 느끼지 못한다.
3. 예산 초과 후에야 문제를 인식한다.
4. 소비를 줄이려 해도 보상감이 없다.

### 5.3 필수 요구 기능

- 변동지출 빠른 추가
- 일일 예산 초과 경고
- 오늘 사용금액과 남은금액 실시간 표시
- 반복 소비 카테고리 요약
- 예산을 지켜낸 날의 성취 표시

### 5.4 UX 적용 기준

- 지출 추가는 2단계 이내로 끝나야 한다.
- 초과 상태는 강한 색상으로 표시하되 비난하지 않는다.
- “절약 실패”보다 “내일부터 다시 납치 가능” 메시지를 제공한다.

## 6. P-03 박도윤: 아르바이트 소득자

| 항목           | 내용                                         |
| -------------- | -------------------------------------------- |
| 나이           | 23세                                         |
| 직업           | 대학생 겸 카페 아르바이트                    |
| 월 수령액      | 약 90만~140만 원 변동                        |
| 급여일         | 매월 5일, 20일 또는 주급                     |
| 주요 지출      | 교통비, 식비, 통신비, 학습비, 취업 준비비    |
| 통장 관리 방식 | 생활비 통장 중심, 저축은 비정기              |
| 소비 습관      | 급여가 들어오면 필요한 물건을 한 번에 구매   |
| 앱 사용 동기   | 다음 급여일까지 얼마를 써도 되는지 알고 싶음 |

### 6.1 핵심 생각

“월급이 정해져 있지 않아서 한 달 예산을 세우기 어렵다. 다음 급여일까지 버틸 수 있는 금액을 계산해주면 좋겠다.”

### 6.2 불편 사항

1. 급여 주기가 월급형 앱과 맞지 않는다.
2. 소득이 적어 한 번의 충동 지출이 큰 영향을 준다.
3. 고정지출은 적지만 필수 생활비 비중이 높다.
4. 저축 목표를 크게 잡으면 금방 포기한다.

### 6.3 필수 요구 기능

- 주급/격주급/월급 선택
- 다음 급여일까지 남은 일수 기준 일일 예산 계산
- 소액 납치금액 목표
- 필수 생활비 우선 배정
- 알바비 수령 회차별 기록

### 6.4 UX 적용 기준

- 월급 직장인 중심 문구만 쓰지 않는다.
- “이번 급여 회차”라는 개념을 제공한다.
- 소액 저축도 성취로 인정한다.

## 7. P-04 정하나: 자기계발형 직장인

| 항목           | 내용                                         |
| -------------- | -------------------------------------------- |
| 나이           | 29세                                         |
| 직업           | IT 기업 프론트엔드 개발자                    |
| 월 수령액      | 약 420만 원                                  |
| 급여일         | 매월 25일                                    |
| 주요 지출      | 자기계발 강의, 도서, 운동, 식비, 구독 서비스 |
| 통장 관리 방식 | 저축통장 분리, 투자 계좌 보유                |
| 소비 습관      | 계획은 세우지만 루틴 유지가 어려움           |
| 앱 사용 동기   | 돈 관리와 성장 루틴을 하나로 보고 싶음       |

### 7.1 핵심 생각

“돈을 아끼는 것도 중요하지만 결국 나를 성장시키는 데 쓰고 싶다. 관리 앱이 너무 절약만 강조하면 오래 쓰기 어렵다.”

### 7.2 불편 사항

1. 금융 앱과 루틴 앱을 따로 써야 한다.
2. 자기계발 비용이 낭비인지 투자자인지 판단하기 어렵다.
3. 목표 달성 과정이 시각적으로 남지 않는다.
4. 혼자 하면 루틴이 끊긴다.

### 7.3 필수 요구 기능

- LV UP 탭
- 독서·뉴스·영어·건강 미션
- 경험치와 레벨
- 마이페이지 성장 기록
- 레벨업 인증 게시판

### 7.4 UX 적용 기준

- 절약한 돈과 성장 활동을 같이 보여준다.
- 자기계발 지출은 일반 소비와 구분할 수 있어야 한다.
- 레벨업은 매일 1개 이상 쉽게 완료 가능해야 한다.

## 8. P-05 최재훈: 고정지출 과다 직장인

| 항목           | 내용                                               |
| -------------- | -------------------------------------------------- |
| 나이           | 38세                                               |
| 직업           | 제조업 영업관리자                                  |
| 월 수령액      | 약 390만 원                                        |
| 급여일         | 매월 21일                                          |
| 주요 지출      | 대출상환, 보험료, 통신비, 가족 생활비, 구독 서비스 |
| 통장 관리 방식 | 여러 계좌와 카드 사용, 자동이체 다수               |
| 소비 습관      | 변동지출보다 고정지출 부담이 큼                    |
| 앱 사용 동기   | 급여가 들어와도 실제로 쓸 수 있는 돈을 알고 싶음   |

### 8.1 핵심 생각

“월급이 적은 편은 아닌데 이미 빠져나갈 돈이 너무 많다. 급여일 전에 고정지출을 한눈에 보고 싶다.”

### 8.2 불편 사항

1. 자동이체와 결제일이 분산되어 있다.
2. 급여 수령 직후 실제 여유자금을 오판한다.
3. 저축과 대출상환이 뒤섞인다.
4. 고정지출을 줄일 항목을 찾기 어렵다.

### 8.3 필수 요구 기능

- 고정지출 캘린더
- 결제 예정 알림
- 고정지출 합계
- 납부완료 상태 처리
- 지출 예정금액 반영 납치금액 계산

### 8.4 UX 적용 기준

- 고정지출은 “이미 예약된 돈”으로 표현한다.
- 고정지출 등록은 항목별 템플릿을 제공한다.
- 월급 홈에서 지출금액을 명확히 분리한다.

## 9. P-06 윤지민: 커뮤니티 참여형 사용자

| 항목           | 내용                                    |
| -------------- | --------------------------------------- |
| 나이           | 26세                                    |
| 직업           | 콘텐츠 마케터                           |
| 월 수령액      | 약 300만 원                             |
| 급여일         | 매월 말일                               |
| 주요 지출      | 취미, 식비, 쇼핑, 모임비, 자기계발      |
| 통장 관리 방식 | 기록은 시도하지만 지속 실패             |
| 소비 습관      | 주변 사람의 인증과 챌린지에 영향을 받음 |
| 앱 사용 동기   | 혼자보다 함께 관리하고 인증하고 싶음    |

### 9.1 핵심 생각

“혼자 하면 금방 포기한다. 다른 직장인들이 어떻게 돈을 아끼고 성장하는지 보면 나도 따라 할 수 있을 것 같다.”

### 9.2 불편 사항

1. 돈 관리 실패를 혼자 감당한다.
2. 소비관리 팁을 현실적으로 얻기 어렵다.
3. 익명성이 없으면 급여·소비 이야기를 하기 부담스럽다.
4. 인증할 공간이 없으면 성취감이 약하다.

### 9.3 필수 요구 기능

- 자유 게시판
- 레벨업 인증 게시판
- 익명 글쓰기
- 좋아요, 댓글, 공유
- 인기 게시글

### 9.4 UX 적용 기준

- 커뮤니티는 금융 비교 경쟁장이 되면 안 된다.
- 급여 금액 노출은 선택 사항이어야 한다.
- 인증은 “얼마 벌었는가”보다 “얼마를 지켜냈는가”에 집중한다.

## 12. 제품 의사결정 적용 규칙

1. 기능 판단이 충돌하면 P-01, P-02, P-05의 급여관리 니즈를 우선한다.
2. LV UP 기능은 P-04, P-06의 리텐션 니즈를 충족하되 급여 홈을 방해하지 않는다.
3. 아르바이트생 대응은 P-03을 기준으로 급여 주기 유연성을 제공한다.
4. 커뮤니티 확장은 P-06을 기준으로 하되 민감한 급여 비교를 자극하지 않는다.
5. 온보딩은 P-01 기준으로 가장 쉬운 흐름을 만든다.

- 후속 사용자 문제 정의, 여정 지도, KPI 문서에서 참조 가능한 구조로 완성했다.
