# 타깃 사용자 정의서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼이 누구를 위해 설계되고, 누구를 우선 만족시켜야 하며, 어떤 사용자는 초기 범위에서 의도적으로 제외하는지를 최종 확정한다. 모든 화면, 기능, 문구, 온보딩, 알림, 커뮤니티, KPI, 수익화 판단은 본 타깃 사용자 정의를 기준으로 한다.

## 2. 타깃 정의 결론

급여납치의 최우선 타깃은 **정기적 또는 반복적 소득을 받지만 월급·급여·일급·주급이 소비와 고정지출에 빠르게 흩어져 남는 돈을 체감하지 못하는 직장인, 사회초년생, 아르바이트생, 소비관리 필요자**다.

급여납치는 단순 가계부 앱이 아니라 **급여 수령 시점부터 지켜야 할 돈을 먼저 분리하고, 매일의 소비·자기계발·커뮤니티 인증을 통해 월급 생활자의 자기관리 습관을 만드는 플랫폼**이다.

## 3. 핵심 타깃 사용자군

| 구분 | 사용자군               | 정의                                                             | 우선순위 | 제품 내 핵심 니즈                                 |
| ---- | ---------------------- | ---------------------------------------------------------------- | -------: | ------------------------------------------------- |
| T-01 | 정규직 직장인          | 매월 정해진 급여일에 월급을 받는 사용자                          |        1 | 급여일 기준 지출·저축·생활비 분리, 월말 잔액 관리 |
| T-02 | 사회초년생             | 첫 월급 이후 소비·저축 습관이 안정되지 않은 사용자               |        1 | 쉬운 계획 입력, 일일 예산, 소비 경고, 성취감      |
| T-03 | 아르바이트생           | 주급·격주급·월급 등 불규칙하거나 작은 소득을 받는 사용자         |        2 | 소득 회차별 예산, 생활비 방어, 소액 저축 습관     |
| T-04 | 소비통제 필요자        | 수입은 있으나 카드값·구독료·변동지출로 월말 현금이 부족한 사용자 |        1 | 고정지출 누락 방지, 오늘 쓸 수 있는 돈 확인       |
| T-05 | 자기계발 병행 사용자   | 돈 관리뿐 아니라 독서·뉴스·영어·운동 루틴을 만들고 싶은 사용자   |        2 | LV UP 미션, 성취 기록, 커뮤니티 인증              |
| T-06 | 직장인 커뮤니티 사용자 | 직장·급여·소비·자기계발 주제로 익명 소통을 원하는 사용자         |        3 | 인증 게시판, 자유게시판, 좋아요/댓글/공유         |

## 4. 1차 핵심 사용자

### 4.1 최우선 사용자

**월급 200만~400만 원대의 20~30대 직장인 및 사회초년생**을 1차 핵심 사용자로 확정한다.

### 4.2 선정 사유

1. 월급 흐름이 단순해 급여일 중심 UX를 빠르게 이해한다.
2. 구독료, 식비, 교통비, 카드값 등 반복 지출 관리 필요성이 높다.
3. 저축·자기계발·커리어 성장 욕구가 동시에 존재한다.
4. 모바일 앱 기반 기록·알림·커뮤니티 참여에 익숙하다.
5. 급여납치의 핵심 지표인 납치금액, 일일 예산, 레벨업 구조를 가장 빠르게 학습한다.

## 5. 사용자 세그먼트 상세 정의

### 5.1 S1. 월급 소진형 직장인

| 항목         | 정의                                                       |
| ------------ | ---------------------------------------------------------- |
| 대표 상황    | 월급을 받으면 카드값, 구독료, 식비, 쇼핑으로 빠르게 소진됨 |
| 핵심 문제    | 월말에 왜 돈이 없는지 알지만 행동을 바꾸지 못함            |
| 급여 패턴    | 매월 고정 급여, 1개 주요 급여일                            |
| 소비 패턴    | 소액 반복 소비, 충동구매, 자동결제 누락                    |
| 앱 사용 동기 | 오늘 쓸 수 있는 돈을 즉시 알고 싶음                        |
| 필수 기능    | 급여 홈, 일일 예산, 변동지출 기록, 초과 알림               |

### 5.2 S2. 계획형 사회초년생

| 항목         | 정의                                              |
| ------------ | ------------------------------------------------- |
| 대표 상황    | 첫 월급 이후 저축·소비·투자 기준을 잡고 싶음      |
| 핵심 문제    | 어디까지 저축하고 얼마를 써야 하는지 모름         |
| 급여 패턴    | 월급 고정, 상여·성과급은 비정기                   |
| 소비 패턴    | 친구 약속, 구독, 배달, 취미 지출이 많음           |
| 앱 사용 동기 | 월급 관리 루틴을 처음부터 제대로 만들고 싶음      |
| 필수 기능    | 급여 계획, 고정저축, 목표 달성률, 납치금액 시각화 |

### 5.3 S3. 아르바이트·파트타임 소득자

| 항목         | 정의                                          |
| ------------ | --------------------------------------------- |
| 대표 상황    | 주급·격주급·월급 등 소득 주기가 다름          |
| 핵심 문제    | 작은 소득이 생활비에 흩어져 저축을 못함       |
| 급여 패턴    | 불규칙 또는 복수 급여일                       |
| 소비 패턴    | 식비, 교통비, 통신비, 학업/취업 준비 비용     |
| 앱 사용 동기 | 다음 급여 전까지 버틸 수 있는 돈을 알고 싶음  |
| 필수 기능    | 급여 회차 관리, 일일 예산, 필수지출 우선 배정 |

### 5.4 S4. 고정지출 누락형 사용자

| 항목         | 정의                                                      |
| ------------ | --------------------------------------------------------- |
| 대표 상황    | 구독료·대출·보험·통신비가 자동으로 빠져나가 계획이 무너짐 |
| 핵심 문제    | 월급에서 이미 빠져나갈 돈을 사전에 체감하지 못함          |
| 급여 패턴    | 정기 급여 중심                                            |
| 소비 패턴    | 고정지출 비중이 높고 관리표가 없음                        |
| 앱 사용 동기 | 결제 예정일과 남는 돈을 미리 보고 싶음                    |
| 필수 기능    | 고정지출 관리, 결제 예정 알림, 납부완료 상태              |

### 5.5 S5. 자기계발 루틴형 사용자

| 항목         | 정의                                                  |
| ------------ | ----------------------------------------------------- |
| 대표 상황    | 돈 관리와 함께 독서·뉴스·영어·운동을 꾸준히 하고 싶음 |
| 핵심 문제    | 목표는 많지만 지속률이 낮음                           |
| 급여 패턴    | 직장인 또는 취업 준비 병행 소득자                     |

| 앱 사용 동기 | 돈을 지켜낸 성취와 자기계발 성취를 함께 보고 싶음     |
| 필수 기능    | LV UP, 경험치, 레벨, 인증 게시판, 알림                |

## 6. 비타깃 및 초기 제외 사용자

| 하루를 시작할 때 | 오늘 얼마까지 써도 되는지 알고 싶다                   | 일일 예산과 남은 금액 제공                     |

| 루틴이 무너질 때 | 돈 관리와 자기계발을 다시 시작하고 싶다               | 레벨업 미션, 알림, 커뮤니티 인증 제공          |

## 9. 온보딩 기준

### 9.1 필수 온보딩 질문

1. 급여 유형: 월급, 주급, 격주급, 일급, 기타
2. 급여일 또는 수령 주기
3. 수령 예정 급여
4. 고정지출 여부
5. 고정저축 여부
6. 하루 생활비 목표
7. 자기계발 관심 분야: 독서, 뉴스, 영어, 건강

### 9.2 온보딩 완료 조건

사용자가 아래 4개 값을 입력하면 온보딩을 완료로 간주한다.

1. 급여 유형
2. 수령 예정 금액
3. 급여일 또는 수령 주기
4. 일일 생활비 기준

고정지출, 고정저축, 자기계발 관심사는 온보딩 완료 후 추가 입력 가능하다.

## 10. 기능 우선순위 반영 기준

| 우선순위 | 기준                                | 해당 기능                                            |
| -------- | ----------------------------------- | ---------------------------------------------------- |
| P1       | 모든 핵심 타깃이 반드시 사용해야 함 | 로그인, 급여계획, 지출기록, 일일 예산, 납치금액 계산 |
| P2       | 사용자 유지와 습관 형성에 필요함    | 알림, 목표 달성률, LV UP, 마이페이지 성과            |
| P3       | 네트워크 효과와 수익화에 필요함     | 커뮤니티, 광고/제휴, 포인트 이벤트                   |
| P4       | 고도화 단계에서 가치가 커짐         | 자동 자산연동, AI 추천, 외부 금융 API 연동           |

## 11. UX 판단 원칙

1. 사용자는 돈 관리 전문가가 아니므로 모든 금액은 쉽게 보여준다.
2. 급여 홈은 사용자가 오늘 행동할 수 있는 숫자만 우선 노출한다.
3. “전체 자산”보다 “이번 급여에서 지켜낼 돈”을 먼저 보여준다.
4. 입력은 짧고 반복은 자동화한다.
5. 실패감보다 회복감을 주는 문구를 사용한다.
6. 자기계발 기능은 금융관리의 보조 루틴으로 배치한다.
7. 커뮤니티는 비교 불안을 만들지 않고 인증·응원 중심으로 운영한다.

## 12. 최종 승인 기준

본 타깃 사용자 정의서는 다음 조건을 충족하므로 최종본으로 확정한다.

- 직장인, 사회초년생, 아르바이트생, 소비관리 필요자를 명확히 구분했다.
- 1차 핵심 사용자와 후순위 사용자를 분리했다.
- 비타깃 사용자를 정의해 제품 범위 확장을 통제했다.
- 기능 우선순위와 UX 판단 기준을 타깃 사용자와 연결했다.
- 후속 문서인 페르소나, 사용자 문제 정의, 여정 지도, 포지셔닝, KPI 문서의 기준 사용자군을 확정했다.
