# 사용자 여정 지도 최종본

## 1. 문서 목적

본 문서는 급여납치 사용자가 급여일 전, 급여일 당일, 급여일 후, 월중, 월말까지 어떤 생각과 행동을 하는지 정의하고, 각 시점에서 플랫폼이 제공해야 할 화면·기능·알림·성과 지표를 확정한다.

## 2. 대표 여정 결론

급여납치의 핵심 여정은 **급여 수령 전 계획 → 급여 수령 직후 납치금액 확인 → 일일 예산 관리 → 지출 기록 → 레벨업 루틴 → 월말 확정 성과 → 다음 달 계획 개선**이다.

이 여정은 단순 기록형 가계부가 아니라 **월급이 사라지기 전에 먼저 확보하고, 매일의 행동으로 지켜내고, 월말에 성취로 확정하는 구조**다.

## 3. 전체 사용자 여정 요약

| 단계 | 시점          | 사용자 상태                  | 핵심 행동                 | 급여납치 제공 가치         |
| ---- | ------------- | ---------------------------- | ------------------------- | -------------------------- |
| J-01 | 최초 진입     | 돈 관리 필요 인식            | 가입/로그인               | 쉽고 빠른 시작             |
| J-02 | 급여일 전     | 이번 달 계획 필요            | 급여·고정지출·저축 입력   | 예상 납치금액 산출         |
| J-03 | 급여일 당일   | 월급 입금으로 소비 유혹 증가 | 수령금액 확인             | 지켜야 할 돈 선점          |
| J-04 | 급여일 후 1주 | 지출이 가장 많이 발생        | 일일 예산 확인·지출 기록  | 오늘 써도 되는 돈 표시     |
| J-05 | 월중          | 루틴 약화                    | 알림·레벨업·커뮤니티 인증 | 재방문 동기 제공           |
| J-06 | 월말          | 성과 확인                    | 확정 납치금액 확인        | 성취감과 다음 달 기준 제공 |
| J-07 | 다음 급여 전  | 개선 필요                    | 지난달 계획 복사/수정     | 반복 관리 루틴 형성        |

## 4. 상세 여정 지도

### 4.1 J-01 최초 진입

| 항목        | 내용                                                                |
| ----------- | ------------------------------------------------------------------- |
| 사용자 생각 | “이번에는 월급 관리를 제대로 해보고 싶다.”                          |
| 사용자 감정 | 기대, 약간의 부담                                                   |
| 행동        | 앱 설치, 로그인, 소셜 로그인 선택                                   |
| 접점 화면   | 스플래시, 로그인                                                    |
| 제품 목표   | 30초 이내 홈 또는 온보딩 진입                                       |
| 필요한 기능 | 일반 로그인, SNS 로그인, 자동 로그인, 회원가입                      |
| 실패 위험   | 가입 절차가 길면 이탈                                               |
| UX 처방     | 소셜 로그인 버튼을 명확히 제공하고, 첫 입력은 급여 정보로 바로 연결 |
| 측정 KPI    | 가입 완료율, 로그인 성공률, 온보딩 시작률                           |

| 항목        | 내용                                                         |

## 5. 사용자 감정 곡선

| 단계          |  감정 강도 | 감정 상태 | 제품 개입                   |
| ------------- | ---------: | --------- | --------------------------- |
| 최초 진입     |         +2 | 기대      | 쉬운 시작, 명확한 가치 문구 |
| 급여일 전     |         -1 | 부담      | 간단한 계획 입력            |
| 급여일 당일   |         +3 | 기쁨/유혹 | 납치금액 선점 표시          |
| 급여일 후 1주 |          0 | 통제/긴장 | 일일 예산과 빠른 기록       |
| 월중          |         -2 | 지루함    | LV UP, 알림, 커뮤니티       |
| 월말          | +1 또는 -1 | 평가      | 성취/회복 메시지            |
| 다음 달       |         +2 | 재도전    | 계획 복사와 개선 제안       |

## 6. 핵심 사용자 플로우

### 6.1 신규 사용자 기본 플로우

1. 앱 실행
2. 로그인 또는 소셜 로그인

## 10. 최종 승인 기준

본 사용자 여정 지도는 다음 조건을 충족하므로 최종본으로 확정한다.

- 급여일 전, 급여일 당일, 급여일 후, 월말 상태를 모두 포함했다.
