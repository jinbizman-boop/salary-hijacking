# 핵심 KPI 정의서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 성과를 판단하기 위한 핵심 KPI, 보조 KPI, 계산식, 이벤트 로그, 측정 주기, 목표 기준을 최종 정의한다. 모든 기능 개발과 운영 개선은 본 KPI 체계를 기준으로 효과를 판단한다.

## 2. KPI 설계 원칙

1. 단순 다운로드보다 실제 급여관리 행동을 우선 측정한다.
2. 지출 기록 수보다 납치금액 확정과 반복 사용을 더 중요하게 본다.
3. 금융관리, 자기계발, 커뮤니티, 수익화, 안정성을 분리해 측정한다.
4. 모든 KPI는 이벤트 로그로 재현 가능해야 한다.
5. KPI는 사용자에게 해로운 과소비, 과도한 알림, 민감정보 노출을 유도해서는 안 된다.

## 3. North Star Metric

### 3.1 최종 North Star Metric

**월간 확정 납치금액 생성 사용자 수, MCHU: Monthly Confirmed Hijack Users**

### 3.2 정의

월간 확정 납치금액 생성 사용자 수는 한 달 동안 아래 조건을 모두 충족한 고유 사용자 수다.

1. 급여계획을 1회 이상 등록 또는 유지했다.
2. 해당 월에 지출 기록 또는 고정지출 확인 행동을 3회 이상 수행했다.
3. 월말 또는 급여 회차 종료 시 확정 납치금액을 산출했다.

### 3.3 선정 이유

| 이유           | 설명                                                               |
| -------------- | ------------------------------------------------------------------ |
| 핵심 가치 반영 | 사용자가 실제로 월급에서 지켜낸 돈을 확인해야 서비스 가치가 발생함 |
| 반복성 반영    | 단순 가입이 아니라 월 단위 루틴을 측정함                           |
| 수익화 연결    | 활성 사용자가 많을수록 광고, 제휴, 프리미엄 전환 가능성이 높아짐   |
| 차별화 반영    | 기존 가계부의 기록 수가 아니라 급여납치 고유 지표를 측정함         |

### 3.4 계산식

```text
MCHU = COUNT(DISTINCT user_id)
WHERE payroll_plan_active = true
AND monthly_finance_action_count >= 3
AND confirmed_hijack_amount_created = true
AND month = target_month
```

## 15. 개인정보 보호형 KPI 처리 원칙

1. 대시보드에는 개인별 실제 급여액을 그대로 노출하지 않는다.
2. 금액은 구간화하여 분석한다.
3. 광고·제휴 분석에는 민감 금융정보를 직접 식별자로 사용하지 않는다.
4. 커뮤니티 데이터와 개인 급여 데이터는 분석 테이블을 분리한다.
5. 내부 운영자는 필요한 권한이 없는 경우 개별 급여·지출 데이터를 조회할 수 없다.

## 19. 최종 승인 기준

본 핵심 KPI 정의서는 다음 조건을 충족하므로 최종본으로 확정한다.

- 가입률, 급여계획 등록률, 일일 지출 기록률, 재방문율, 납치금액 증가율을 모두 포함했다.
- North Star Metric을 급여납치 고유 가치인 월간 확정 납치금액 생성 사용자 수로 확정했다.
- 금융관리, 자기계발, 커뮤니티, 수익화, 안정성 KPI를 분리했다.
- 이벤트 로그와 계산식을 정의해 개발·분석 구현 기준으로 사용할 수 있다.
- 개인정보 보호형 KPI 처리 원칙을 포함했다.
