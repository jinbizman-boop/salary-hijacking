# 04. 파일 업로드 명세서 최종본

## 1. 문서 목적

본 문서는 급여납치 커뮤니티 글쓰기, 프로필 이미지, 공지/배너 운영에서 사용하는 파일 업로드 기준을 최종 확정한다. 파일 저장소는 Cloudflare R2를 기준으로 하며, API 서버는 파일을 직접 중계하지 않고 사전 서명 URL 기반 업로드를 사용한다.

## 2. 적용 위치

| 위치            | 파일 유형         | 필수 여부 |
| --------------- | ----------------- | --------: |
| 커뮤니티 글쓰기 | 이미지, 문서 첨부 |      선택 |
| 댓글            | 이미지 첨부       |      선택 |
| 프로필 설정     | 프로필 이미지     |      선택 |
| 관리자 공지     | 이미지/문서       |      선택 |
| 광고/제휴 배너  | 배너 이미지       |      필수 |
| 레벨업 콘텐츠   | 썸네일 이미지     |      선택 |

## 3. 저장 구조

| 항목           | 기준                                                            |
| -------------- | --------------------------------------------------------------- |
| Object Storage | Cloudflare R2                                                   |
| 공개 접근      | 기본 비공개, 서명 URL 또는 CDN 공개 URL을 파일 유형별로 구분    |
| CDN 도메인     | `https://cdn.salary-hijacking.app`                              |
| Object Key     | `{env}/{domain}/{yyyy}/{mm}/{userId 또는 adminId}/{uuid}.{ext}` |
| 메타데이터 DB  | `files` 테이블과 연결 테이블 사용                               |
| 삭제           | DB soft delete 후 object delete queue 처리                      |

## 4. 허용 파일 정책

| 구분          | 허용 확장자               | MIME                                         | 최대 크기 |     최대 개수 |
| ------------- | ------------------------- | -------------------------------------------- | --------: | ------------: |
| 프로필 이미지 | jpg, jpeg, png, webp      | image/jpeg, image/png, image/webp            |       5MB |             1 |
| 게시글 이미지 | jpg, jpeg, png, webp, gif | image/jpeg, image/png, image/webp, image/gif |      10MB | 게시글당 10개 |
| 게시글 문서   | pdf                       | application/pdf                              |      20MB |  게시글당 3개 |
| 공지 이미지   | jpg, jpeg, png, webp      | image/\*                                     |      10MB |    공지당 5개 |
| 광고 배너     | jpg, jpeg, png, webp      | image/\*                                     |       5MB | 캠페인당 20개 |

금지: exe, apk, ipa, js, html, svg, zip, rar, 7z, macro 포함 office 문서, 비밀번호 압축 파일, MIME과 확장자가 불일치하는 파일.

## 5. 업로드 플로우

1. 클라이언트가 업로드할 파일명, MIME, 크기, 용도를 서버에 전달한다.
2. 서버는 사용자 권한, 크기, MIME, 확장자, 업로드 횟수 제한을 검증한다.
3. 서버는 `PENDING` 상태의 파일 메타데이터를 생성한다.
4. 서버는 R2 사전 서명 PUT URL을 발급한다.
5. 클라이언트가 R2로 직접 업로드한다.
6. 클라이언트가 `/files/complete`를 호출한다.
7. 서버는 object 존재 여부와 크기를 검증한다.
8. 이미지인 경우 썸네일 생성 작업을 queue에 등록한다.
9. 파일 상태를 `READY`로 변경한다.
10. 게시글/프로필/배너 저장 시 `attachmentIds`로 연결한다.

## 6. API 명세

### 6.1 업로드 URL 발급

`POST /api/v1/files/upload-url`

```json
{
  "purpose": "COMMUNITY_POST",
  "originalFileName": "workout-proof.png",
  "contentType": "image/png",
  "fileSize": 2481200
}
```

Response `201`

```json
{
  "success": true,
  "data": {
    "fileId": "uuid-file-001",
    "uploadUrl": "https://r2-upload-presigned-url",
    "method": "PUT",
    "headers": {
      "Content-Type": "image/png"
    },
    "objectKey": "prod/community/2026/06/user-uuid/uuid-file-001.png",
    "expiresInSeconds": 600
  }
}
```

### 6.2 업로드 완료 등록

`POST /api/v1/files/complete`

```json
{
  "fileId": "uuid-file-001",
  "objectKey": "prod/community/2026/06/user-uuid/uuid-file-001.png"
}
```

Response

```json
{
  "success": true,
  "data": {
    "fileId": "uuid-file-001",
    "status": "READY",
    "fileUrl": "https://cdn.salary-hijacking.app/prod/community/2026/06/user-uuid/uuid-file-001.png",
    "thumbnailUrl": null
  }
}
```

### 6.3 파일 삭제

`DELETE /api/v1/files/{fileId}`

정책: 파일 소유자 또는 관리자만 삭제 가능하다. 커뮤니티 게시글에 연결된 파일은 게시글 삭제 시 함께 soft delete된다.

## 7. 파일 상태값

| 상태    | 의미                        | 다음 상태               |
| ------- | --------------------------- | ----------------------- |
| PENDING | URL 발급 후 업로드 대기     | READY, EXPIRED, DELETED |
| READY   | 업로드 완료 및 사용 가능    | LINKED, DELETED         |
| LINKED  | 게시글/프로필/배너에 연결됨 | DELETED                 |
| EXPIRED | 제한 시간 내 업로드 미완료  | DELETED                 |
| BLOCKED | 보안/운영 정책 위반         | DELETED                 |
| DELETED | 삭제 완료 또는 삭제 예약    | 없음                    |

## 8. 썸네일 정책

| 원본 유형     | 썸네일                                 |
| ------------- | -------------------------------------- |
| 프로필 이미지 | 256x256 webp                           |
| 게시글 이미지 | 320px width webp, 원본 비율 유지       |
| 광고 배너     | placement별 사이즈 검증 후 webp 생성   |
| GIF           | 첫 프레임 정적 썸네일 생성             |
| PDF           | 썸네일 생성하지 않음, 문서 아이콘 사용 |

## 9. 보안 정책

1. 서버는 클라이언트가 보낸 MIME만 신뢰하지 않고 object metadata와 magic byte를 검증한다.
2. 파일명은 사용자 입력 원본을 저장하되 object key에는 사용하지 않는다.
3. 공개 URL은 사용자 개인정보가 포함되지 않는 key만 사용한다.
4. 바이러스/악성 파일 스캔이 실패하면 `BLOCKED` 상태로 전환한다.
5. 미사용 PENDING 파일은 24시간 후 자동 삭제한다.
6. 탈퇴 사용자의 프로필 이미지는 즉시 비활성화하고, 보존 대상 커뮤니티 첨부는 익명화 정책을 따른다.

## 10. 오류 코드

| 코드                    | HTTP | 상황                      |
| ----------------------- | ---: | ------------------------- |
| FILE_SIZE_EXCEEDED      |  413 | 파일 크기 초과            |
| FILE_TYPE_NOT_ALLOWED   |  415 | 허용하지 않는 확장자/MIME |
| FILE_UPLOAD_URL_EXPIRED |  410 | 사전 서명 URL 만료        |
| FILE_NOT_FOUND          |  404 | 파일 없음 또는 접근 불가  |
| FILE_OWNER_MISMATCH     |  403 | 파일 소유자 불일치        |
| FILE_SCAN_FAILED        |  422 | 보안 스캔 실패            |
| FILE_COUNT_EXCEEDED     |  422 | 첨부 개수 초과            |
