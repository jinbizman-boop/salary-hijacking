# 일일 예산 관리 명세서 최종본

## 1. 문서 목적

본 문서는 사용자의 하루 소비 가능 금액을 설정하고, 사용금액과 남은금액을 실시간으로 비교하는 일일 예산 관리 기능을 최종 확정한다. 일일 예산은 급여납치 앱의 매일 반복 사용을 만드는 핵심 기능이다.

## 공통 전제

| 구분        | 최종 기준                                                                     |
| ----------- | ----------------------------------------------------------------------------- |
| 서비스 유형 | 급여관리·가계부·자기계발·커뮤니티 결합형 모바일 애플리케이션                  |
| 핵심 사용자 | 직장인, 사회초년생, 아르바이트생, 소비관리 필요자                             |
| 핵심 지표   | 수령금액, 지출금액, 납치금액, 누적 납치금액, 목표 달성률, 일일 남은금액, 레벨 |

| 개인정보    | 급여·대출·저축·소비 내역은 민감 금융성 생활 데이터로 분류                     |
| 광고 데이터 | 급여·대출·저축 원천 데이터와 광고 추천/클릭 로그는 분리 저장                  |

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |
| -------- | ------------------------------ | ------------------------------------ |
| P1       | 핵심 급여관리 기능             | MVP 출시 전 반드시 완료              |
| P2       | 유지율·커뮤니티·운영 핵심 기능 | 정식 출시 전 완료 또는 단계적 활성화 |
| P3       | 수익화·고도화 기능             | 안정화 이후 활성화 가능              |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 3. 입력/출력 필드

| 항목            | 필수 | 타입    | 설명           | 검증            |
| --------------- | ---- | ------- | -------------- | --------------- |
| dailyLimit      | Y    | integer | 일일 설정금액  | 1원 이상        |
| date            | Y    | date    | 적용일         | Asia/Seoul 기준 |
| usedAmount      | Y    | integer | 오늘 사용금액  | 시스템 계산     |
| remainingAmount | Y    | integer | 오늘 남은금액  | 시스템 계산     |
| status          | Y    | enum    | 정상/소진/초과 | 시스템 계산     |

## 4. 사용자 및 시스템 흐름

### 1. 예산 설정

사용자는 계획 화면에서 일일 생활비를 입력한다. 시스템은 월별 생활비 계획과 DailyBudget 기본값을 생성한다.

### 2. 홈 표시

홈 화면은 오늘 날짜의 설정금액, 사용금액, 남은금액을 표시한다.

### 3. 지출 반영

변동지출이 추가되면 사용금액과 남은금액을 즉시 재계산한다.

### 4. 초과 처리

남은금액이 0원 미만이면 초과 상태와 알림을 생성한다.

## 5. 검증 규칙

| 검증 항목     | 기준                 | 실패 처리                        |
| ------------- | -------------------- | -------------------------------- |
| 일일 설정금액 | 1원 이상             | 일일 예산을 숫자로 입력해주세요. |
| 날짜          | 유효 날짜            | 날짜를 확인해주세요.             |
| 중복 예산     | 사용자·날짜 기준 1개 | 이미 해당 날짜 예산이 있습니다.  |

## 6. 예외 처리

| 예외 상황   | 처리 방식             | 사용자 안내                    |
| ----------- | --------------------- | ------------------------------ |
| 예산 미설정 | 기본 설정 유도        | 일일 예산을 먼저 설정해주세요. |
| 자정 변경   | 새 날짜 예산으로 전환 |                                |
| 초과 상태   | 저장 허용, 경고 표시  | 오늘 예산을 초과했습니다.      |

| ------------- | ------------------- | ----------------------------- |
| 예산 저장     | DailyBudget 생성    | 20,000원 저장 후 홈 표시      |
| 사용금액 반영 | 지출 추가 즉시 변경 | 13,000원 사용 시 남은 7,000원 |

## 10. 보안·개인정보 기준

1. 본 기능에서 생성되는 사용자 개인 데이터는 로그인 사용자 본인만 조회·수정·삭제할 수 있다.
2. 급여, 지출, 저축, 소비, 프로필, 알림 수신 설정은 사용자 식별 데이터와 분리 가능한 구조로 관리한다.
3. 관리자 또는 운영자 접근이 필요한 경우 역할 기반 권한을 적용하고 모든 조작을 감사 로그로 저장한다.
4. 광고·제휴 분석에 민감 급여 데이터를 직접 사용하지 않는다.

## 11. 최종 완료 판정

본 문서는 일일 예산 관리 명세서의 구현 범위, 입력값, 출력값, 검증, 예외, 데이터, 수락 기준을 모두 확정했다. 본 문서 기준으로 추가 기획 없이 구현·테스트·검수가 가능하므로 문서상·이론상 추가 작업은 필요하지 않다.
