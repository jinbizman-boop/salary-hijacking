# 광고/제휴 배너 기능 명세서 최종본

## 1. 문서 목적

본 문서는 홈, LV UP, 마이페이지 등 광고 영역에서 배너를 노출하고, 노출/클릭 로그와 랜딩 URL 이동을 처리하는 광고·제휴 기능을 최종 확정한다. 광고 기능은 서비스 수익화 기능이지만 민감 급여 데이터 보호 원칙을 반드시 따른다.

## 공통 전제

| 구분        | 최종 기준                                                                     |
| ----------- | ----------------------------------------------------------------------------- |
| 서비스 유형 | 급여관리·가계부·자기계발·커뮤니티 결합형 모바일 애플리케이션                  |
| 핵심 사용자 | 직장인, 사회초년생, 아르바이트생, 소비관리 필요자                             |
| 핵심 지표   | 수령금액, 지출금액, 납치금액, 누적 납치금액, 목표 달성률, 일일 남은금액, 레벨 |
| 기본 통화   | KRW, 원 단위 정수                                                             |
| 금액 표시   | 천 단위 콤마 + `원` 표기                                                      |
| 금액 저장   | 숫자형 정수 저장, 화면 표시 시 포맷팅                                         |
| 음수 처리   | 납치금액은 0원 미만 표시 금지, 남은금액은 초과 상태 판단을 위해 음수 허용     |
| 인증 상태   | 비로그인 사용자는 로그인/회원가입/스플래시 외 핵심 기능 접근 불가             |
| 개인정보    | 급여·대출·저축·소비 내역은 민감 금융성 생활 데이터로 분류                     |
| 광고 데이터 | 급여·대출·저축 원천 데이터와 광고 추천/클릭 로그는 분리 저장                  |

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |
| -------- | ------------------------------ | ------------------------------------ |
| P1       | 핵심 급여관리 기능             | MVP 출시 전 반드시 완료              |
| P2       | 유지율·커뮤니티·운영 핵심 기능 | 정식 출시 전 완료 또는 단계적 활성화 |
| P3       | 수익화·고도화 기능             | 안정화 이후 활성화 가능              |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 3. 입력/출력 필드

| 항목          | 필수 | 타입     | 설명          | 검증        |
| ------------- | ---- | -------- | ------------- | ----------- |
| campaignId    | Y    | string   | 캠페인 ID     | 존재 캠페인 |
| placement     | Y    | enum     | 노출 위치     | 허용값      |
| imageUrl      | Y    | url      | 배너 소재     | 유효 URL    |
| landingUrl    | Y    | url      | 클릭 랜딩     | https 권장  |
| startAt/endAt | Y    | datetime | 캠페인 기간   | 유효 기간   |
| priority      | Y    | integer  | 노출 우선순위 | 0 이상      |

## 4. 사용자 및 시스템 흐름

### 1. 광고 조회

화면 진입 시 placement 기준으로 활성 캠페인을 조회한다.

### 2. 노출

활성 소재를 표시하고 노출 로그를 저장한다.

### 3. 클릭

사용자가 배너를 클릭하면 클릭 로그 저장 후 랜딩 URL로 이동한다.

### 4. 미노출

활성 캠페인이 없으면 광고 영역을 숨기거나 기본 콘텐츠 영역으로 대체한다.

## 5. 검증 규칙

| 검증 항목   | 기준                  | 실패 처리                       |
| ----------- | --------------------- | ------------------------------- |
| 캠페인 기간 | 현재 시간이 기간 내   | 활성 기간이 아닙니다.           |
| 소재 URL    | 유효 URL              | 광고 소재를 불러올 수 없습니다. |
| 랜딩 URL    | 유효 URL              | 이동할 수 없는 링크입니다.      |
| 민감 타겟팅 | 별도 동의 없이는 금지 |                                 |

## 6. 예외 처리

| 예외 상황      | 처리 방식                             | 사용자 안내                 |
| -------------- | ------------------------------------- | --------------------------- |
| 캠페인 없음    | 광고 영역 미노출                      | 표시할 광고가 없습니다.     |
| 소재 로딩 실패 | 대체 영역 표시                        | 광고를 불러오지 못했습니다. |
| 클릭 로그 실패 | 사용자 이동은 허용하되 실패 로그 저장 |                             |

## 7. 데이터 객체

| 데이터 객체  | 주요 필드                                                                   | 처리 기준 |
| ------------ | --------------------------------------------------------------------------- | --------- |
| AdCampaign   | id, name, placement, imageUrl, landingUrl, startAt, endAt, priority, status | 캠페인    |
| AdImpression | campaignId, userId, placement, shownAt                                      | 노출 로그 |
| AdClick      | campaignId, userId, placement, clickedAt                                    | 클릭 로그 |

## 8. 수락 기준

| 수락 기준 | 완료 판정           | 테스트 조건            |
| --------- | ------------------- | ---------------------- |
| 노출 로그 | AdImpression 저장   | 화면 진입 후 로그 확인 |
| 클릭 로그 | AdClick 저장        | 배너 클릭 후 로그 확인 |
| 랜딩 이동 | 외부/내부 링크 이동 | landingUrl 도달 확인   |

## 10. 보안·개인정보 기준

1. 본 기능에서 생성되는 사용자 개인 데이터는 로그인 사용자 본인만 조회·수정·삭제할 수 있다.
2. 급여, 지출, 저축, 소비, 프로필, 알림 수신 설정은 사용자 식별 데이터와 분리 가능한 구조로 관리한다.
3. 관리자 또는 운영자 접근이 필요한 경우 역할 기반 권한을 적용하고 모든 조작을 감사 로그로 저장한다.
4. 광고·제휴 분석에 민감 급여 데이터를 직접 사용하지 않는다.

## 11. 최종 완료 판정

본 문서는 광고/제휴 배너 기능 명세서의 구현 범위, 입력값, 출력값, 검증, 예외, 데이터, 수락 기준을 모두 확정했다. 본 문서 기준으로 추가 기획 없이 구현·테스트·검수가 가능하므로 문서상·이론상 추가 작업은 필요하지 않다.
