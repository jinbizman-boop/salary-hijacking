# 급여 관리 기능 명세서 최종본

## 1. 문서 목적

본 문서는 급여일, 수령 예정 급여, 실제 수령 급여, 목표 납치금액, 급여 주기, 급여 계획 수정 기능을 최종 확정한다. 급여 관리는 급여납치 플랫폼의 시작점이며 모든 지출·예산·납치금액 계산의 기준 데이터다.

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 4. 사용자 및 시스템 흐름

### 1. 신규 급여 계획 생성

사용자는 초기 설정에서 급여일과 수령 예정 급여를 입력한다. 시스템은 입력값을 검증한 뒤 PayrollPlan을 생성하고 홈 화면으로 이동시킨다.

### 2. 급여 계획 수정

사용자는 계획 화면에서 급여 금액과 목표금액을 수정한다. 시스템은 예상 납치금액과 목표 달성률을 재계산한다.

### 3. 실제 급여 확정

월 마감 시 사용자는 실제 수령 급여를 확정할 수 있다. 실제값이 존재하면 집계와 마이페이지는 실제값을 우선 사용한다.

### 4. 급여일 변경

급여일 변경은 변경 시점 이후의 계획에 적용한다. 과거 확정 데이터는 변경하지 않는다.

| 중복 계획      | 사용자·월 기준 1개 기본 | 이미 해당 월의 급여 계획이 있습니다.  |

## 6. 예외 처리

| 예외 상황                         | 처리 방식                    | 사용자 안내                             |
| --------------------------------- | ---------------------------- | --------------------------------------- |
| 급여보다 지출이 큼                | 저장 허용, 납치금액 0원 처리 | 지출 예정 금액이 급여보다 큽니다.       |
| 급여일 31일인데 해당 월 31일 없음 | 말일로 보정                  | 해당 월은 말일 기준으로 적용됩니다.     |
| 네트워크 오류                     | 임시 저장 또는 재시도        | 저장에 실패했습니다. 다시 시도해주세요. |

## 9. API 기준

### 급여 계획 생성

```http
POST /api/payroll-plans
Content-Type: application/json

{"planMonth":"2026-06","payday":25,"expectedSalary":2700000,"targetHijackAmount":500000}
```

### 급여 계획 수정

```http
PATCH /api/payroll-plans/{planId}
Content-Type: application/json

{"expectedSalary":2800000}
```

## 10. 보안·개인정보 기준

1. 본 기능에서 생성되는 사용자 개인 데이터는 로그인 사용자 본인만 조회·수정·삭제할 수 있다.
2. 급여, 지출, 저축, 소비, 프로필, 알림 수신 설정은 사용자 식별 데이터와 분리 가능한 구조로 관리한다.
3. 관리자 또는 운영자 접근이 필요한 경우 역할 기반 권한을 적용하고 모든 조작을 감사 로그로 저장한다.
4. 광고·제휴 분석에 민감 급여 데이터를 직접 사용하지 않는다.
