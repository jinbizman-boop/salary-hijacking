# 레벨업 기능 명세서 최종본

## 1. 문서 목적

본 문서는 독서, 뉴스, 영어, 건강/홈트 미션과 경험치, 레벨, 완료 이력, 추천 콘텐츠 표시 기준을 최종 확정한다. 레벨업 기능은 급여관리 앱을 자기계발 플랫폼으로 확장하는 핵심 리텐션 장치다.

## 공통 전제

| 구분        | 최종 기준                                                                     |
| ----------- | ----------------------------------------------------------------------------- |
| 서비스 유형 | 급여관리·가계부·자기계발·커뮤니티 결합형 모바일 애플리케이션                  |
| 핵심 사용자 | 직장인, 사회초년생, 아르바이트생, 소비관리 필요자                             |
| 핵심 지표   | 수령금액, 지출금액, 납치금액, 누적 납치금액, 목표 달성률, 일일 남은금액, 레벨 |
| 기본 통화   | KRW, 원 단위 정수                                                             |
| 금액 표시   | 천 단위 콤마 + `원` 표기                                                      |

| 음수 처리   | 납치금액은 0원 미만 표시 금지, 남은금액은 초과 상태 판단을 위해 음수 허용     |
| 인증 상태   | 비로그인 사용자는 로그인/회원가입/스플래시 외 핵심 기능 접근 불가             |
| 개인정보    | 급여·대출·저축·소비 내역은 민감 금융성 생활 데이터로 분류                     |
| 광고 데이터 | 급여·대출·저축 원천 데이터와 광고 추천/클릭 로그는 분리 저장                  |

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |
| -------- | ------------------------------ | ------------------------------------ |
| P1       | 핵심 급여관리 기능             | MVP 출시 전 반드시 완료              |
| P2       | 유지율·커뮤니티·운영 핵심 기능 | 정식 출시 전 완료 또는 단계적 활성화 |
| P3       | 수익화·고도화 기능             | 안정화 이후 활성화 가능              |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 3. 입력/출력 필드

| 항목        | 필수 | 타입     | 설명                | 검증        |
| ----------- | ---- | -------- | ------------------- | ----------- |
| taskType    | Y    | enum     | 독서/뉴스/영어/건강 | 허용값      |
| contentId   | Y    | string   | 콘텐츠 ID           | 존재 콘텐츠 |
| status      | Y    | enum     | 미완료/완료         | 허용값      |
| expReward   | Y    | integer  | 경험치              | 0 이상      |
| completedAt | N    | datetime | 완료 시간           | 시스템 처리 |
| level       | Y    | integer  | 사용자 레벨         | 1~999       |

## 4. 사용자 및 시스템 흐름

### 1. 미션 조회

사용자는 LV UP 화면에서 오늘의 독서, 뉴스, 영어, 홈트 미션을 확인한다.

### 2. 카테고리 탐색

독서/뉴스 화면은 카테고리 탭 전환을 지원한다.

### 3. 완료 처리

완료 버튼을 누르면 GrowthTask 완료 이력을 저장하고 경험치를 지급한다.

### 4. 레벨업

경험치가 기준치를 넘으면 레벨을 증가시키고 토스트/알림을 표시한다.

## 5. 검증 규칙

| 검증 항목 | 기준                   | 실패 처리                      |
| --------- | ---------------------- | ------------------------------ |
| 미션 유형 | 허용값                 | 지원하지 않는 미션입니다.      |
| 중복 완료 | 사용자·날짜·콘텐츠 1회 | 이미 완료한 미션입니다.        |
| 경험치    | 0 이상                 | 경험치 값이 올바르지 않습니다. |
| 레벨      | 1~999                  | 최고 레벨에 도달했습니다.      |

## 6. 예외 처리

| 예외 상황      | 처리 방식                  | 사용자 안내                    |
| -------------- | -------------------------- | ------------------------------ |
| 콘텐츠 없음    | 빈 상태 표시               | 오늘 표시할 콘텐츠가 없습니다. |
| 외부 링크 실패 | 재시도 또는 원문 보기 숨김 | 콘텐츠를 불러오지 못했습니다.  |
| 경험치 중복    | 지급 차단                  | 이미 보상을 받았습니다.        |

## 7. 데이터 객체

| 데이터 객체      | 주요 필드                                       | 처리 기준   |
| ---------------- | ----------------------------------------------- | ----------- |
| GrowthTask       | id, type, title, category, expReward, contentId | 미션 정의   |
| GrowthCompletion | id, userId, taskId, completedAt                 | 완료 이력   |
| UserLevel        | userId, level, exp                              | 레벨/경험치 |

## 8. 수락 기준

| 수락 기준   | 완료 판정               | 테스트 조건                   |
| ----------- | ----------------------- | ----------------------------- |
| 미션 완료   | 완료 이력 저장          | 독서 미션 완료 후 status 확인 |
| 경험치 지급 | User exp 증가           | 완료 전/후 exp 비교           |
| 레벨업      | 기준 도달 시 level 증가 | exp 95에서 10 지급            |

## 10. 보안·개인정보 기준

1. 본 기능에서 생성되는 사용자 개인 데이터는 로그인 사용자 본인만 조회·수정·삭제할 수 있다.
2. 급여, 지출, 저축, 소비, 프로필, 알림 수신 설정은 사용자 식별 데이터와 분리 가능한 구조로 관리한다.
3. 관리자 또는 운영자 접근이 필요한 경우 역할 기반 권한을 적용하고 모든 조작을 감사 로그로 저장한다.
4. 광고·제휴 분석에 민감 급여 데이터를 직접 사용하지 않는다.

## 11. 최종 완료 판정

본 문서는 레벨업 기능 명세서의 구현 범위, 입력값, 출력값, 검증, 예외, 데이터, 수락 기준을 모두 확정했다. 본 문서 기준으로 추가 기획 없이 구현·테스트·검수가 가능하므로 문서상·이론상 추가 작업은 필요하지 않다.
