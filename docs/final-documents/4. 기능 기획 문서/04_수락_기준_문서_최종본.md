# 수락 기준 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 기능별 개발 완료 판단 기준을 최종 확정한다. 개발 완료는 단순 화면 구현이 아니라 입력 검증, 계산 반영, 데이터 저장, 오류 처리, 화면 동기화, 보안 조건, QA 테스트 통과까지 포함한다.

## 공통 전제

| ----------- | ----------------------------------------------------------------------------- |

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |
| -------- | ------------------------------ | ------------------------------------ |
| P1       | 핵심 급여관리 기능             | MVP 출시 전 반드시 완료              |
| P2       | 유지율·커뮤니티·운영 핵심 기능 | 정식 출시 전 완료 또는 단계적 활성화 |
| P3       | 수익화·고도화 기능             | 안정화 이후 활성화 가능              |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 2. Definition of Done

| 구분        | 완료 기준                                                    |
| ----------- | ------------------------------------------------------------ |
| 기획 일치   | 기능 명세서와 화면 설계서의 요구사항을 모두 충족한다.        |
| UI 구현     | 지정된 화면, 컴포넌트, 문구, 상태값이 구현된다.              |
| 데이터 저장 | 생성/수정/삭제 데이터가 DB에 정상 반영된다.                  |
| 계산 정확성 | 납치금액, 남은금액, 목표 달성률 계산이 테스트를 통과한다.    |
| 예외 처리   | 필수값 누락, 네트워크 오류, 권한 오류, 중복 요청을 처리한다. |
| 보안        | 인증이 필요한 기능은 비로그인 접근을 차단한다.               |
| QA          | 정상/예외 테스트 케이스가 통과한다.                          |
| 회귀        | 기존 핵심 기능이 깨지지 않는다.                              |

## 3. 기능별 수락 기준 요약

| 기능 ID | 기능        | 완료 조건                                              | 예외 검증                                      |
| ------- | ----------- | ------------------------------------------------------ | ---------------------------------------------- |
| FUN-001 | 로그인      | 정상 계정으로 로그인 시 홈 또는 초기 설정으로 이동한다 | 비밀번호 오류, 정지 계정, 토큰 만료를 표시한다 |
| FUN-002 | 급여 정보   | 급여일/수령금액 저장 후 납치금액이 계산된다            | 금액 오류와 필수값 누락을 차단한다             |
| FUN-003 | 고정지출    | 고정지출 등록 후 월 지출합계에 반영된다                | 중복/날짜/금액 오류를 검증한다                 |
| FUN-004 | 고정저축    | 고정저축 등록 후 계획 화면에 표시된다                  | 0원 이하 입력을 차단한다                       |
| FUN-005 | 일일 예산   | 지출 추가 시 남은금액이 즉시 변경된다                  | 초과 시 음수와 경고를 표시한다                 |
| FUN-006 | 변동지출    | 등록/수정/삭제 후 집계가 재계산된다                    | 중복 요청을 방지한다                           |
| FUN-007 | 납치금액    | 모든 화면에 동일한 계산값이 표시된다                   | 0원 미만은 0원 표시한다                        |
| FUN-008 | 목표 달성률 | 목표금액 변경 시 달성률이 즉시 변경된다                | 목표 0원은 설정 유도                           |
| FUN-009 | 알림        | 알림 클릭 시 관련 화면으로 이동한다                    | 권한 거부 시 앱 내 알림 유지                   |
| FUN-010 | 레벨업      | 미션 완료 시 경험치와 레벨이 반영된다                  | 중복 지급을 차단한다                           |
| FUN-011 | 커뮤니티    | 게시글 등록 후 목록/상세에 표시된다                    | 금칙어/신고/첨부 실패 처리                     |
| FUN-012 | 마이페이지  | 성과/활동/설정 정보가 정확히 표시된다                  | 권한 없는 접근 차단                            |
| FUN-013 | 광고        | 노출/클릭 로그가 저장된다                              | 캠페인 없음 시 미노출                          |

## 4. 상세 수락 기준

### 4.1 인증/계정

| 기준             | Given                      | When                         | Then                                                    |
| ---------------- | -------------------------- | ---------------------------- | ------------------------------------------------------- |
| 일반 로그인 성공 | 유효 계정이 있을 때        | 아이디/비밀번호로 로그인하면 | access token이 발급되고 홈 또는 초기 설정으로 이동한다. |
| 일반 로그인 실패 | 비밀번호가 틀렸을 때       | 로그인하면                   | “아이디 또는 비밀번호를 확인해주세요.”를 표시한다.      |
| 소셜 로그인 성공 | 유효한 소셜 토큰이 있을 때 | 소셜 버튼을 누르면           | 계정이 생성 또는 매핑되고 로그인된다.                   |
| 자동 로그인      | refresh token이 유효할 때  | 앱을 재실행하면              | 스플래시 후 홈으로 이동한다.                            |
| 탈퇴 계정        | 탈퇴 상태일 때             | 로그인하면                   | 접근을 차단하고 안내 문구를 표시한다.                   |

### 4.2 급여 계획

| 기준        | Given                           | When                | Then                                       |
| ----------- | ------------------------------- | ------------------- | ------------------------------------------ |
| 급여 등록   | 급여일과 수령금액이 입력됐을 때 | 저장을 누르면       | PayrollPlan이 생성된다.                    |
| 급여 수정   | 기존 계획이 있을 때             | 수령금액을 수정하면 | 홈과 계획의 수령금액이 변경된다.           |
| 급여일 계산 | 급여일이 25일일 때              | 홈을 열면           | 이번 급여일과 다음 급여일이 계산된다.      |
| 금액 오류   | 문자가 입력됐을 때              | 저장을 누르면       | 저장을 차단하고 숫자 입력 안내를 표시한다. |
| 지출 초과   | 수령금액보다 지출이 클 때       | 저장하면            | 납치금액은 0원, 초과 경고를 표시한다.      |

### 4.3 고정지출/고정저축

| 기준           | Given                          | When                 | Then                                 |
| -------------- | ------------------------------ | -------------------- | ------------------------------------ |
| 고정지출 등록  | 항목명/금액/지출일이 유효할 때 | 저장하면             | 월별 고정지출 합계에 반영된다.       |
| 고정지출 수정  | 기존 항목이 있을 때            | 금액을 수정하면      | 수정 월 이후 합계가 변경된다.        |
| 결제 예정 알림 | 지출일이 다가올 때             | 알림 조건이 충족되면 | 알림이 생성된다.                     |
| 고정저축 등록  | 저축명/금액/이체일이 유효할 때 | 저장하면             | 고정저축 계획에 표시된다.            |
| 삭제           | 항목 삭제를 확인했을 때        | 삭제하면             | 목록에서 제외되고 집계가 재계산된다. |

### 4.4 일일 예산/변동지출

| 기준           | Given                                  | When                    | Then                                     |
| -------------- | -------------------------------------- | ----------------------- | ---------------------------------------- |
| 일일 예산 설정 | 일일 금액이 입력됐을 때                | 저장하면                | DailyBudget이 생성된다.                  |
| 지출 추가      | 오늘 예산이 있을 때                    | 5,000원 지출을 추가하면 | 오늘 사용금액이 5,000원 증가한다.        |
| 남은금액 계산  | 일일 예산 20,000원, 사용 13,000원일 때 | 홈을 보면               | 남은금액 7,000원이 표시된다.             |
| 예산 초과      | 사용금액이 예산보다 클 때              | 지출을 추가하면         | 초과 상태와 알림이 표시된다.             |
| 지출 삭제      | 지출 항목이 있을 때                    | 삭제하면                | 사용금액이 감소하고 남은금액이 증가한다. |

### 4.5 납치금액/목표 달성률

| 기준          | Given                          | When              | Then                            |
| ------------- | ------------------------------ | ----------------- | ------------------------------- |
| 예상 납치금액 | 수령금액과 예정 지출이 있을 때 | 계획 저장 시      | 예상 납치금액이 계산된다.       |
| 실제 납치금액 | 실제 지출이 갱신됐을 때        | 홈 재진입 시      | 최신 납치금액이 표시된다.       |
| 누적 납치금액 | 월 마감이 완료됐을 때          | 마이페이지를 열면 | 누적 납치금액이 표시된다.       |
| 목표 달성률   | 목표금액과 누적금액이 있을 때  | 계획 화면을 열면  | 퍼센트와 진행바가 일치한다.     |
| 100% 초과     | 누적금액이 목표보다 클 때      | 달성률 계산 시    | 100% 초과 달성 상태를 처리한다. |

### 4.6 알림

| 목표 달성 | 달성률이 기준에 도달했을 때 | 계산 이벤트가 발생하면 | 목표 달성 알림이 생성된다. |

### 4.7 레벨업

| 기준          | Given                         | When                 | Then                                        |
| ------------- | ----------------------------- | -------------------- | ------------------------------------------- |
| 미션 완료     | 미완료 미션이 있을 때         | 완료 버튼을 누르면   | 완료 이력과 경험치가 저장된다.              |
| 레벨업        | 경험치가 기준 이상일 때       | 미션 완료 시         | 레벨이 1 증가하고 경험치 잔여분이 반영된다. |
| 중복 완료     | 이미 완료한 미션일 때         | 다시 완료를 시도하면 | 경험치 중복 지급을 차단한다.                |
| 카테고리 전환 | 독서 탭에서 카테고리를 누르면 | 탭 전환 시           | 해당 카테고리 콘텐츠가 표시된다.            |

### 4.8 커뮤니티/글쓰기

| 기준        | Given                        | When                  | Then                                           |
| ----------- | ---------------------------- | --------------------- | ---------------------------------------------- |
| 게시글 목록 | 게시글이 있을 때             | 커뮤니티 화면 진입 시 | 인기글과 게시판 목록이 표시된다.               |
| 글쓰기 성공 | 제목/본문/게시판이 유효할 때 | 완료를 누르면         | 게시글이 등록된다.                             |
| 익명 글     | 익명 옵션이 켜져 있을 때     | 게시글 등록 시        | 일반 사용자에게 작성자 정보가 노출되지 않는다. |
| 질문 글     | 질문 옵션이 켜져 있을 때     | 게시글 등록 시        | 질문 표시가 적용된다.                          |
| 신고        | 신고 사유를 선택했을 때      | 신고하면              | 신고 이력이 저장되고 중복 신고가 차단된다.     |

### 4.9 마이페이지/광고/운영

| 기준        | Given                        | When                          | Then                                           |
| ----------- | ---------------------------- | ----------------------------- | ---------------------------------------------- |
| 성과 표시   | 사용자 집계 데이터가 있을 때 | 마이페이지 진입 시            | 누적 납치금액, 레벨, 자기관리 성과가 표시된다. |
| 내 게시글   | 작성글이 있을 때             | 내 게시글 관리 선택 시        | 작성글 목록이 표시된다.                        |
| 광고 노출   | 활성 캠페인이 있을 때        | 광고 영역이 있는 화면 진입 시 | 배너가 표시되고 노출 로그가 저장된다.          |
| 광고 클릭   | 배너를 클릭했을 때           | 랜딩 이동 전                  | 클릭 로그가 저장된다.                          |
| 운영자 처리 | 운영자 권한이 있을 때        | 신고 처리 시                  | 게시글 상태와 감사 로그가 저장된다.            |
