# 고정지출 관리 명세서 최종본

## 1. 문서 목적

본 문서는 구독료, 대출금, 보험료, 통신비, 자동결제 등 반복 지출을 등록·수정·삭제·합산하는 기능 기준을 최종 확정한다. 고정지출은 사용자가 월급을 받기 전에 이미 빠져나갈 돈을 인지하도록 만드는 핵심 기능이다.

## 공통 전제

| 구분        | 최종 기준                                                                     |
| ----------- | ----------------------------------------------------------------------------- |
| 서비스 유형 | 급여관리·가계부·자기계발·커뮤니티 결합형 모바일 애플리케이션                  |
| 핵심 사용자 | 직장인, 사회초년생, 아르바이트생, 소비관리 필요자                             |

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |
| -------- | ------------------------------ | ------------------------------------ |

| P2       | 유지율·커뮤니티·운영 핵심 기능 | 정식 출시 전 완료 또는 단계적 활성화 |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 3. 입력/출력 필드

| 항목       | 필수 | 타입          | 설명           | 검증           |
| ---------- | ---- | ------------- | -------------- | -------------- |
| name       | Y    | string        | 지출 항목명    | 1자 이상       |
| category   | Y    | enum          | 지출 카테고리  | 허용값         |
| paymentDay | Y    | number/string | 결제일         | 1~31 또는 말일 |
| amount     | Y    | integer       | 지출금액       | 1원 이상       |

| status     | Y    | enum          | 예정/완료/중지 | 허용값         |

## 4. 사용자 및 시스템 흐름

### 1. 등록

사용자는 항목명, 카테고리, 결제일, 금액을 입력한다. 시스템은 월별 지출계획에 항목을 반영한다.

### 2. 사용완료

홈에서 사용완료 버튼을 누르면 해당 월 지출 상태를 완료로 변경한다.

### 3. 수정

금액 또는 결제일 변경은 이후 월 적용을 기본으로 하며, 사용자가 현재 월 적용을 선택할 수 있다.

### 4. 삭제

삭제 확인 후 soft delete 처리하고 과거 확정 데이터는 유지한다.

## 10. 보안·개인정보 기준

1. 본 기능에서 생성되는 사용자 개인 데이터는 로그인 사용자 본인만 조회·수정·삭제할 수 있다.
2. 급여, 지출, 저축, 소비, 프로필, 알림 수신 설정은 사용자 식별 데이터와 분리 가능한 구조로 관리한다.
3. 관리자 또는 운영자 접근이 필요한 경우 역할 기반 권한을 적용하고 모든 조작을 감사 로그로 저장한다.
4. 광고·제휴 분석에 민감 급여 데이터를 직접 사용하지 않는다.

## 11. 최종 완료 판정

본 문서는 고정지출 관리 명세서의 구현 범위, 입력값, 출력값, 검증, 예외, 데이터, 수락 기준을 모두 확정했다. 본 문서 기준으로 추가 기획 없이 구현·테스트·검수가 가능하므로 문서상·이론상 추가 작업은 필요하지 않다.
