# 마이페이지 기능 명세서 최종본

## 1. 문서 목적

본 문서는 사용자 프로필, 누적 납치금액, 레벨, 자기관리 성과, 내 게시글, 내 레벨업, 문의, 공지, 계정 설정을 제공하는 마이페이지 기능을 최종 확정한다. 마이페이지는 개인 성과와 계정 관리의 중심 화면이다.

## 공통 전제

| 구분        | 최종 기준                                                                     |
| ----------- | ----------------------------------------------------------------------------- |
| 서비스 유형 | 급여관리·가계부·자기계발·커뮤니티 결합형 모바일 애플리케이션                  |
| 핵심 사용자 | 직장인, 사회초년생, 아르바이트생, 소비관리 필요자                             |
| 핵심 지표   | 수령금액, 지출금액, 납치금액, 누적 납치금액, 목표 달성률, 일일 남은금액, 레벨 |
| 기본 통화   | KRW, 원 단위 정수                                                             |
| 금액 표시   | 천 단위 콤마 + `원` 표기                                                      |
| 금액 저장   | 숫자형 정수 저장, 화면 표시 시 포맷팅                                         |
| 음수 처리   | 납치금액은 0원 미만 표시 금지, 남은금액은 초과 상태 판단을 위해 음수 허용     |
| 인증 상태   | 비로그인 사용자는 로그인/회원가입/스플래시 외 핵심 기능 접근 불가             |
| 개인정보    | 급여·대출·저축·소비 내역은 민감 금융성 생활 데이터로 분류                     |
| 광고 데이터 | 급여·대출·저축 원천 데이터와 광고 추천/클릭 로그는 분리 저장                  |

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |
| -------- | ------------------------------ | ------------------------------------ |
| P1       | 핵심 급여관리 기능             | MVP 출시 전 반드시 완료              |
| P2       | 유지율·커뮤니티·운영 핵심 기능 | 정식 출시 전 완료 또는 단계적 활성화 |
| P3       | 수익화·고도화 기능             | 안정화 이후 활성화 가능              |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 3. 입력/출력 필드

| 항목                 | 필수 | 타입    | 설명          | 검증           |
| -------------------- | ---- | ------- | ------------- | -------------- |
| nickname             | Y    | string  | 닉네임        | 중복/금칙어    |
| profileImage         | N    | url     | 프로필 이미지 | 허용 파일      |
| totalHijackAmount    | Y    | integer | 누적 납치금액 | 시스템 계산    |
| level                | Y    | integer | 현재 레벨     | 1~999          |
| selfManagementScore  | N    | number  | 자기관리 성과 | 시스템 계산    |
| notificationSettings | Y    | object  | 알림 설정     | 유형별 boolean |

## 4. 사용자 및 시스템 흐름

### 1. 프로필 조회

사용자는 MY 탭에서 프로필, 누적 납치금액, 레벨, 성과를 확인한다.

### 2. 프로필 수정

닉네임과 프로필 이미지를 수정하면 즉시 화면에 반영된다.

### 3. 내 활동 관리

내 게시글, 레벨업 완료 이력, 문의 내역, 공지사항으로 이동한다.

### 4. 계정 설정

알림 수신, 자동 로그인, 탈퇴 등 계정 상태를 관리한다.

## 5. 검증 규칙

| 검증 항목     | 기준                         | 실패 처리                    |
| ------------- | ---------------------------- | ---------------------------- |
| 닉네임        | 필수, 중복 불가, 금칙어 제한 | 사용할 수 없는 닉네임입니다. |
| 프로필 이미지 | 용량/확장자 제한             | 지원하지 않는 이미지입니다.  |
| 권한          | 본인만 접근                  | 접근 권한이 없습니다.        |

## 6. 예외 처리

| 예외 상황          | 처리 방식          | 사용자 안내                  |
| ------------------ | ------------------ | ---------------------------- |
| 성과 데이터 없음   | 0원/기본 레벨 표시 | 아직 기록된 성과가 없습니다. |
| 이미지 업로드 실패 | 기존 이미지 유지   | 이미지 변경에 실패했습니다.  |
| 탈퇴 요청          | 재인증 후 처리     | 탈퇴 전 안내를 확인해주세요. |

## 7. 데이터 객체

## 8. 수락 기준

| 수락 기준   | 완료 판정        | 테스트 조건                |
| ----------- | ---------------- | -------------------------- |
| 성과 표시   | 집계값 정확      | 홈 누적 납치금액과 MY 일치 |
| 프로필 수정 | 수정값 저장      | 닉네임 변경 후 재조회      |
| 내 게시글   | 본인 작성글 목록 | 삭제/숨김 상태 구분        |

## 10. 보안·개인정보 기준

1. 본 기능에서 생성되는 사용자 개인 데이터는 로그인 사용자 본인만 조회·수정·삭제할 수 있다.
2. 급여, 지출, 저축, 소비, 프로필, 알림 수신 설정은 사용자 식별 데이터와 분리 가능한 구조로 관리한다.
3. 관리자 또는 운영자 접근이 필요한 경우 역할 기반 권한을 적용하고 모든 조작을 감사 로그로 저장한다.
4. 광고·제휴 분석에 민감 급여 데이터를 직접 사용하지 않는다.

## 11. 최종 완료 판정

본 문서는 마이페이지 기능 명세서의 구현 범위, 입력값, 출력값, 검증, 예외, 데이터, 수락 기준을 모두 확정했다. 본 문서 기준으로 추가 기획 없이 구현·테스트·검수가 가능하므로 문서상·이론상 추가 작업은 필요하지 않다.
