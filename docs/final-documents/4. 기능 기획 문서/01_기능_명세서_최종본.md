# 기능 명세서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 모든 사용자 기능, 운영 기능, 수익화 기능의 구현 기준을 최종 확정한다. 각 기능은 기능 ID, 설명, 입력값, 출력값, 우선순위, 화면 연결, 데이터 객체, 예외 처리, 수락 기준을 포함한다. 개발자는 본 문서를 기준으로 백로그, API, 화면, QA 테스트를 생성한다.

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 4. 상세 기능 명세

### 4.1 FUN-001 회원 로그인

| 항목      | 최종 명세                                                                      |
| --------- | ------------------------------------------------------------------------------ |
| 목적      | 사용자 인증 후 개인 급여관리 데이터에 접근하게 한다.                           |
| 화면      | SCR-002 로그인, SCR-003 회원가입                                               |
| 입력값    | 아이디, 비밀번호, 소셜 로그인 토큰, 자동 로그인 여부                           |
| 출력값    | access token, refresh token, userId, onboardingCompleted, nextScreen           |
| 정상 흐름 | 입력 → 인증 요청 → 토큰 발급 → 급여 설정 여부 확인 → 홈 또는 초기 설정 이동    |
| 예외      | 비밀번호 불일치, 탈퇴 계정, 정지 계정, 소셜 토큰 만료, 네트워크 실패           |
| 보안      | 비밀번호 평문 저장 금지, 토큰 보안 저장소 사용, refresh token 재발급 정책 적용 |
| 수락 기준 | 인증 성공 시 로그인 상태가 유지되고, 실패 시 명확한 오류가 표시된다.           |

### 4.2 FUN-002 급여 정보 등록

| 항목      | 최종 명세                                                           |
| --------- | ------------------------------------------------------------------- |
| 목적      | 월급관리의 기준 데이터를 생성한다.                                  |
| 화면      | SCR-004 초기 급여 설정, SCR-008 계획/설정                           |
| 입력값    | 급여일, 수령 예정 급여, 지출 예정 금액, 목표 납치금액, 적용 월      |
| 출력값    | PayrollPlan, expectedHijackAmount, monthlyTargetRate                |
| 정상 흐름 | 급여 정보 입력 → 금액 검증 → 납치금액 계산 → 저장 → 홈 반영         |
| 예외      | 급여일 미입력, 0원 이하 급여, 급여보다 지출이 큰 경우, 중복 월 계획 |
| 수락 기준 | 저장 즉시 급여 홈과 계획 화면의 수령금액·납치금액이 일치한다.       |

### 4.3 FUN-003 고정지출 관리

| 항목      | 최종 명세                                                            |
| --------- | -------------------------------------------------------------------- |
| 목적      | 매월 반복되는 지출을 등록하고 지출 예정 금액에 자동 반영한다.        |
| 입력값    | 지출명, 카테고리, 지출일, 금액, 반복주기, 결제수단, 메모             |
| 출력값    | FixedExpense, monthlyFixedExpenseTotal, upcomingPaymentAlert         |
| 정상 흐름 | 항목 생성 → 반복 규칙 저장 → 월별 합산 → 결제 예정 알림 생성         |
| 예외      | 금액 오류, 날짜 오류, 중복 항목, 삭제된 항목의 과거 데이터 보존 여부 |
| 수락 기준 | 등록된 고정지출은 해당 월 지출금액에 합산되고 급여 홈에 표시된다.    |

### 4.4 FUN-004 고정저축 관리

| 항목      | 최종 명세                                                                                                    |
| --------- | ------------------------------------------------------------------------------------------------------------ |
| 목적      | 청약, 적금, 투자 등 고정 저축 계획을 관리한다.                                                               |
| 입력값    | 저축명, 카테고리, 이체일, 금액, 반복주기, 계좌/기관 메모                                                     |
| 출력값    | SavingsPlan, monthlySavingsTotal, hijackProtectedAmount                                                      |
| 정책      | 고정저축은 사용자가 지켜낸 금액으로 분류할 수 있으나 총 현금흐름 계산에서는 지출성 차감 항목으로도 집계한다. |
| 수락 기준 | 고정저축 합계가 계획 화면에 표시되고 목표 달성률 계산 기준에 반영된다.                                       |

### 4.5 FUN-005 일일 생활비 관리

| 항목      | 최종 명세                                                  |
| --------- | ---------------------------------------------------------- |
| 목적      | 하루 단위로 소비 가능 금액을 통제한다.                     |
| 입력값    | 일일 설정금액, 적용 기간, 오늘 사용금액                    |
| 출력값    | DailyBudget, usedAmount, remainingAmount, overBudgetStatus |
| 계산      | 오늘 남은금액 = 일일 설정금액 - 오늘 사용금액              |
| 예외      | 예산 초과, 날짜 변경, 타임존 차이, 과거 날짜 지출 추가     |
| 수락 기준 | 변동지출 추가 즉시 사용금액과 남은금액이 재계산된다.       |

### 4.6 FUN-006 변동지출 기록

| 항목      | 최종 명세                                                    |
| --------- | ------------------------------------------------------------ |
| 목적      | 매일 발생하는 소비를 직접 기록한다.                          |
| 입력값    | 항목명, 카테고리, 금액, 날짜, 메모, 결제수단                 |
| 출력값    | VariableExpense, dailyUsedAmount, monthlyVariableTotal       |
| 정상 흐름 | 지출 입력 → 금액 검증 → 저장 → 일일 예산/납치금액 재계산     |
| 예외      | 빈 항목명, 0원 이하 금액, 미래 날짜, 중복 요청, 삭제 후 복원 |
| 수락 기준 | 등록/수정/삭제가 홈, 계획, 마이페이지 집계에 일관 반영된다.  |

### 4.7 FUN-007 납치금액 계산

| 항목      | 최종 명세                                                                  |
| --------- | -------------------------------------------------------------------------- |
| 목적      | 사용자가 지켜낸 금액을 핵심 성과 지표로 제공한다.                          |
| 입력값    | 수령금액, 고정지출, 고정저축, 변동지출, 생활비 사용액                      |
| 출력값    | expectedHijackAmount, actualHijackAmount, totalHijackAmount                |
| 원칙      | 납치금액은 0원 미만으로 표시하지 않는다. 초과 지출은 별도 경고로 표시한다. |
| 수락 기준 | 동일 월·동일 사용자 기준 모든 화면의 납치금액이 동일하다.                  |

### 4.8 FUN-008 목표 달성률 계산

| 항목      | 최종 명세                                       |
| --------- | ----------------------------------------------- |
| 목적      | 목표 대비 누적 납치 성과를 시각화한다.          |
| 입력값    | 목표금액, 누적 납치금액, 월별 확정 납치금액     |
| 출력값    | achievementRate, progressBar, achievementStatus |
| 계산      | 목표 달성률 = 누적 납치금액 ÷ 목표금액 × 100    |
| 예외      | 목표금액 0원, 목표 미설정, 100% 초과 달성       |
| 수락 기준 | 목표금액이 변경되면 달성률이 즉시 재계산된다.   |

### 4.9 FUN-009 알림 발송

| 항목      | 최종 명세                                                       |
