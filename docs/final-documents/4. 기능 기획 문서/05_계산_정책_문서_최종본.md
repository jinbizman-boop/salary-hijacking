# 계산 정책 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼에서 사용되는 모든 금액·성과·레벨 계산식을 최종 확정한다. 급여납치의 핵심 가치는 “이번 월급에서 얼마를 지켜냈는가”를 정확히 보여주는 데 있으므로, 본 문서는 프론트엔드, 백엔드, DB 집계, QA 테스트의 단일 계산 기준이다.

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 3. 핵심 계산식

| 계산명           | 산식                                              | 표시 기준 | 예외 처리                |
| ---------------- | ------------------------------------------------- | --------- | ------------------------ |
| 총 예정 지출금액 | 예정 고정지출 + 예정 생활비 총액 + 기타 예정 지출 | 계획 화면 | 항목 없으면 0원          |
| 예상 납치금액    | 수령 예정 급여 - 총 예정 지출금액                 | 홈/계획   | 0원 미만이면 0원 표시    |
| 오늘 사용금액    | 오늘 날짜의 변동지출 합계                         | 홈        | 삭제 항목 제외           |
| 오늘 남은금액    | 일일 설정금액 - 오늘 사용금액                     | 홈        | 음수 허용, 초과 표시     |
| 월 변동지출 합계 | 해당 월 변동지출 합계                             | 계획/MY   | 삭제 항목 제외           |
| 실제 지출금액    | 고정지출 확정액 + 월 변동지출 합계                | 홈/계획   | 동일 월 기준             |
| 실제 납치금액    | 실제 수령금액 - 실제 지출금액                     | 월 마감   | 0원 미만이면 0원 확정    |
| 누적 납치금액    | 이전 누적 납치금액 + 월별 확정 납치금액           | 홈/MY     | 확정 월만 누적           |
| 목표 달성률      | 누적 납치금액 ÷ 목표금액 × 100                    | 계획      | 목표금액 0원은 계산 불가 |
| 레벨 경험치율    | 현재 레벨 내 경험치 ÷ 다음 레벨 필요 경험치 × 100 | LV/MY     | 100% 도달 시 레벨업      |

## 7. 일일 예산 계산 정책

### 7.1 기본 산식

```text
오늘 남은금액 = 일일 설정금액 - 오늘 사용금액
오늘 사용금액 = 오늘 날짜의 변동지출 합계
```

### 7.2 예산 초과 처리

| 조건              | 처리                              |
| ----------------- | --------------------------------- |
| 오늘 남은금액 > 0 | 정상, 초록색 표시                 |
| 오늘 남은금액 = 0 | 예산 소진, 중립 안내              |
| 오늘 남은금액 < 0 | 초과 상태, 빨간색 표시, 알림 생성 |

### 7.3 날짜 변경

1. 하루 기준은 Asia/Seoul 00:00~23:59:59다.
2. 자정 이후 오늘 사용금액은 새 날짜 기준으로 0원에서 시작한다.
3. 과거 날짜에 지출을 추가하면 해당 날짜의 DailyBudget 집계만 변경한다.

## 8. 변동지출 계산 정책

| 항목      | 기준                                                  |
| --------- | ----------------------------------------------------- |
| 합산 대상 | 삭제되지 않은 지출, 금액 1원 이상, 사용자 본인 데이터 |
| 제외 대상 | 삭제 상태, 저장 실패, 임시 입력, 타 사용자 데이터     |
| 수정      | 이전 금액과 새 금액 차액만큼 집계 변경                |
| 삭제      | soft delete 후 집계 제외                              |
| 카테고리  | 통계용이며 합계에는 모든 카테고리 포함                |

## 9. 납치금액 계산 정책

### 9.1 예상 납치금액

```text
예상 납치금액 = max(0, 수령 예정 급여 - 예정 고정지출 - 예정 고정저축 - 예정 생활비 총액 - 기타 예정 지출)
```

### 9.2 현재 납치금액

```text
현재 납치금액 = max(0, 현재 수령금액 - 현재까지 확정/사용 지출금액)
```

### 9.3 월별 확정 납치금액

```text
월별 확정 납치금액 = max(0, 실제 수령금액 - 실제 고정지출 - 실제 변동지출)
```

### 9.4 누적 납치금액

```text
누적 납치금액 = 사용자 기존 누적 납치금액 + 확정된 월별 납치금액의 합
```

### 9.5 표시 정책

| 상황          | 표시                                |
| ------------- | ----------------------------------- |
| 납치금액 양수 | `1,234,000원`                       |
| 납치금액 0    | `0원` + 지출 초과 안내 가능         |
| 계산 불가     | `급여 계획을 먼저 입력해주세요.`    |
| 확정 전       | 예상 또는 현재 기준임을 라벨로 표시 |

## 10. 목표 달성률 계산 정책

```text
목표 달성률(%) = floor((누적 납치금액 / 목표금액) × 100)
```

| 조건                   | 처리                                      |
| ---------------------- | ----------------------------------------- |
| 목표금액 > 0           | 정상 계산                                 |
| 목표금액 = 0 또는 없음 | 계산하지 않고 목표 설정 유도              |
| 달성률 < 0             | 0% 표시                                   |
| 달성률 0~100           | 진행바에 그대로 반영                      |
| 달성률 > 100           | 수치는 초과 표시 가능, 진행바는 100% 고정 |

## 11. 레벨/경험치 계산 정책

| 항목        | 기준                                            |
| ----------- | ----------------------------------------------- |
| 기본 경험치 | 미션 유형별 고정 경험치 지급                    |
| 중복 지급   | 같은 사용자·같은 미션·같은 날짜 1회만 허용      |
| 레벨업      | 현재 경험치가 다음 레벨 요구치를 넘으면 레벨 +1 |
| 잔여 경험치 | 초과분은 다음 레벨 경험치로 이월                |
| 최고 레벨   | 문서상 999Lv까지 허용                           |

```text
미션 완료 후 경험치 = 기존 경험치 + 미션 보상 경험치
레벨업 조건 = 미션 완료 후 경험치 >= 다음 레벨 필요 경험치
```

## 12. 알림 계산/발송 조건

| 알림          | 조건                     | 중복 방지                     |
| ------------- | ------------------------ | ----------------------------- |
| 예산 초과     | 오늘 남은금액 < 0        | 사용자·날짜·알림유형 1회 기본 |
| 결제 예정     | 고정지출일 D-1 또는 당일 | 항목별 1회                    |
| 목표 달성     | 달성률 100% 최초 도달    | 목표별 1회                    |
| 레벨업        | 레벨 증가 발생           | 레벨별 1회                    |
| 미션 리마인드 | 지정 시간 미완료         | 하루 유형별 1회               |

## 14. 최종 완료 판정

본 계산 정책 문서는 급여납치 플랫폼의 금액, 예산, 납치금액, 누적 성과, 목표 달성률, 레벨 경험치 계산 기준을 모두 확정했다. 본 문서 기준으로 계산 로직 구현과 자동 테스트 작성이 가능하므로 추가 계산 정책 작업은 필요하지 않다.
