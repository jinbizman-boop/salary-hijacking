# 글쓰기 기능 명세서 최종본

## 1. 문서 목적

본 문서는 커뮤니티 게시글 작성 화면에서 제목, 본문, 첨부, 게시판 선택, 익명 여부, 질문 여부를 입력하고 게시글을 등록하는 기능을 최종 확정한다. 글쓰기는 커뮤니티 생성 콘텐츠의 핵심 진입 기능이다.

## 공통 전제

| 구분        | 최종 기준                                                                     |
| ----------- | ----------------------------------------------------------------------------- |
| 서비스 유형 | 급여관리·가계부·자기계발·커뮤니티 결합형 모바일 애플리케이션                  |
| 핵심 사용자 | 직장인, 사회초년생, 아르바이트생, 소비관리 필요자                             |
| 핵심 지표   | 수령금액, 지출금액, 납치금액, 누적 납치금액, 목표 달성률, 일일 남은금액, 레벨 |
| 기본 통화   | KRW, 원 단위 정수                                                             |
| 금액 표시   | 천 단위 콤마 + `원` 표기                                                      |
| 금액 저장   | 숫자형 정수 저장, 화면 표시 시 포맷팅                                         |
| 음수 처리   | 납치금액은 0원 미만 표시 금지, 남은금액은 초과 상태 판단을 위해 음수 허용     |
| 인증 상태   | 비로그인 사용자는 로그인/회원가입/스플래시 외 핵심 기능 접근 불가             |
| 개인정보    | 급여·대출·저축·소비 내역은 민감 금융성 생활 데이터로 분류                     |
| 광고 데이터 | 급여·대출·저축 원천 데이터와 광고 추천/클릭 로그는 분리 저장                  |

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |
| -------- | ------------------------------ | ------------------------------------ |
| P1       | 핵심 급여관리 기능             | MVP 출시 전 반드시 완료              |
| P2       | 유지율·커뮤니티·운영 핵심 기능 | 정식 출시 전 완료 또는 단계적 활성화 |
| P3       | 수익화·고도화 기능             | 안정화 이후 활성화 가능              |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 3. 입력/출력 필드

| 항목        | 필수 | 타입    | 설명        | 검증       |
| ----------- | ---- | ------- | ----------- | ---------- |
| boardType   | Y    | enum    | 게시판 선택 | 허용값     |
| title       | Y    | string  | 제목        | 1~100자    |
| body        | Y    | string  | 본문        | 1자 이상   |
| attachments | N    | array   | 첨부 파일   | 허용 파일  |
| isAnonymous | Y    | boolean | 익명 여부   | 기본 false |
| isQuestion  | Y    | boolean | 질문 여부   | 기본 false |

## 4. 사용자 및 시스템 흐름

### 1. 작성 시작

커뮤니티 화면의 글쓰기 버튼을 누르면 글쓰기 화면이 열린다.

### 2. 입력

사용자는 게시판, 제목, 본문, 첨부, 질문/익명 옵션을 설정한다.

### 3. 검증

완료 버튼 선택 시 필수값, 금칙어, 첨부 상태를 검증한다.

### 4. 등록

정상 등록 후 글쓰기 화면을 닫고 커뮤니티 목록에 새 게시글을 반영한다.

## 5. 검증 규칙

| 검증 항목 | 기준             | 실패 처리                  |
| --------- | ---------------- | -------------------------- |
| 제목      | 필수, 최대 길이  | 제목을 입력해주세요.       |
| 본문      | 필수             | 본문을 입력해주세요.       |
| 게시판    | 필수             | 게시판을 선택해주세요.     |
| 첨부      | 허용 확장자/용량 | 첨부할 수 없는 파일입니다. |

## 6. 예외 처리

| 예외 상황        | 처리 방식                          | 사용자 안내                                     |
| ---------------- | ---------------------------------- | ----------------------------------------------- |
| 첨부 업로드 실패 | 등록 차단 또는 첨부 제거 후 재시도 | 첨부 업로드에 실패했습니다.                     |
| 중복 등록        | 버튼 비활성화/멱등 처리            | 등록 중입니다.                                  |
| 금칙어           | 등록 차단                          | 운영 정책에 맞지 않는 표현이 포함되어 있습니다. |

## 7. 데이터 객체

| ModerationLog | postId, result, reason                          | 금칙어/검수 결과 |

| 질문 적용 | 질문 배지 표시     | 목록/상세 배지 확인   |

## 10. 보안·개인정보 기준

1. 본 기능에서 생성되는 사용자 개인 데이터는 로그인 사용자 본인만 조회·수정·삭제할 수 있다.
2. 급여, 지출, 저축, 소비, 프로필, 알림 수신 설정은 사용자 식별 데이터와 분리 가능한 구조로 관리한다.
3. 관리자 또는 운영자 접근이 필요한 경우 역할 기반 권한을 적용하고 모든 조작을 감사 로그로 저장한다.
4. 광고·제휴 분석에 민감 급여 데이터를 직접 사용하지 않는다.

## 11. 최종 완료 판정

본 문서는 글쓰기 기능 명세서의 구현 범위, 입력값, 출력값, 검증, 예외, 데이터, 수락 기준을 모두 확정했다. 본 문서 기준으로 추가 기획 없이 구현·테스트·검수가 가능하므로 문서상·이론상 추가 작업은 필요하지 않다.
