# 유스케이스 명세서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 주요 기능별 정상 흐름, 대체 흐름, 예외 흐름, 사후 조건을 최종 확정한다. QA는 본 문서를 기준으로 시나리오 테스트를 작성하고, 개발자는 기능 구현 시 누락 흐름을 방지한다.

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 4. 유스케이스 완료 기준

| 기준        | 완료 판정                                                           |
| ----------- | ------------------------------------------------------------------- |
| 정상 흐름   | 모든 주요 유스케이스가 성공 시나리오로 동작해야 한다.               |
| 예외 흐름   | 사용자 오류, 네트워크 오류, 권한 오류, 데이터 오류에 대응해야 한다. |
| 화면 동기화 | 저장·수정·삭제 후 연결 화면의 수치와 목록이 일치해야 한다.          |
| 감사 가능성 | 운영자 조작과 민감 변경은 로그로 추적 가능해야 한다.                |
| QA 가능성   | 각 유스케이스는 테스트 케이스로 전환 가능해야 한다.                 |

## 5. 최종 완료 판정

본 유스케이스 명세서는 급여납치 플랫폼의 주요 정상/예외 흐름을 모두 확정했다. 본 문서 기준으로 기능 구현과 시나리오 QA가 가능하므로 추가 유스케이스 정의 작업은 필요하지 않다.
