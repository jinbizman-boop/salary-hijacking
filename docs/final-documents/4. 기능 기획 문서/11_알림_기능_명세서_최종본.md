> 본 문서는 급여납치 플랫폼의 기능 구현, 데이터 처리, 예외 처리, QA 검수, 배포 승인에 적용되는 최종 기준이다. 본 문서에 정의된 기능·정책·수락 기준은 별도 변경 승인 전까지 최종 구현 기준으로 적용한다.

# 알림 기능 명세서 최종본

## 1. 문서 목적

본 문서는 예산 초과, 결제 예정, 목표 달성, 레벨업 미션, 이벤트, 공지 알림의 생성·조회·읽음·푸시 발송 기준을 최종 확정한다. 알림은 급여관리 행동을 놓치지 않게 만드는 리텐션 기능이다.

## 공통 전제

| 구분        | 최종 기준                                                                     |
| ----------- | ----------------------------------------------------------------------------- |
| 서비스 유형 | 급여관리·가계부·자기계발·커뮤니티 결합형 모바일 애플리케이션                  |
| 핵심 사용자 | 직장인, 사회초년생, 아르바이트생, 소비관리 필요자                             |
| 핵심 지표   | 수령금액, 지출금액, 납치금액, 누적 납치금액, 목표 달성률, 일일 남은금액, 레벨 |
| 기본 통화   | KRW, 원 단위 정수                                                             |
| 금액 표시   | 천 단위 콤마 + `원` 표기                                                      |
| 금액 저장   | 숫자형 정수 저장, 화면 표시 시 포맷팅                                         |
| 음수 처리   | 납치금액은 0원 미만 표시 금지, 남은금액은 초과 상태 판단을 위해 음수 허용     |
| 인증 상태   | 비로그인 사용자는 로그인/회원가입/스플래시 외 핵심 기능 접근 불가             |
| 개인정보    | 급여·대출·저축·소비 내역은 민감 금융성 생활 데이터로 분류                     |
| 광고 데이터 | 급여·대출·저축 원천 데이터와 광고 추천/클릭 로그는 분리 저장                  |

## 공통 우선순위 정의

| 우선순위 | 의미                           | 출시 기준                            |
| -------- | ------------------------------ | ------------------------------------ |
| P1       | 핵심 급여관리 기능             | MVP 출시 전 반드시 완료              |
| P2       | 유지율·커뮤니티·운영 핵심 기능 | 정식 출시 전 완료 또는 단계적 활성화 |
| P3       | 수익화·고도화 기능             | 안정화 이후 활성화 가능              |

## 공통 오류 처리 원칙

1. 사용자가 수정 가능한 오류는 필드 하단 문구와 토스트로 안내한다.
2. 서버·네트워크 오류는 재시도 버튼을 제공한다.
3. 금액 계산 오류는 저장을 차단하고 원인 필드를 강조한다.
4. 중복 요청은 클라이언트 버튼 비활성화와 서버 멱등성 키로 방지한다.
5. 저장 성공 후 관련 화면의 수치와 목록은 즉시 재조회 또는 로컬 상태 갱신으로 동기화한다.

## 3. 입력/출력 필드

| 항목         | 필수 | 타입     | 설명              | 검증        |
| ------------ | ---- | -------- | ----------------- | ----------- |
| type         | Y    | enum     | 알림 유형         | 허용값      |
| title        | Y    | string   | 알림 제목         | 1자 이상    |
| body         | Y    | string   | 알림 내용         | 1자 이상    |
| targetScreen | N    | string   | 클릭 시 이동 화면 | 존재 화면   |
| readAt       | N    | datetime | 읽음 시간         | 시스템 처리 |
| scheduledAt  | N    | datetime | 발송 예정 시간    | 유효 시간   |

## 4. 사용자 및 시스템 흐름

### 1. 알림 생성

이벤트 조건이 발생하면 Notification 레코드를 생성한다.

### 2. 푸시 발송

사용자가 푸시 수신에 동의했고 방해금지 조건이 아니면 푸시를 발송한다.

### 3. 목록 조회

알림 화면은 최신순으로 알림을 보여준다.

### 4. 클릭 처리

알림을 누르면 읽음 처리 후 관련 화면으로 이동한다.

## 5. 검증 규칙

| 검증 항목 | 기준             | 실패 처리                      |
| --------- | ---------------- | ------------------------------ |
| 유형      | 허용값           | 알림 유형이 올바르지 않습니다. |
| 제목/내용 | 필수             | 알림 내용을 입력해주세요.      |
| 중복      | 이벤트별 중복키  | 중복 알림을 생성하지 않습니다. |
| 수신 설정 | 사용자 설정 우선 |                                |

## 6. 예외 처리

| 예외 상황      | 처리 방식              | 사용자 안내                     |
| -------------- | ---------------------- | ------------------------------- |
| 푸시 권한 거부 | 앱 내 알림만 생성      | 푸시 알림이 꺼져 있습니다.      |
| 연결 화면 삭제 | 기본 화면으로 이동     | 연결된 화면을 찾을 수 없습니다. |
| 야간 시간      | 발송 지연 또는 앱 내만 |                                 |

## 7. 데이터 객체

| Notification           | id, userId, type, title, body, targetScreen, readAt, createdAt | 앱 내 알림     |
| PushToken              | userId, deviceId, token, enabled                               | 푸시 발송 대상 |

## 8. 수락 기준

| 수락 기준 | 완료 판정                     | 테스트 조건           |
| --------- | ----------------------------- | --------------------- |
| 알림 목록 | 최신순 노출                   | 복수 알림 시간순 확인 |
| 읽음 처리 | 클릭 후 readAt 저장           | 읽음 상태 확인        |
| 예산 초과 | 초과 이벤트 발생 시 알림 생성 | 초과 지출 입력 테스트 |

## 10. 보안·개인정보 기준

1. 본 기능에서 생성되는 사용자 개인 데이터는 로그인 사용자 본인만 조회·수정·삭제할 수 있다.
2. 급여, 지출, 저축, 소비, 프로필, 알림 수신 설정은 사용자 식별 데이터와 분리 가능한 구조로 관리한다.
3. 관리자 또는 운영자 접근이 필요한 경우 역할 기반 권한을 적용하고 모든 조작을 감사 로그로 저장한다.
4. 광고·제휴 분석에 민감 급여 데이터를 직접 사용하지 않는다.

## 11. 최종 완료 판정

본 문서는 알림 기능 명세서의 구현 범위, 입력값, 출력값, 검증, 예외, 데이터, 수락 기준을 모두 확정했다. 본 문서 기준으로 추가 기획 없이 구현·테스트·검수가 가능하므로 문서상·이론상 추가 작업은 필요하지 않다.
