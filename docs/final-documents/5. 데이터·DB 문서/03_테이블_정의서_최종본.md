# 03. 테이블 정의서 최종본

> 문서 상태: 최종본  
> 적용 서비스: 급여납치(Salary Hijacking) 모바일 플랫폼  
> 적용 범위: 급여관리 · 가계부 · 자기계발 레벨업 · 커뮤니티 · 알림 · 광고/제휴 · 운영 데이터  
> 기준 DB: Neon DB / PostgreSQL  
> 기준일: 2026-06-15  
> 원칙: 본 문서는 문서상·이론상 추가 작업 없이 구현 기준으로 사용할 수 있도록 완결된 기준값, 필드, 관계, 정책, 예외를 포함한다.

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 Neon DB/PostgreSQL 구현을 위한 테이블, 컬럼, 타입, Nullable, 기본값, 제약조건, 설명을 정의한다.

## 2. 공통 컬럼 기준

| 컬럼명     | 타입        | Nullable | 기본값 | 설명                                 |
| ---------- | ----------- | -------: | ------ | ------------------------------------ |
| created_at | TIMESTAMPTZ |        N | now()  | 생성 시각                            |
| updated_at | TIMESTAMPTZ |        N | now()  | 수정 시각                            |
| deleted_at | TIMESTAMPTZ |        Y | null   | soft delete 시각                     |
| created_by | UUID        |        Y | null   | 생성자 user_id 또는 관리자 ID        |
| updated_by | UUID        |        Y | null   | 마지막 수정자 user_id 또는 관리자 ID |

## 3. PostgreSQL 기본 확장

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;
```

## 4. 테이블 정의

### 4.1 users

| 컬럼명        | 타입        | Null | 기본값            | 제약/설명                                             |
| ------------- | ----------- | ---: | ----------------- | ----------------------------------------------------- |
| user_id       | UUID        |    N | gen_random_uuid() | PK                                                    |
| email         | TEXT        |    Y | null              | 이메일 로그인 또는 소셜 이메일, 활성 사용자 중 unique |
| phone_number  | TEXT        |    Y | null              | 선택 입력, 국제번호 문자열 권장                       |
| nickname      | TEXT        |    N | -                 | 앱 내부 기본 닉네임                                   |
| status        | TEXT        |    N | 'ACTIVE'          | ACTIVE, SUSPENDED, WITHDRAWN, DELETED                 |
| last_login_at | TIMESTAMPTZ |    Y | null              | 마지막 로그인                                         |
| created_at    | TIMESTAMPTZ |    N | now()             | 생성일                                                |
| updated_at    | TIMESTAMPTZ |    N | now()             | 수정일                                                |
| deleted_at    | TIMESTAMPTZ |    Y | null              | 탈퇴/삭제일                                           |

```sql
CREATE TABLE users (
  user_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT,
  phone_number TEXT,
  nickname TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'ACTIVE' CHECK (status IN ('ACTIVE','SUSPENDED','WITHDRAWN','DELETED')),
  last_login_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at TIMESTAMPTZ
);
CREATE UNIQUE INDEX uq_users_email_active ON users(email) WHERE email IS NOT NULL AND deleted_at IS NULL;
```

### 4.2 auth_identities

| 컬럼명            | 타입        | Null | 기본값            | 제약/설명                                    |
| ----------------- | ----------- | ---: | ----------------- | -------------------------------------------- |
| identity_id       | UUID        |    N | gen_random_uuid() | PK                                           |
| user_id           | UUID        |    N | -                 | FK users                                     |
| provider          | TEXT        |    N | -                 | EMAIL, NAVER, KAKAO, GOOGLE, FACEBOOK, APPLE |
| provider_user_key | TEXT        |    N | -                 | provider별 고유 식별자                       |
| email             | TEXT        |    Y | null              | provider에서 제공한 이메일                   |
| linked_at         | TIMESTAMPTZ |    N | now()             | 연결일                                       |
| revoked_at        | TIMESTAMPTZ |    Y | null              | 연결 해제일                                  |

```sql
CREATE TABLE auth_identities (
  identity_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
  provider TEXT NOT NULL CHECK (provider IN ('EMAIL','NAVER','KAKAO','GOOGLE','FACEBOOK','APPLE')),
  provider_user_key TEXT NOT NULL,
  email TEXT,
  linked_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  revoked_at TIMESTAMPTZ,
  UNIQUE(provider, provider_user_key)
);
CREATE INDEX idx_auth_user ON auth_identities(user_id);
```

### 4.5 payroll_plans

| 컬럼명                  | 타입        | Null | 기본값            | 제약/설명                       |
| ----------------------- | ----------- | ---: | ----------------- | ------------------------------- |
| payroll_plan_id         | UUID        |    N | gen_random_uuid() | PK                              |
| user_id                 | UUID        |    N | -                 | FK users                        |
| year_month              | CHAR(7)     |    N | -                 | YYYY-MM                         |
| payday                  | SMALLINT    |    N | -                 | 1~31                            |
| expected_salary_amount  | BIGINT      |    N | 0                 | 수령 예정 급여                  |
| expected_expense_amount | BIGINT      |    N | 0                 | 지출 예정 금액                  |
| target_hijack_amount    | BIGINT      |    N | 0                 | 목표 납치금액                   |
| expected_hijack_amount  | BIGINT      |    N | 0                 | 예상 납치금액                   |
| confirmed_hijack_amount | BIGINT      |    N | 0                 | 월말 확정 납치금액              |
| status                  | TEXT        |    N | 'ACTIVE'          | DRAFT, ACTIVE, CLOSED, ARCHIVED |
| created_at              | TIMESTAMPTZ |    N | now()             | 생성일                          |
| updated_at              | TIMESTAMPTZ |    N | now()             | 수정일                          |

```sql
CREATE TABLE payroll_plans (
  payroll_plan_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
  year_month CHAR(7) NOT NULL CHECK (year_month ~ '^\d{4}-\d{2}$'),
  payday SMALLINT NOT NULL CHECK (payday BETWEEN 1 AND 31),
  expected_salary_amount BIGINT NOT NULL DEFAULT 0 CHECK (expected_salary_amount >= 0),
  expected_expense_amount BIGINT NOT NULL DEFAULT 0 CHECK (expected_expense_amount >= 0),
  target_hijack_amount BIGINT NOT NULL DEFAULT 0 CHECK (target_hijack_amount >= 0),
  expected_hijack_amount BIGINT NOT NULL DEFAULT 0 CHECK (expected_hijack_amount >= 0),
  confirmed_hijack_amount BIGINT NOT NULL DEFAULT 0 CHECK (confirmed_hijack_amount >= 0),
  status TEXT NOT NULL DEFAULT 'ACTIVE' CHECK (status IN ('DRAFT','ACTIVE','CLOSED','ARCHIVED')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX uq_payroll_user_month_active ON payroll_plans(user_id, year_month) WHERE status = 'ACTIVE';
CREATE INDEX idx_payroll_user_month ON payroll_plans(user_id, year_month DESC);
```

### 4.6 fixed_expenses

| 컬럼명           | 타입        | Null | 기본값            | 설명                                                         |
| ---------------- | ----------- | ---: | ----------------- | ------------------------------------------------------------ |
| fixed_expense_id | UUID        |    N | gen_random_uuid() | PK                                                           |
| user_id          | UUID        |    N | -                 | FK users                                                     |
| payroll_plan_id  | UUID        |    N | -                 | FK payroll_plans                                             |
| expense_day      | SMALLINT    |    N | -                 | 결제일 1~31                                                  |
| category         | TEXT        |    N | -                 | SUBSCRIPTION, LOAN, INSURANCE, TELECOM, RENT, TRANSPORT, ETC |
| name             | TEXT        |    N | -                 | 항목명                                                       |
| amount           | BIGINT      |    N | -                 | 금액, 1원 이상                                               |
| recurrence_type  | TEXT        |    N | 'MONTHLY'         | ONCE, DAILY, WEEKLY, MONTHLY, YEARLY                         |
| status           | TEXT        |    N | 'SCHEDULED'       | SCHEDULED, PAID, SKIPPED, CANCELLED                          |
| paid_at          | TIMESTAMPTZ |    Y | null              | 납부 완료일                                                  |

```sql
CREATE TABLE fixed_expenses (
  fixed_expense_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
  payroll_plan_id UUID NOT NULL REFERENCES payroll_plans(payroll_plan_id) ON DELETE CASCADE,
  expense_day SMALLINT NOT NULL CHECK (expense_day BETWEEN 1 AND 31),
  category TEXT NOT NULL CHECK (category IN ('SUBSCRIPTION','LOAN','INSURANCE','TELECOM','RENT','TRANSPORT','CARD','ETC')),
  name TEXT NOT NULL,
  amount BIGINT NOT NULL CHECK (amount > 0),
  recurrence_type TEXT NOT NULL DEFAULT 'MONTHLY' CHECK (recurrence_type IN ('ONCE','DAILY','WEEKLY','MONTHLY','YEARLY')),
  status TEXT NOT NULL DEFAULT 'SCHEDULED' CHECK (status IN ('SCHEDULED','PAID','SKIPPED','CANCELLED')),
  paid_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_fixed_user_plan ON fixed_expenses(user_id, payroll_plan_id, expense_day);
```

## 5. 계산 재동기화 함수 기준

`VariableExpense` 생성/수정/삭제 시 `DailyBudget`은 반드시 다음 기준으로 재계산한다.

```sql
-- 개념 SQL: 실제 구현 시 트랜잭션 내부에서 실행
UPDATE daily_budgets db
SET used_amount = COALESCE((
      SELECT SUM(amount)
      FROM variable_expenses ve
      WHERE ve.daily_budget_id = db.daily_budget_id
        AND ve.status = 'ACTIVE'
    ), 0),
    remaining_amount = GREATEST(daily_limit_amount - COALESCE((
      SELECT SUM(amount)
      FROM variable_expenses ve
      WHERE ve.daily_budget_id = db.daily_budget_id
        AND ve.status = 'ACTIVE'
    ), 0), 0),
    over_amount = GREATEST(COALESCE((
      SELECT SUM(amount)
      FROM variable_expenses ve
      WHERE ve.daily_budget_id = db.daily_budget_id
        AND ve.status = 'ACTIVE'
    ), 0) - daily_limit_amount, 0),
    status = CASE
      WHEN COALESCE((SELECT SUM(amount) FROM variable_expenses ve WHERE ve.daily_budget_id = db.daily_budget_id AND ve.status = 'ACTIVE'), 0) > daily_limit_amount THEN 'EXCEEDED'
      WHEN COALESCE((SELECT SUM(amount) FROM variable_expenses ve WHERE ve.daily_budget_id = db.daily_budget_id AND ve.status = 'ACTIVE'), 0) >= daily_limit_amount * 0.8 THEN 'WARNING'
      ELSE 'NORMAL'
    END,
    updated_at = now()
WHERE db.daily_budget_id = :daily_budget_id;
```
