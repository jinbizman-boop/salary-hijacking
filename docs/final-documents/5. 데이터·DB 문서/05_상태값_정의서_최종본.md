# 05. 상태값 정의서 최종본

> 문서 상태: 최종본  
> 적용 서비스: 급여납치(Salary Hijacking) 모바일 플랫폼  
> 적용 범위: 급여관리 · 가계부 · 자기계발 레벨업 · 커뮤니티 · 알림 · 광고/제휴 · 운영 데이터  
> 기준 DB: Neon DB / PostgreSQL  
> 기준일: 2026-06-15  
> 원칙: 본 문서는 문서상·이론상 추가 작업 없이 구현 기준으로 사용할 수 있도록 완결된 기준값, 필드, 관계, 정책, 예외를 포함한다.

## 1. 문서 목적

본 문서는 급여납치 플랫폼에서 사용하는 모든 상태값을 정의한다. 상태값은 화면 표시, API 응답, DB 저장, QA 테스트, 운영자 처리 기준의 공통 계약이다.

## 2. 상태값 설계 원칙

1. 상태값은 대문자 영문 스네이크 케이스를 사용한다.
2. 화면 표시 문구는 상태값과 분리한다.
3. 삭제는 물리 삭제보다 상태 전환을 우선한다.
4. 금액 계산 대상은 활성 상태값만 포함한다.
5. 상태 전환은 허용된 전환만 가능하다.

## 3. 회원 상태 `users.status`

| 상태값    | 한글명   | 설명                        | 화면 처리                  | 데이터 처리               |
| --------- | -------- | --------------------------- | -------------------------- | ------------------------- |
| ACTIVE    | 활성     | 정상 이용 가능              | 모든 기능 사용             | 전체 데이터 활성          |

## 18. 상태값 QA 기준

| 검증 항목          | 합격 기준                                                              |
| ------------------ | ---------------------------------------------------------------------- |
| 잘못된 상태값 저장 | DB CHECK 제약 또는 서버 검증으로 실패해야 한다.                        |
| 삭제된 지출 계산   | `DELETED`, `REFUNDED`, `ADJUSTED`는 사용금액 합계에서 제외되어야 한다. |
| 읽은 알림 표시     | `READ` 알림은 새 알림 배지에 포함하지 않아야 한다.                     |
| 익명 게시글 표시   | `is_anonymous=true`일 경우 작성자 이름을 표시하지 않아야 한다.         |
| 숨김 게시글 목록   | `HIDDEN`, `BLOCKED` 게시글은 일반 목록에서 제외되어야 한다.            |
| 종료 광고 노출     | `ENDED`, `PAUSED`, `ARCHIVED` 광고는 노출되지 않아야 한다.             |
