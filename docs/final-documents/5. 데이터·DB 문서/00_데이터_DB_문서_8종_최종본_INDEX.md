## 2. 공통 설계 기준

급여납치 플랫폼의 모든 데이터는 다음 기준을 따른다.

1. 모든 핵심 테이블의 기본키는 `UUID`를 사용한다.
2. 모든 시간 컬럼은 `TIMESTAMPTZ`를 사용하고 서버 저장 기준은 UTC, 앱 표시는 Asia/Seoul 기준으로 변환한다.
3. 모든 금액은 원화 `KRW` 기준 `BIGINT` 정수 원 단위로 저장한다.
4. 지출·저축·급여·예산 금액에는 음수를 저장하지 않는다.
5. 취소·환불·정정은 음수 금액이 아니라 별도 상태값 또는 별도 정정 레코드로 처리한다.
6. 사용자별 데이터는 반드시 `user_id`를 기준으로 소유권을 분리한다.
7. 급여·대출·저축·소비 정보는 민감 재무 데이터로 분류한다.
8. 광고/제휴 데이터와 재무 원천 데이터는 논리적으로 분리한다.
9. 커뮤니티 익명 기능은 화면상 익명성을 보장하되, 신고/수사/운영 대응을 위해 내부 식별자는 분리 보관한다.
10. 삭제는 서비스 기능 삭제, 개인정보 삭제, 법정/운영 보존, 백업 삭제 주기를 구분하여 처리한다.
