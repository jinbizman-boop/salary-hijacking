# 04. 데이터 사전 최종본

> 문서 상태: 최종본  
> 적용 서비스: 급여납치(Salary Hijacking) 모바일 플랫폼  
> 적용 범위: 급여관리 · 가계부 · 자기계발 레벨업 · 커뮤니티 · 알림 · 광고/제휴 · 운영 데이터  
> 기준 DB: Neon DB / PostgreSQL  
> 기준일: 2026-06-15  
> 원칙: 본 문서는 문서상·이론상 추가 작업 없이 구현 기준으로 사용할 수 있도록 완결된 기준값, 필드, 관계, 정책, 예외를 포함한다.

## 1. 문서 목적

본 문서는 급여납치 플랫폼에서 사용하는 주요 데이터 필드의 의미, 타입, 단위, 예시값, 민감도, 사용 화면을 정의한다. 개발자, 기획자, 디자이너, QA, 운영자가 같은 용어를 사용하도록 고정한다.

## 2. 공통 필드 사전

| 필드명      | 한글명    | 타입        | 단위/형식  | 예시값                               | 민감도 | 설명                |
| ----------- | --------- | ----------- | ---------- | ------------------------------------ | ------ | ------------------- |
| user_id     | 사용자 ID | UUID        | UUID v4    | 11111111-1111-1111-1111-111111111111 | S3     | 사용자 고유 식별자  |
| created_at  | 생성일시  | TIMESTAMPTZ | UTC 저장   | 2026-06-15T00:00:00Z                 | S2     | 레코드 생성 시각    |
| updated_at  | 수정일시  | TIMESTAMPTZ | UTC 저장   | 2026-06-15T01:00:00Z                 | S2     | 레코드 수정 시각    |

| budget_date | 예산일    | DATE        | YYYY-MM-DD | 2026-06-15                           | S2     | 일일예산 기준일     |

## 13. 민감도 등급 정의

| 등급 | 의미             | 예시                           | 접근 기준                     |
| ---- | ---------------- | ------------------------------ | ----------------------------- |
| S1   | 공개 데이터      | 공개 게시글, 댓글              | 앱 사용자 조회 가능           |
