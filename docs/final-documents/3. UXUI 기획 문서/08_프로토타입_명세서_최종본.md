# 프로토타입 명세서 최종본

## 1. 문서 목적

본 문서는 기존 HTML 작동형 UI/UX 예시안을 실제 모바일 앱 구현 기준으로 변환하기 위한 최종 명세서다. HTML에서 검증된 화면 전환, 지출 추가, 계획 수정, 레벨업 완료, 커뮤니티 글쓰기 등록 동작을 앱 기능 요구사항으로 확정한다.

## 2. 프로토타입 범위

| 기능        | HTML 예시 동작                | 앱 구현 기준                           | 우선순위 |
| ----------- | ----------------------------- | -------------------------------------- | -------- |
| 로그인      | 소셜 버튼 클릭 시 홈 이동     | 인증 성공 후 홈 또는 초기 설정 이동    | P1       |
| 하단 탭     | 급여/계획/LV/커뮤니티/MY 전환 | 네이티브/웹뷰 라우팅 전환              | P1       |
| 지출 추가   | 금액 입력 후 홈 수치 변경     | API 저장 후 DailyBudget 갱신           | P1       |
| 계획 수정   | 급여/지출/목표 입력 후 재계산 | PayrollPlan 저장 후 전역 수치 갱신     | P1       |
| 레벨업 완료 | 경험치 증가, 레벨업 표시      | GrowthTask 완료 저장 후 UserLevel 갱신 | P2       |
| 커뮤니티 탭 | 게시판 필터 변경              | boardType 기준 목록 조회               | P2       |
| 글쓰기      | 작성 레이어에서 등록          | CommunityPost 생성 후 목록 반영        | P2       |
| 알림        | 아이콘 클릭 시 알림 화면      | Notification 목록 조회 및 읽음 처리    | P2       |

## 3. 상태 모델

```json
{
  "user": {
    "loggedIn": true,
    "nickname": "홍길동",
    "jobTitle": "기획자",
    "level": 18,
    "exp": 38
  },
  "payroll": {
    "salary": 2700000,
    "fixedExpense": 60000,
    "dailyLimit": 20000,
    "dailyUsed": 13000,
    "totalHijack": 5780000,
    "target": 6600000
  },
  "community": {
    "selectedBoard": "전체 게시판"
  },
  "levelUp": {
    "selectedCategory": "AI 추천"
  }
}
```

## 4. 계산 함수 명세

| 함수               | 입력                            | 출력      | 기준                                         |
| ------------------ | ------------------------------- | --------- | -------------------------------------------- |
| formatCurrency     | number                          | “1,000원” | ko-KR 콤마 + 원                              |
| getHijackAmount    | salary, fixedExpense, dailyUsed | number    | max(0, salary - fixedExpense - dailyUsed)    |
| getRemainingAmount | dailyLimit, dailyUsed           | number    | dailyLimit - dailyUsed                       |
| getAchievementRate | totalHijack, target             | number    | min(100, round(totalHijack / target \* 100)) |
| getLevelProgress   | exp                             | number    | 0~99 범위                                    |

## 6. 급여 홈 프로토타입 동작

### 6.1 초기 표시 데이터

| 항목               | 기본 예시값 | 앱 구현 기준              |
| ------------------ | ----------: | ------------------------- |
| 수령금액           | 2,700,000원 | 사용자 PayrollPlan 기준   |
| 고정지출           |    60,000원 | FixedExpense 합계         |
| 일일 설정금액      |    20,000원 | DailyBudget.dailyLimit    |
| 오늘 사용금액      |    13,000원 | 당일 VariableExpense 합계 |
| 전체 누적 납치금액 | 5,780,000원 | 확정 월별 누적합          |

### 6.2 빠른 지출 추가

| 단계 | HTML 동작                   | 앱 구현 기준                         |
| ---- | --------------------------- | ------------------------------------ |
| 1    | 입력창에 텍스트와 숫자 입력 | 항목명/금액 분리 또는 별도 필드 사용 |
| 2    | 추가 버튼 클릭              | 필수값 검증                          |
| 3    | 숫자 추출                   | 금액 필드는 숫자만 허용              |
| 4    | variableExpenses에 추가     | VariableExpense 생성 API 호출        |
| 5    | dailyUsed 증가              | 서버 응답 기준으로 갱신              |
| 6    | 홈 재렌더링                 | 상태관리 store 갱신                  |

## 7. 계획 수정 프로토타입 동작

| 항목         | 명세                                                   |
| ------------ | ------------------------------------------------------ |

## 12. 프로토타입 → 실제 앱 전환 기준

| 영역      | 프로토타입            | 실제 앱                       |
| --------- | --------------------- | ----------------------------- |
| 상태 저장 | 브라우저 메모리 state | 서버 DB + 클라이언트 상태관리 |
| 인증      | 버튼 클릭 가상 로그인 | OAuth/세션/토큰 인증          |
| 계산      | 클라이언트 함수       | 서버 검증 + 클라이언트 표시   |
| 게시글    | 배열 추가             | API 생성/조회/수정/삭제       |
| 알림      | 정적 배열             | 알림 테이블/푸시 시스템       |
| 콘텐츠    | 정적 카드             | 운영 콘텐츠 또는 외부 API     |
| 파일 첨부 | UI만 표시             | 업로드 저장소와 메타데이터    |
