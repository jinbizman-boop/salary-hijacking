# 반응형 디바이스 대응 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 모바일 앱이 iPhone, Android 주요 해상도, 작은 화면, 큰 화면, safe area, 키보드 표시 상황에서도 화면 깨짐 없이 동작하도록 반응형/디바이스 대응 기준을 최종 확정한다.

## 2. 대응 원칙

| 원칙             | 기준                                           |
| ---------------- | ---------------------------------------------- |
| 모바일 우선      | 360px 폭 기준에서도 핵심 기능 사용 가능        |
| 세로 스크롤 기본 | 가로 스크롤은 표 등 예외 상황만 허용           |
| Safe Area 준수   | 노치, 홈 인디케이터, 상태바 침범 금지          |
| 터치 우선        | 모든 터치 대상 최소 44x44px 확보               |
| 키보드 대응      | 입력 필드가 키보드에 가려지지 않아야 함        |
| 데이터 우선      | 작은 화면에서도 금액, CTA, 오류 문구 우선 노출 |

## 3. 기준 해상도

| 구분    |    기준 폭 | 예시 기기                    | 대응 수준                              |
| ------- | ---------: | ---------------------------- | -------------------------------------- |
| 최소 폭 |      320px | iPhone SE 구형, 소형 Android | 필수 대응                              |
| 소형    |      360px | 일반 Android 소형            | 필수 대응                              |
| 표준    |      390px | iPhone 12~15 표준            | 기본 디자인 기준                       |
| 대형    |      430px | iPhone Pro Max, 대형 Android | 확장 대응                              |
| 태블릿  | 768px 이상 | iPad/Android Tablet          | 1차 앱에서는 모바일 레이아웃 중앙 정렬 |

## 4. 레이아웃 브레이크포인트

| 범위       | 규칙                                                    |
| ---------- | ------------------------------------------------------- |
| 320~359px  | 카드 내부 2열 요소는 1열 전환 가능, 폰트 최소값 적용    |
| 360~389px  | 표준 모바일 압축 레이아웃                               |
| 390~430px  | 기본 디자인 레이아웃                                    |
| 431~767px  | 콘텐츠 최대 폭 430px, 중앙 정렬                         |
| 768px 이상 | 태블릿에서도 모바일 앱 프레임 유지 또는 2열 관리자 제외 |

## 5. Safe Area 기준

| 영역          | 기준                                                          |
| ------------- | ------------------------------------------------------------- |
| 상단 상태바   | OS safe area top + 앱 헤더 padding 적용                       |
| 하단 탭       | OS safe area bottom 반영                                      |
| FAB           | 하단 탭 + safe area 위에 배치                                 |
| 모달/바텀시트 | 홈 인디케이터 영역 침범 금지                                  |
| 스크롤 끝     | 마지막 콘텐츠가 하단 탭에 가려지지 않도록 padding-bottom 적용 |

## 6. 화면별 반응형 기준

### 6.1 급여 홈

| 요소            | 320~359px                               | 390px 이상   |
| --------------- | --------------------------------------- | ------------ |
| 급여 메트릭     | 2열 유지 불가 시 1열 전환               | 2열 유지     |
| 납치금액        | 한 줄 우선, 불가 시 글자 크기 축소      | 큰 숫자 강조 |
| 고정지출 리스트 | 금액/항목/버튼 줄바꿈 허용              | 한 행 유지   |
| 일일 예산 3분할 | 3열 유지, 폰트 축소                     | 3열 기본     |
| 빠른 추가       | 입력+버튼 2열, 320px에서는 버튼 폭 축소 | 2열 기본     |

### 6.2 계획

| 요소             | 기준                                |
| ---------------- | ----------------------------------- |
| 목표 달성률 카드 | 수치와 진행바 우선 표시             |
| 입력 그리드      | 390px 미만에서는 1열                |
| 계획표           | 가로 스크롤 허용, 첫 컬럼 고정 권장 |
| 저장 버튼        | 화면 폭 전체 사용                   |

### 6.3 커뮤니티

| 요소        | 기준                                     |
| ----------- | ---------------------------------------- |
| 게시판 탭   | 가로 스크롤 또는 4분할, 텍스트 잘림 금지 |
| 게시글 카드 | 썸네일 82px 기준, 320px에서는 64px       |
| 제목        | 최대 2줄, 이후 말줄임                    |
| FAB         | 우측 하단, 하단 탭과 16px 이상 간격      |

### 6.4 글쓰기

| 요소        | 기준                                       |
| ----------- | ------------------------------------------ |
| 키보드 표시 | 제목/본문 필드가 가려지지 않게 스크롤 이동 |
| 완료 버튼   | 상단 우측 고정                             |
| 첨부 영역   | 1행 가로 스크롤 가능                       |
| 본문        | 남은 높이 활용, 최소 160px                 |

## 8. 키보드 대응

| 화면           | 대응 기준                                            |
| -------------- | ---------------------------------------------------- |
| 로그인         | 비밀번호 입력 시 로그인 버튼까지 스크롤 가능         |
| 초기 급여 설정 | 다음/저장 버튼이 키보드에 가려지지 않음              |
| 지출 추가      | 금액 키패드 표시 시 추가 버튼 접근 가능              |
| 계획 입력      | 입력 필드 focus 시 해당 카드 상단으로 스크롤         |
| 글쓰기         | 본문 입력 시 완료 버튼은 상단 고정, 본문 영역 스크롤 |
| 문의 작성      | 첨부/보내기 버튼 접근 가능                           |
