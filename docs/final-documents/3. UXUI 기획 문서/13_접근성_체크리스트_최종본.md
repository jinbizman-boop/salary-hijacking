# 접근성 체크리스트 최종본

## 1. 문서 목적

본 문서는 급여납치 앱의 글자 크기, 색상 대비, 터치 영역, 스크린리더, 키보드/포커스, 오류 안내, 동작 접근성 기준을 최종 확정한다. 모든 화면은 본 체크리스트를 기준으로 디자인·개발·QA 검수를 수행한다.

## 2. 접근성 목표

| 목표          | 기준                                                        |
| ------------- | ----------------------------------------------------------- |
| 읽기 쉬움     | 핵심 금액과 버튼 문구는 작은 화면에서도 명확히 읽혀야 한다. |
| 누르기 쉬움   | 주요 버튼과 탭은 한 손 조작이 가능해야 한다.                |
| 이해하기 쉬움 | 오류와 상태 변화는 색상만이 아니라 문구로도 전달한다.       |
| 보조기기 대응 | 스크린리더 사용자가 주요 기능을 수행할 수 있어야 한다.      |
| 안전한 입력   | 급여·지출 입력 실수를 줄이고 수정 가능해야 한다.            |

## 3. 색상 대비 체크리스트

| 항목           | 기준                                       | 적용 상태 |
| -------------- | ------------------------------------------ | --------- |
| 본문 텍스트    | 배경 대비 최소 4.5:1                       | 필수      |
| 큰 금액 텍스트 | 배경 대비 최소 3:1 이상, 권장 4.5:1        | 필수      |
| 버튼 텍스트    | 버튼 배경 대비 최소 4.5:1                  | 필수      |
| 오류 텍스트    | 색상 + 아이콘/문구 병행                    | 필수      |
| 활성 탭        | 색상만 사용하지 않고 굵기/아이콘 변화 병행 | 필수      |
| 예산 초과      | 빨간색 + “초과” 문구 병행                  | 필수      |

## 4. 글자 크기 체크리스트

| 항목         |                      최소 기준 |
| ------------ | -----------------------------: |
| 본문         |                      14px 이상 |
| 보조 설명    |           11px 이상, 권장 12px |
| 버튼 라벨    |                      13px 이상 |
| 하단 탭 라벨 |                      11px 이상 |
| 금액         | 16px 이상, 핵심 금액 24px 이상 |
| 오류 문구    |                      12px 이상 |

## 5. 터치 영역 체크리스트

| 요소        |           최소 영역 | 기준 |
| ----------- | ------------------: | ---- |
| 하단 탭     |             44x44px | 필수 |
| 주요 버튼   |      높이 44px 이상 | 필수 |
| 아이콘 버튼 |   44x44px 터치 영역 | 필수 |
| 체크박스    |   44x44px 터치 영역 | 필수 |
| FAB         |        50x50px 이상 | 필수 |
| 게시글 카드 | 전체 카드 터치 가능 | 권장 |
| 알림 행     |   전체 행 터치 가능 | 권장 |

## 6. 스크린리더 라벨

| UI 요소        | 접근성 라벨 예시                |
| -------------- | ------------------------------- |
| 로고           | 급여납치 로고                   |
| 알림 버튼      | 알림 열기                       |
| 하단 급여 탭   | 급여 홈 탭, 선택됨              |
| 하단 계획 탭   | 계획 탭                         |
| 지출 추가 버튼 | 변동지출 추가하기               |
| 사용완료 버튼  | 해당 지출 사용완료 처리         |
| 납치금액       | 이번 급여에서 지켜낸 금액, n원  |
| 남은금액       | 오늘 사용 가능한 남은 금액, n원 |
| 글쓰기 FAB     | 커뮤니티 글쓰기                 |
| 닫기 버튼      | 글쓰기 닫기                     |
| 완료 버튼      | 게시글 등록 완료                |

## 7. 포커스 순서

| 화면       | 포커스 순서 기준                                                       |
| ---------- | ---------------------------------------------------------------------- |
| 로그인     | 앱명 → 아이디 → 비밀번호 → 로그인 → 소셜 버튼 → 회원가입 → 자동 로그인 |
| 급여 홈    | 헤더 → 알림 → 급여 카드 → 고정지출 → 일일 예산 → 지출 추가 → 하단 탭   |
| 계획       | 헤더 → 목표 달성률 → 급여 입력 → 저장 → 각 계획표 → 하단 탭            |
| LV UP      | 헤더 → 미션 버튼 → 레벨 카드 → 콘텐츠 탭 → 콘텐츠 카드 → 하단 탭       |
| 커뮤니티   | 헤더 → 게시판 탭 → 인기글 → 게시글 목록 → 글쓰기 → 하단 탭             |
| 글쓰기     | 닫기 → 제목 → 완료 → 첨부 → 옵션 → 게시판 → 제목 입력 → 본문 입력      |
| 마이페이지 | 프로필 → 설정 버튼 → 성과 카드 → 관리 메뉴 → 하단 탭                   |

## 8. 입력 접근성

| 항목        | 기준                                                    |
| ----------- | ------------------------------------------------------- |
| 라벨        | 모든 입력 필드는 시각적 라벨 또는 접근성 라벨을 가진다. |
| 오류        | 오류 발생 시 필드와 오류 문구가 연결되어야 한다.        |
| 키보드      | 금액 입력은 숫자 키패드를 기본으로 한다.                |
| 자동 포맷   | 콤마 포맷은 실제 값과 혼동되지 않게 처리한다.           |
| 포커스 유지 | 오류 발생 시 첫 오류 필드로 포커스를 이동한다.          |
| 저장 실패   | 입력값을 지우지 않고 유지한다.                          |

## 9. 동작/애니메이션 접근성

| 항목      | 기준                                     |
| --------- | ---------------------------------------- |
| 화면 전환 | 과도한 애니메이션 사용 금지              |
| 토스트    | 중요한 정보는 토스트만으로 전달하지 않음 |
| 자동 이동 | 스플래시 외 자동 이동 최소화             |
| 모션 감소 | OS의 Reduce Motion 설정 존중             |
| 깜빡임    | 초당 3회 이상 깜빡임 금지                |

## 10. 콘텐츠 접근성

| 항목   | 기준                                                       |
| ------ | ---------------------------------------------------------- |
| 이미지 | 의미 있는 이미지는 대체 텍스트 제공                        |
| 광고   | 광고임을 구분 가능한 형태로 표시                           |
| 아이콘 | 아이콘만으로 의미 전달 금지, 라벨 병행                     |
| 표     | 계획표는 헤더와 셀 관계가 이해 가능해야 함                 |
| 금액   | 색상 외에 수치와 단위 명확히 표시                          |
| 시간   | “8시간 전”과 같은 상대 시간은 상세에서 절대 시간 제공 가능 |

## 11. 화면별 접근성 체크리스트

| 화면       | 필수 체크                                      |
| ---------- | ---------------------------------------------- |
| 스플래시   | 로고 대체 텍스트, 자동 이동 후 포커스 정상     |
| 로그인     | 입력 필드 라벨, 비밀번호 보안, 소셜 버튼 라벨  |
| 급여 홈    | 핵심 금액 스크린리더 읽기, 예산 초과 문구 병행 |
| 계획       | 금액 입력 오류 연결, 표 가로 스크롤 안내       |
| LV UP      | 미션 완료 상태 문구, 진행바 값 읽기            |
| 커뮤니티   | 게시판 선택 상태, 게시글 카드 정보 읽기        |
| 글쓰기     | 필수값 오류, 작성 취소 확인, 첨부 라벨         |
| 알림       | 읽음/안읽음 상태, 알림 유형 라벨               |
| 마이페이지 | 성과 카드 수치 읽기, 설정 메뉴 라벨            |

## 12. QA 검수 체크리스트

| 번호     | 항목                            | 통과 기준                     | 상태 |
| -------- | ------------------------------- | ----------------------------- | ---- |
| A11Y-001 | 모든 버튼에 접근성 라벨 존재    | 스크린리더에서 의미 확인 가능 | 필수 |

| A11Y-004 | 오류가 색상만으로 표시되지 않음 | 문구/아이콘 병행              | 필수 |
| A11Y-005 | 하단 탭 선택 상태 전달          | “선택됨” 상태 제공            | 필수 |
| A11Y-006 | 입력 오류 시 포커스 이동        | 첫 오류 필드로 이동           | 필수 |
| A11Y-007 | 키보드로 입력창이 가려지지 않음 | 모든 입력 화면 통과           | 필수 |
| A11Y-008 | 이미지/광고 대체 정보 제공      | 의미 전달 가능                | 필수 |
| A11Y-009 | 진행바 값 전달                  | 퍼센트 또는 레벨 정보 제공    | 필수 |
