> 본 문서는 급여납치 플랫폼의 UX/UI 설계, 화면 구현, 인터랙션 구현, 디자인 시스템 적용, QA 검수의 최종 기준이다. 본 문서에 정의된 내용은 별도 변경 승인 전까지 최종 기준으로 적용한다.

# 디자인 시스템 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 앱의 색상, 폰트, 버튼, 카드, 입력창, 탭, 아이콘, 간격, 그림자, 상태 표현을 최종 정의한다. 모든 화면과 컴포넌트는 본 디자인 시스템을 기준으로 제작한다.

## 2. 브랜드 디자인 방향

| 항목        | 기준                                       |
| ----------- | ------------------------------------------ |
| 브랜드 인상 | 신뢰감 있는 금융관리 + 친근한 자기관리     |
| 주요 감정   | 월급을 지켜냈다는 안도감, 성취감, 통제감   |
| 시각 톤     | 초록 기반, 흰색 카드, 명확한 숫자 강조     |
| 금지 톤     | 과도한 도박성, 공격적 절약 강요, 불안 조장 |

## 4. 컬러 사용 규칙

1. Primary Green은 저장, 추가, 완료, 활성 상태에 사용한다.
2. Yellow는 금액 성과와 보상에만 제한적으로 사용한다.
3. Red는 오류, 초과, 삭제 등 주의가 필요한 상황에만 사용한다.
4. 광고 배너 색상은 본문 카드와 명확히 구분한다.
5. 급여·지출·저축 수치에는 대비 4.5:1 이상을 확보한다.

## 8. 버튼 시스템

| 버튼      | 스타일                     | 용도                  | 상태                             |
| --------- | -------------------------- | --------------------- | -------------------------------- |
| Primary   | 초록 배경, 흰 텍스트       | 저장, 추가, 완료      | default/pressed/disabled/loading |
| Secondary | 연초록 배경, 초록 텍스트   | 설정, 관리, 보조 이동 | default/pressed/disabled         |
| Ghost     | 흰 배경, 라인              | 취소, 필터            | default/selected                 |
| Danger    | 빨간 배경 또는 빨간 텍스트 | 삭제, 탈퇴            | default/confirm                  |
| Icon      | 배경 없음                  | 알림, 닫기, 뒤로가기  | default/pressed                  |
| FAB       | 원형 초록 버튼             | 커뮤니티 글쓰기       | default/pressed/hidden           |
