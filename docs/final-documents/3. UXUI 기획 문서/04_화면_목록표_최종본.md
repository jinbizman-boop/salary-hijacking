# 화면 목록표 최종본

## 1. 문서 목적

본 문서는 급여납치 앱 구현에 필요한 전체 사용자 화면과 개발 상태 관리 기준을 최종 확정한다. 화면 누락 방지, 우선순위 관리, QA 범위 산정, 디자인/개발 작업 분배의 기준으로 사용한다.

## 2. 우선순위 정의

| 우선순위 | 의미                  | 출시 기준                |
| -------- | --------------------- | ------------------------ |
| P1       | 핵심 급여관리 기능    | MVP 필수 포함            |
| P2       | 재방문·참여·운영 기능 | 1차 정식 출시 포함       |
| P3       | 고도화·운영 편의 기능 | 출시 후 순차 고도화 가능 |

## 3. 개발 상태 정의

| 상태         | 의미                               |
| ------------ | ---------------------------------- |
| 확정         | 문서상 최종 확정, 추가 기획 불필요 |
| 디자인대상   | UI 시안 제작 대상                  |
| 개발대상     | 프론트/백엔드 구현 대상            |
| QA대상       | 테스트 케이스 작성 및 검수 대상    |
| 완료기준정의 | 수락 기준까지 정의됨               |

## 5. 화면별 데이터 의존성

| 화면       | 필수 데이터                              | 없을 때 처리                |
| ---------- | ---------------------------------------- | --------------------------- |
| 급여 홈    | PayrollPlan, DailyBudget, ExpenseSummary | 초기 급여 설정 유도         |
| 계획       | PayrollPlan, FixedExpense, SavingsPlan   | 빈 표 + 추가 버튼           |
| LV UP      | UserLevel, GrowthTask                    | 기본 레벨 1, 오늘 미션 표시 |
| 커뮤니티   | CommunityPost                            | 빈 게시판 문구 + 글쓰기     |
| 마이페이지 | UserProfile, UserSummary                 | 기본 프로필 + 설정 유도     |
| 알림       | Notification                             | 빈 알림 상태 표시           |

## 6. 화면 완료 기준

| 검수 영역   | 기준                                           |
| ----------- | ---------------------------------------------- |
| UI 요소     | 문서에 정의된 주요 컴포넌트가 누락 없이 노출   |
| 입력 검증   | 필수값, 금액, 날짜, 글자 수 검증 적용          |
| 상태 처리   | 로딩, 빈 상태, 오류, 성공 상태 표시            |
| 연결        | 진입/복귀/탭 이동/상세 이동 정상 동작          |
| 데이터 반영 | 저장·삭제 후 관련 화면 수치와 목록 동기화      |
| 접근성      | 터치 영역, 대비, 대체 텍스트, 포커스 순서 준수 |

## 7. 최종 판정

본 화면 목록표에 정의된 화면 30개, 모달 10개, 상태 화면 8개를 기준으로 급여납치 앱의 사용자 화면 범위는 문서상·이론상 최종 확정한다.
