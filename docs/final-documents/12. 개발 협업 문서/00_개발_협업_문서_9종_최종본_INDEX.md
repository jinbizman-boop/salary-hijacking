## 3. 전역 협업 원칙

1. 모든 작업은 GitHub Issue로 등록된 뒤 수행한다.
2. Issue 없이 Pull Request를 생성할 수 없다.
3. main 브랜치 직접 푸시는 금지한다.
4. dev 브랜치도 Pull Request를 통해서만 병합한다.
5. 모든 기능은 기능 문서의 기능 ID와 연결한다.
6. 모든 화면 작업은 UX/UI 문서의 화면 ID와 연결한다.
7. 모든 API 작업은 API·백엔드 문서의 endpoint와 연결한다.
8. 모든 DB 변경은 데이터·DB 문서의 테이블 정의와 마이그레이션 기록을 함께 수정한다.
9. 모든 보안·개인정보 관련 작업은 보안·개인정보·정책 문서와 충돌하지 않아야 한다.
10. 모든 배포 후보는 QA·테스트 문서의 회귀 테스트와 출시 문서의 출시 체크리스트를 통과해야 한다.

개발팀은 본 문서 세트를 기준으로 작업 등록, 우선순위 산정, 스프린트 운영, 코드 작성, 리뷰, 병합, 배포, 회고를 수행한다.
