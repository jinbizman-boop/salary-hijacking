# 작업 백로그 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 모든 개발 작업을 Epic, Feature, Task 단위로 분해하고 우선순위, 담당자, 상태, 산출물을 관리하기 위한 최종 기준이다.

## 2. 백로그 계층 구조

```text
Product Goal
└── Epic
    └── Feature
        └── Task
            └── Sub-task
```

| 계층         | 설명             | 예시                                               |
| ------------ | ---------------- | -------------------------------------------------- |
| Product Goal | 제품 목표        | 월급 수령 후 지출을 통제하고 납치금액을 시각화한다 |
| Epic         | 큰 기능 묶음     | 급여 관리                                          |
| Feature      | 사용자 가치 단위 | 급여 계획 등록                                     |
| Task         | 개발 작업 단위   | POST /v1/payroll-plans API 구현                    |
| Sub-task     | 세부 작업        | 입력값 검증, 테스트 작성, 문서 수정                |

## 3. 우선순위 기준

| 우선순위 | 의미                       | 처리 기준                   |
| -------- | -------------------------- | --------------------------- |
| P0       | 서비스 중단/보안/출시 차단 | 즉시 착수, 당일 해결 원칙   |
| P1       | MVP 핵심 기능              | 해당 스프린트 내 완료       |
| P2       | 핵심 경험 보완             | 차기 스프린트 계획          |
| P3       | 개선/편의 기능             | 백로그 유지 후 여유 시 처리 |
| P4       | 아이디어                   | 검토 전까지 개발 착수 금지  |

## 4. 상태값

| 상태          | 의미           | 다음 상태            |
| ------------- | -------------- | -------------------- |
| `Backlog`     | 등록만 된 상태 | Ready                |
| `Ready`       | 요구사항 확정  | In Progress          |
| `In Progress` | 개발 중        | In Review            |
| `In Review`   | PR 리뷰 중     | QA                   |
| `QA`          | 테스트 중      | Done 또는 Rework     |
| `Rework`      | 수정 필요      | In Progress          |
| `Blocked`     | 차단           | Ready 또는 Cancelled |
| `Done`        | 완료           | 종료                 |
| `Cancelled`   | 취소           | 종료                 |

## 7. Task 작성 규칙

Task는 아래 필드를 반드시 가진다.

| 필드         | 설명                               |
| ------------ | ---------------------------------- |
| Task ID      | TASK-YYYY-NNN                      |
| 연결 Feature | FEAT-XXX                           |
| 담당자       | 1명 명확히 지정                    |
| 리뷰어       | 최소 1명                           |
| 예상 난이도  | S/M/L                              |

## 9. 백로그 정리 주기

| 주기      | 작업                              |
| --------- | --------------------------------- |
| 매일      | Blocked Issue 확인                |
| 매주      | 스프린트 백로그 정리              |
| 격주      | P2/P3 우선순위 재검토             |
| 매월      | Epic 진행률 점검                  |
| 릴리즈 전 | 출시 범위와 백로그 일치 여부 점검 |

## 10. 완료 기준

백로그는 모든 작업이 다음 조건을 만족할 때 정상 관리 상태로 본다.

- 모든 Issue가 Epic/Feature/Task 중 하나와 연결되어 있다.
- 우선순위와 상태값이 누락되지 않았다.
- P0/P1 항목은 담당자와 완료 예정 스프린트가 명시되어 있다.
- 완료된 작업은 PR, 테스트 결과, 릴리즈 노트와 연결되어 있다.
