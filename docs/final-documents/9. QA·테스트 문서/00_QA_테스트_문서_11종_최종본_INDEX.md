## 3. 공통 테스트 원칙

1. 급여·대출·저축·소비 데이터는 고위험 재무성 개인정보로 간주한다.
2. 금액 계산은 서버와 클라이언트가 동일한 정책을 사용하되, 최종 원장은 서버 계산값을 기준으로 한다.
3. 사용자가 금액, 날짜, 게시글, 알림, 권한, 파일을 조작할 수 있는 모든 지점은 정상/예외/경계/권한 테스트를 수행한다.
4. 출시 승인 기준은 QA 종료 기준, 보안 테스트 통과, UAT 승인, 회귀 테스트 통과가 모두 충족된 상태다.
5. 본 문서 세트 내 모든 항목은 우선순위 P1/P2/P3로 관리하되, P1 실패는 출시 차단 결함으로 본다.

## 4. 우선순위 정의

| 우선순위 | 의미                                              | 출시 영향                     |
| -------- | ------------------------------------------------- | ----------------------------- |
| P1       | 핵심 급여관리·로그인·금액 계산·보안·개인정보 기능 | 실패 시 출시 불가             |
| P2       | 알림·레벨업·커뮤니티·마이페이지 주요 기능         | 다수 실패 시 출시 보류        |
| P3       | 광고·제휴·편의 기능·비핵심 UI                     | 실패 시 핫픽스/차기 배포 가능 |

## 5. 결함 심각도 정의

| 심각도   | 정의                        | 예시                                     | 조치                   |
| -------- | --------------------------- | ---------------------------------------- | ---------------------- |
| Blocker  | 앱 사용 또는 출시 자체 불가 | 로그인 불가, DB 저장 실패, 개인정보 노출 | 즉시 수정, 출시 중단   |
| Critical | 핵심 금액/권한/보안 오류    | 납치금액 오계산, 타인 게시글 수정 가능   | 출시 전 수정 필수      |
| Major    | 주요 기능 실패              | 지출 삭제 실패, 알림 읽음 처리 실패      | 출시 전 수정 원칙      |
| Minor    | 일부 UI/문구/편의 오류      | 문구 오탈자, 정렬 깨짐                   | 배포 전 또는 차기 수정 |
| Trivial  | 사용 영향 낮음              | 내부 로그 문구, 색상 미세 차이           | 백로그 등록            |

## 6. 최종 완료 선언

본 QA·테스트 문서 세트는 급여납치 플랫폼의 문서상·이론상 테스트 준비, 검증 기준, 승인 기준을 모두 포함한다. 본 문서 세트 기준으로 별도의 추가 QA 기획 문서는 요구되지 않는다.
