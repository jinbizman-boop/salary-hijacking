# 09. 회귀 테스트 체크리스트 최종본

## 1. 문서 목적

본 문서는 수정, 리팩터링, 배포 전후에 기존 기능이 손상되지 않았는지 반복 확인하기 위한 회귀 테스트 기준을 정의한다.

## 2. 회귀 테스트 수행 시점

| 시점               | 수행 범위               |
| ------------------ | ----------------------- |
| 기능 병합 후       | 수정 기능 + 영향 기능   |
| 핫픽스 후          | 수정 기능 + P1 smoke    |
| 배포 후보 생성 후  | 전체 필수 회귀          |
| DB 마이그레이션 후 | API/계산/조회/저장 전체 |
| 인증/권한 수정 후  | 모든 보호 API           |
| 금액 계산 수정 후  | 금액 계산 전체          |

| 기준      | 완료 조건              |
| --------- | ---------------------- |
| Smoke     | 100% PASS              |
| P1 회귀   | 100% PASS              |
| 금액 계산 | 100% PASS              |
| 인증/권한 | 100% PASS              |

## 12. 최종 완료 기준

본 회귀 테스트 체크리스트는 급여납치 플랫폼의 수정 후 기존 기능 보호를 위한 모든 반복 검증 항목을 포함한다. 배포 후보는 본 문서의 필수 회귀 항목을 통과해야 한다.
