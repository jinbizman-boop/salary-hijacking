# 05. 금액 계산 테스트 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 핵심 금액 계산 로직을 검증하기 위한 테스트 기준을 정의한다. 금액 계산 오류는 사용자 신뢰와 서비스 핵심 가치에 직접 영향을 주므로 P1 품질 항목으로 관리한다.

## 2. 금액 처리 기본 원칙

| 항목       | 정책                                       |
| ---------- | ------------------------------------------ |
| 통화       | KRW 원화                                   |
| 자료형     | 정수 integer 또는 bigint                   |
| 소수점     | 허용하지 않음                              |
| 반올림     | 없음                                       |
| 음수 입력  | 저장 차단                                  |
| 표시       | 사용자 화면에서는 천 단위 콤마와 `원` 표시 |
| 서버 기준  | 최종 계산 원장은 서버 계산값 기준          |
| 클라이언트 | 즉시성 제공용 미리보기 계산 가능           |

## 3. 핵심 계산식

| 계산 항목      | 공식                                             | 설명                     |
| -------------- | ------------------------------------------------ | ------------------------ |
| 지출 예정 금액 | `고정지출합계 + 월생활비계획 + 기타예정지출`     | 월 기준 예상 지출        |
| 예상 납치금액  | `max(0, 수령예정급여 - 지출예정금액)`            | 음수는 0 처리            |
| 오늘 사용금액  | `해당 날짜 변동지출 합계`                        | 삭제/수정 반영           |
| 오늘 남은금액  | `일일설정금액 - 오늘사용금액`                    | 음수 허용 표시 가능      |
| 오늘 초과금액  | `max(0, 오늘사용금액 - 일일설정금액)`            | 초과 알림 기준           |
| 누적 납치금액  | `이전 누적 납치금액 + 월별 확정 납치금액`        | 월 마감 시 확정          |
| 목표 달성률    | `min(999, floor(누적납치금액 / 목표금액 * 100))` | 목표 0이면 계산 차단     |
| 레벨 경험치    | `기존경험치 + 획득경험치`                        | 100 이상이면 레벨업 처리 |

## 4. 기본 계산 테스트

| ID       | 입력                            | 기대결과                                       |
| -------- | ------------------------------- | ---------------------------------------------- |
| CALC-001 | 급여 2,700,000 / 지출 600,000   | 예상 납치금액 2,100,000                        |
| CALC-002 | 급여 2,700,000 / 지출 2,700,000 | 예상 납치금액 0                                |
| CALC-003 | 급여 2,700,000 / 지출 3,000,000 | 예상 납치금액 0, 초과 경고                     |
| CALC-004 | 일일예산 20,000 / 사용 13,000   | 남은금액 7,000                                 |
| CALC-005 | 일일예산 20,000 / 사용 20,000   | 남은금액 0                                     |
| CALC-006 | 일일예산 20,000 / 사용 23,500   | 남은금액 -3,500, 초과금액 3,500                |
| CALC-007 | 누적 5,780,000 / 목표 6,600,000 | 달성률 87%                                     |
| CALC-008 | 누적 6,600,000 / 목표 6,600,000 | 달성률 100%                                    |
| CALC-009 | 누적 7,000,000 / 목표 6,600,000 | 달성률 106% 또는 UI 정책상 100% 표시 여부 확인 |

## 5. 입력값 경계 테스트

| ID           | 입력                         | 기대결과                                            |
| ------------ | ---------------------------- | --------------------------------------------------- |
| CALC-BND-001 | 금액 빈 값                   | 저장 차단                                           |
| CALC-BND-002 | 금액 문자열 `abc`            | 저장 차단                                           |
| CALC-BND-003 | 금액 `10,000`                | 서버 전송 시 10000 정수 변환                        |
| CALC-BND-004 | 금액 `10000.5`               | 저장 차단                                           |
| CALC-BND-005 | 금액 `-1`                    | 저장 차단                                           |
| CALC-BND-006 | 금액 `0`                     | 항목별 정책에 따라 차단 또는 허용, 급여/예산은 차단 |
| CALC-BND-007 | 매우 큰 금액 999,999,999,999 | bigint 범위 내 저장, UI 깨짐 없음                   |
| CALC-BND-008 | 앞뒤 공백 포함 금액          | trim 후 검증                                        |

## 12. 최종 완료 기준

본 금액 계산 테스트 문서는 급여납치 플랫폼의 핵심 금액 로직 검증에 필요한 모든 기준을 포함한다. 납치금액, 일일 예산, 누적 납치금액, 목표 달성률 계산은 본 문서를 기준으로 통과 여부를 판단한다.
