# 03. 시나리오 테스트 문서 최종본

## 1. 문서 목적

본 문서는 실제 사용자가 급여납치 앱을 사용하는 흐름을 기준으로 E2E 테스트 시나리오를 정의한다. 단일 기능 검증이 아니라 사용자의 목적 달성 가능 여부를 확인한다.

## 2. 공통 테스트 사용자

| 사용자      | 유형                 | 특징                                              |
| ----------- | -------------------- | ------------------------------------------------- |
| qa_user_001 | 사회초년생 직장인    | 월급 2,700,000원, 구독료 다수, 일일 예산 20,000원 |
| qa_user_002 | 아르바이트생         | 비정기 급여, 적은 고정지출, 소비 기록 중심        |
| qa_user_003 | 소비관리 필요자      | 예산 초과가 자주 발생, 알림 활용                  |
| qa_user_004 | 자기계발 중심 사용자 | 레벨업 미션과 커뮤니티 인증 사용                  |

## 3. E2E-001 신규 가입 후 급여 계획 완성

| 항목     | 내용                                                                       |
| -------- | -------------------------------------------------------------------------- |
| 목적     | 신규 사용자가 첫 진입 후 급여 관리 핵심 루프를 완성할 수 있는지 검증       |
| 우선순위 | P1                                                                         |
| 사전조건 | 앱 신규 설치, 계정 없음                                                    |
| 데이터   | 급여 2,700,000원, 급여일 매월 25일, 고정지출 315,000원, 일일 예산 20,000원 |

### 절차

1. 앱 실행
2. 스플래시 확인
3. 회원가입 또는 소셜 로그인 진행
4. 급여일 입력
5. 수령 예정 급여 입력
6. 고정지출 3건 입력
7. 고정저축 2건 입력
8. 일일 생활비 입력
9. 급여 홈 진입
10. 수령금액, 지출금액, 납치금액, 누적 납치금액 확인

### 기대결과

- 사용자는 3분 이내에 첫 급여 계획을 완성할 수 있다.
- 필수 입력 누락 시 저장이 차단된다.
- 급여 홈에 계산 결과가 즉시 표시된다.
- 납치금액이 계산 정책과 일치한다.

## 4. E2E-002 급여일 당일 소비 기록과 예산 초과 대응

| 항목     | 내용                                                            |
| -------- | --------------------------------------------------------------- |
| 목적     | 사용자가 당일 지출을 기록하고 예산 상태를 확인할 수 있는지 검증 |
| 우선순위 | P1                                                              |
| 사전조건 | 급여 계획과 일일 예산이 존재                                    |
| 데이터   | 일일 예산 20,000원, 지출 6,500원/2,000원/15,000원               |

### 절차

1. 급여 홈 진입
2. 일일 사용 예산 카드 확인
3. 점심식사 6,500원 지출 추가
4. 커피 2,000원 지출 추가
5. 게임 결제 15,000원 지출 추가
6. 사용금액과 남은금액 확인
7. 예산 초과 알림 확인

### 기대결과

- 사용금액은 23,500원으로 표시된다.
- 남은금액은 -3,500원 또는 초과 3,500원으로 표시된다.
- 예산 초과 알림이 생성된다.
- 납치금액 또는 관리 가능 금액이 정책에 따라 재계산된다.

## 5. E2E-003 고정지출 결제 예정 알림 확인

| 항목     | 내용                                           |
| -------- | ---------------------------------------------- |
| 목적     | 고정지출 예정일 알림과 납부완료 처리 흐름 검증 |
| 우선순위 | P2                                             |
| 사전조건 | 고정지출 항목 존재, 푸시 권한 허용             |
| 데이터   | 유튜브 프리미엄 14,900원, 결제일 D-1           |

### 절차

1. 배치 스케줄러 실행 또는 예정일 조건 생성
2. 알림 목록 확인
3. 푸시 수신 확인
4. 알림 클릭
5. 고정지출 상세 이동
6. 납부완료 처리

### 기대결과

- 결제 예정 알림이 중복 없이 생성된다.
- 푸시 클릭 시 해당 고정지출 상세로 이동한다.
- 납부완료 상태가 저장된다.

## 6. E2E-004 레벨업 미션 완료와 커뮤니티 인증

| 항목     | 내용                                                              |
| -------- | ----------------------------------------------------------------- |
| 목적     | 자기계발 미션 완료 후 커뮤니티 인증까지 이어지는 리텐션 흐름 검증 |
| 우선순위 | P2                                                                |
| 사전조건 | 로그인, 레벨업 탭 접근 가능                                       |

### 절차

1. LV UP 탭 진입
2. 오늘의 독서 미션 선택
3. 미션 완료 처리
4. 경험치 증가 확인
5. 레벨업 조건 충족 여부 확인
6. 커뮤니티 탭 이동
7. 레벨업 인증 게시판 선택
8. 인증 글 작성
9. 게시글 등록 확인

### 기대결과

- 미션 완료 상태가 당일 1회만 저장된다.
- 경험치와 레벨이 정확히 반영된다.
- 인증 글은 레벨업 인증 게시판에 노출된다.

## 7. E2E-005 커뮤니티 신고와 관리자 조치

| 항목     | 내용                                                     |
| -------- | -------------------------------------------------------- |
| 목적     | 사용자 신고부터 관리자 숨김/제재 처리까지 운영 흐름 검증 |
| 우선순위 | P2                                                       |
| 사전조건 | 게시글 존재, 관리자 계정 존재                            |

### 절차

1. 일반 사용자가 게시글 신고
2. 신고 사유 선택
3. 중복 신고 여부 확인
4. 관리자 로그인
5. 신고 목록 조회
6. 게시글 숨김 처리
7. 사용자 제재 처리
8. 일반 사용자 화면에서 노출 여부 확인

### 기대결과

- 신고가 정상 접수된다.
- 관리자만 신고 처리 API에 접근 가능하다.
- 숨김 처리된 게시글은 일반 목록에서 보이지 않는다.
- 제재 사용자는 글쓰기 제한을 받는다.

## 8. E2E-006 개인정보 삭제/탈퇴 흐름

| 항목     | 내용                                              |
| -------- | ------------------------------------------------- |
| 목적     | 회원 탈퇴 시 개인정보 삭제 및 보존 정책 적용 검증 |
| 우선순위 | P1                                                |
| 사전조건 | 급여/지출/게시글 데이터 보유 계정                 |

### 절차

1. 마이페이지 진입
2. 계정 설정 선택
3. 회원 탈퇴 선택
4. 경고 문구 확인
5. 탈퇴 확인
6. 재로그인 시도
7. DB 데이터 삭제/익명화 상태 확인

### 기대결과

- 탈퇴 후 로그인할 수 없다.
- 급여·지출·저축 데이터는 삭제된다.
- 법적/운영상 보존 대상은 정책에 맞게 분리 보존된다.
- 커뮤니티 글은 정책에 따라 삭제 또는 작성자 익명화된다.

## 9. E2E-007 네트워크 불안정 상태 지출 기록

| 항목     | 내용                                                 |
| -------- | ---------------------------------------------------- |
| 목적     | 네트워크 불안정 시 데이터 유실과 중복 저장 방지 검증 |
| 우선순위 | P1                                                   |
| 사전조건 | 로그인, 일일 예산 존재                               |

### 절차

1. 네트워크를 느린 3G 또는 불안정 상태로 전환
2. 지출 5,000원 등록
3. 저장 중 앱을 백그라운드로 전환
4. 앱 복귀
5. 지출 목록과 서버 데이터 확인

### 기대결과

- 중복 저장되지 않는다.
- 실패 시 재시도 문구가 표시된다.
- 성공 시 로컬/서버 상태가 일치한다.

## 10. 시나리오 종료 기준

| 기준          | 완료 조건                                              |
| ------------- | ------------------------------------------------------ |
| 핵심 루프     | 신규 사용자 급여 계획 → 지출 기록 → 납치금액 확인 성공 |
| 리텐션 루프   | 레벨업 미션 → 알림 → 커뮤니티 인증 성공                |
| 운영 루프     | 신고 → 관리자 처리 → 사용자 화면 반영 성공             |
| 개인정보 루프 | 탈퇴 → 삭제/익명화 → 재접근 차단 성공                  |

## 11. 최종 완료 기준

본 시나리오 테스트 문서는 급여납치 플랫폼의 실제 사용자 흐름 검증 기준을 모두 포함한다. 본 문서의 P1 시나리오가 모두 통과해야 출시 후보 빌드로 인정한다.
