# 11. UAT 사용자 인수 테스트 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼이 실제 대표 사용자 관점에서 출시 가능한 품질인지 검증하기 위한 사용자 인수 테스트(UAT) 기준을 정의한다.

## 2. UAT 목표

1. 사용자가 급여 계획을 이해하고 설정할 수 있는지 확인한다.
2. 사용자가 일일 지출을 쉽게 기록하고 남은 예산을 확인할 수 있는지 확인한다.
3. 납치금액이라는 핵심 개념이 직관적으로 전달되는지 확인한다.
4. 레벨업 미션과 커뮤니티 인증 흐름이 자연스러운지 확인한다.
5. 개인정보와 급여 데이터 보호 안내가 신뢰감을 주는지 확인한다.
6. 출시 전 치명적인 UX/기능/문구 문제를 제거한다.

## 3. UAT 참여자

| 그룹              | 인원 | 조건                  | 검증 포인트        |
| ----------------- | ---: | --------------------- | ------------------ |
| 사회초년생 직장인 |  3명 | 월급제, 고정지출 있음 | 급여계획/예산 이해 |
| 일반 직장인       |  3명 | 구독/대출/저축 있음   | 고정지출/저축 관리 |
| 아르바이트생      |  2명 | 비정기 급여           | 급여일/예산 유연성 |
| 소비관리 필요자   |  2명 | 예산 초과 경험 많음   | 알림/지출 기록     |
| 자기계발 관심자   |  2명 | 독서/운동/영어 관심   | LV UP/커뮤니티     |

## 6. UAT 시나리오 1: 첫 사용과 급여 계획 설정

| 항목      | 내용                                              |
| --------- | ------------------------------------------------- |
| 목적      | 첫 사용자가 앱 목적과 핵심 개념을 이해하는지 확인 |
| 제한 시간 | 5분                                               |

### 수행 과제

1. 앱을 실행한다.
2. 로그인 또는 소셜 로그인한다.
3. 급여일을 입력한다.
4. 수령 예정 급여를 입력한다.
5. 고정지출 2건을 등록한다.
6. 일일 예산을 설정한다.
7. 급여 홈에서 납치금액을 확인한다.

### 성공 기준

- 사용자가 도움 없이 급여 계획을 완성한다.
- 납치금액 의미를 설명할 수 있다.
- 입력 오류 발생 시 스스로 수정할 수 있다.

## 7. UAT 시나리오 2: 오늘 지출 기록

| 항목 | 내용                                                 |
| ---- | ---------------------------------------------------- |
| 목적 | 사용자가 소비 직후 지출을 쉽게 기록할 수 있는지 확인 |

### 수행 과제

1. 급여 홈에 진입한다.
2. 커피 2,000원 지출을 추가한다.
3. 점심 8,000원 지출을 추가한다.
4. 사용금액과 남은금액을 확인한다.
5. 잘못 입력한 지출을 수정 또는 삭제한다.

### 성공 기준

- 사용자가 지출 추가 위치를 바로 찾는다.
- 남은금액 변화가 즉시 이해된다.
- 수정/삭제 후 계산 결과를 신뢰한다.

## 8. UAT 시나리오 3: 예산 초과와 알림

| 항목 | 내용                                                              |
| ---- | ----------------------------------------------------------------- |
| 목적 | 예산 초과 상황에서 사용자가 경고를 이해하고 행동할 수 있는지 확인 |

### 수행 과제

1. 일일 예산보다 큰 금액을 입력한다.
2. 초과 표시를 확인한다.
3. 알림 화면에 진입한다.
4. 예산 초과 알림을 확인한다.

### 성공 기준

- 사용자가 초과 상태를 즉시 인지한다.
- 알림 문구가 비난이 아닌 관리 유도 방식으로 느껴진다.

## 9. UAT 시나리오 4: 레벨업과 커뮤니티 인증

| 항목 | 내용                                                   |
| ---- | ------------------------------------------------------ |
| 목적 | 자기계발 루틴과 커뮤니티 인증 흐름이 자연스러운지 확인 |

### 수행 과제

1. LV UP 탭에 진입한다.
2. 오늘의 독서 또는 운동 미션을 완료한다.
3. 경험치 변화를 확인한다.
4. 커뮤니티로 이동한다.
5. 레벨업 인증 글을 작성한다.

### 성공 기준

- 미션 완료 보상이 명확하다.
- 인증 글 작성 흐름이 어렵지 않다.
- 익명/질문 옵션 의미가 명확하다.

## 10. UAT 시나리오 5: 개인정보 신뢰 확인

| 항목 | 내용                                                    |
| ---- | ------------------------------------------------------- |
| 목적 | 급여·소비 데이터 입력 전 사용자가 신뢰할 수 있는지 확인 |

### 수행 과제

1. 개인정보/약관 동의 화면을 확인한다.
2. 급여 데이터가 어떻게 쓰이는지 확인한다.
3. 광고 데이터 분리 안내를 확인한다.
4. 마이페이지에서 탈퇴/문의 경로를 확인한다.

### 성공 기준

- 사용자가 급여 데이터를 입력해도 되는 이유를 이해한다.
- 광고와 급여 데이터가 분리된다는 안내를 확인한다.
- 탈퇴/삭제 권리를 찾을 수 있다.
