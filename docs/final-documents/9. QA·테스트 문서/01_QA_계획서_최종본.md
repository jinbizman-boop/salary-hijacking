# 01. QA 계획서 최종본

## 1. 문서 목적

본 문서는 급여납치 모바일 플랫폼의 전체 QA 전략, 테스트 범위, 테스트 수준, 역할, 환경, 일정, 품질 게이트, 진입·종료 기준을 확정하는 최상위 QA 계획서다. 본 문서를 기준으로 기능 테스트, API 테스트, 성능 테스트, 보안 테스트, 회귀 테스트, UAT가 수행된다.

## 7. 테스트 데이터 원칙

1. 실사용자 급여·대출·저축·소비 데이터는 테스트 환경에 반입하지 않는다.
2. 테스트 사용자는 `qa_user_001`, `qa_worker_001`, `qa_admin_001`처럼 식별 가능한 가명 계정만 사용한다.
3. 금액 데이터는 현실적인 원 단위 예시를 사용하되 실제 개인의 급여와 연결하지 않는다.
4. 커뮤니티 테스트 글에는 개인정보, 실명, 회사명, 계좌번호를 포함하지 않는다.
5. 이미지/파일 테스트는 악성 파일이 아닌 무해한 테스트 파일과 확장자 위장 파일을 별도로 사용한다.

## 11. 결함 처리 흐름

1. QA가 결함을 등록한다.
2. 결함 심각도와 우선순위를 지정한다.
3. 담당 개발자가 원인을 분석한다.
4. 수정 브랜치에서 패치한다.
5. 코드리뷰와 자동 테스트를 통과한다.
6. Staging에 재배포한다.
7. QA가 재검증한다.
8. 회귀 테스트 범위에 포함한다.
9. 종료 또는 재오픈 처리한다.

## 12. 최종 승인 기준

본 QA 계획서는 급여납치 플랫폼의 출시 검증에 필요한 테스트 체계, 환경, 역할, 품질 기준, 종료 조건을 모두 정의한다. 본 문서 기준을 충족하면 QA 관점에서 배포 후보 빌드를 승인할 수 있다.
