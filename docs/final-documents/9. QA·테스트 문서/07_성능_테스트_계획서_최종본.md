# 07. 성능 테스트 계획서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 API 부하, 게시글 목록, 알림 발송, DB 쿼리, 동시 접속, 스파이크, 내구성 성능 테스트 기준을 확정한다.

## 4. 테스트 유형

| 유형        | 목적                      | 주요 대상                     |
| ----------- | ------------------------- | ----------------------------- |
| Load Test   | 목표 부하에서 안정성 검증 | 홈/API/DB                     |
| Stress Test | 한계점 확인               | API 서버, DB 커넥션           |
| Spike Test  | 순간 트래픽 대응          | 로그인, 알림 클릭             |
| Soak Test   | 장시간 안정성             | 메모리 누수, DB 커넥션 누수   |
| Volume Test | 대량 데이터 처리          | 게시글, 지출, 알림            |
| Batch Test  | 스케줄러 성능             | 월말 납치금액 확정, 알림 생성 |

## 7. 캐싱 성능 테스트

| 대상             | 캐시 정책                         | 검증                       |
| ---------------- | --------------------------------- | -------------------------- |
| 앱 공지          | CDN/Edge Cache                    | 최신 공지 갱신 반영        |
| 배너             | Edge Cache + TTL                  | 비활성 배너 미노출         |
| 콘텐츠 추천      | TTL Cache                         | 오래된 추천 과다 노출 없음 |
| 게시글 목록      | 짧은 TTL 또는 캐시 제한           | 댓글/좋아요 정합성 유지    |
| 개인 급여 데이터 | 캐시 금지 또는 사용자별 엄격 캐시 | 타 사용자 데이터 노출 없음 |
