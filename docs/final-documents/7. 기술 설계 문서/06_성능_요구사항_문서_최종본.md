# 06. 성능 요구사항 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 모바일 앱과 API 서버의 성능 목표, 측정 기준, 병목 관리, 테스트 기준을 최종 확정한다.

## 2. 성능 목표 요약

| 영역                 | 기준                                       |
| -------------------- | ------------------------------------------ |
| 앱 실행 후 첫 화면   | 캐시 사용자 2초 이하, 신규 로그인 3초 이하 |
| 하단 탭 전환         | 300ms 이하 체감                            |
| 일반 API p95         | 300ms 이하                                 |
| 쓰기 API p95         | 500ms 이하                                 |
| 복합 통계 API p95    | 1,000ms 이하                               |
| 커뮤니티 목록 p95    | 500ms 이하                                 |
| 파일 업로드 URL 발급 | 300ms 이하                                 |
| 푸시 생성            | 단건 300ms 이하, 대량은 비동기             |
| 오류율               | 5xx 0.5% 이하, 인증/검증 오류 제외         |

## 3. 화면별 성능 기준

| 화면       |       목표 | 필수 최적화                                     |
| ---------- | ---------: | ----------------------------------------------- |
| 로그인     |   2초 이하 | 소셜 버튼 즉시 반응, provider 연동 loading 표시 |
| 급여 홈    |   2초 이하 | summary API, skeleton, cache 우선 표시          |
| 계획       |   2초 이하 | 월별 데이터 lazy load                           |
| LV UP      |   2초 이하 | 콘텐츠 cache, 이미지 lazy load                  |
| 커뮤니티   | 2.5초 이하 | cursor pagination, 썸네일 최적화                |
| 글쓰기     |  즉시 진입 | draft local state, 첨부 업로드 비동기           |
| 마이페이지 |   2초 이하 | profile + stats 병렬 조회                       |
| 알림       |   2초 이하 | 목록 pagination, 읽음 bulk 처리                 |

## 4. API 성능 예산

| API 유형    |   p50 |     p95 |     p99 |    Timeout |
| ----------- | ----: | ------: | ------: | ---------: |
| 인증/토큰   | 150ms |   400ms |   800ms |         3s |
| 단건 조회   | 100ms |   300ms |   700ms |         3s |
| 목록 조회   | 200ms |   500ms | 1,000ms |         5s |
| 쓰기        | 200ms |   500ms | 1,000ms |         5s |
| 복합 요약   | 300ms | 1,000ms | 1,500ms |         5s |
| 관리자 통계 | 500ms | 2,000ms | 5,000ms |        10s |
| 배치        |     - |       - |       - | job별 정의 |

## 5. 앱 최적화 기준

| 항목          | 기준                                          |
| ------------- | --------------------------------------------- |
| 초기 번들     | 불필요 라이브러리 제거, 화면 단위 lazy import |
| 이미지        | WebP/압축, 썸네일 사용, 원본 직접 표시 금지   |
| 리스트        | FlatList/FlashList 사용, keyExtractor 필수    |
| 금액 입력     | debounce 150ms 이하, 계산은 local 즉시 표시   |
| API 중복 호출 | TanStack Query dedupe 사용                    |
| cache         | 홈/알림/프로필은 stale cache 우선 표시        |
| 푸시 진입     | 딥링크 후 해당 화면만 데이터 로드             |
| 로그          | 사용자 행동 로그는 batch 전송                 |

## 6. DB 성능 기준

| 기준              | 최종값                                    |
| ----------------- | ----------------------------------------- |
| 단일 쿼리 목표    | 100ms 이하, 복잡 쿼리 300ms 이하          |
| API 쿼리 timeout  | 3초                                       |
| 배치 쿼리 timeout | 30초                                      |
| transaction       | 3초 이하, 외부 API 호출 포함 금지         |
| 목록 조회         | `LIMIT` 필수, 기본 20, 최대 100           |
| 커뮤니티 조회     | cursor 기반 권장                          |
| 금액 집계         | 실시간 full scan 금지, summary table 사용 |

## 7. 이미지/파일 성능 기준

| 항목          | 기준                                          |
| ------------- | --------------------------------------------- |
| 프로필 이미지 | 512x512 이하 썸네일 표시                      |
| 게시글 썸네일 | 목록 320px 기준 별도 썸네일                   |
| 배너 이미지   | 사전 리사이징, CDN cache                      |
| 업로드        | API 서버 binary 중계 금지, presigned URL 사용 |
| 다운로드      | R2/CDN 직접 제공                              |

## 8. 성능 테스트 시나리오

| 시나리오       |               부하 | 통과 기준                      |
| -------------- | -----------------: | ------------------------------ |
| 로그인 피크    |          1,000 RPS | 5xx 0.5% 이하, p95 500ms 이하  |
| 급여 홈 조회   |          3,000 RPS | p95 300ms 이하                 |
| 지출 추가      |            500 RPS | 중복 저장 0건, p95 500ms 이하  |
| 커뮤니티 목록  |          2,000 RPS | p95 500ms 이하                 |
| 알림 읽음 bulk |            500 RPS | p95 500ms 이하                 |
| 배너 조회      |          5,000 RPS | p95 200ms 이하, cache hit 확인 |
| 월말 확정 배치 | 100만 사용자 chunk | 실패 chunk 재시도 가능         |

## 9. 성능 저하 대응 기준

| 징후               |             임계값 | 조치                                       |
| ------------------ | -----------------: | ------------------------------------------ |
| API p95 초과       | 5분 연속 목표 초과 | 배포/DB/외부연동 확인, cache TTL 임시 상향 |
| DB connection 대기 |      3분 연속 증가 | write API rate limit, batch 중지           |
| 5xx 증가           |          0.5% 초과 | rollback 검토, incident 생성               |
| 커뮤니티 지연      |       p95 1초 초과 | 인기글 cache, 목록 size 축소               |
| 푸시 실패          |     실패율 5% 초과 | 재시도 job 가동, provider 상태 확인        |
