# 04. 인프라 설계서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 Cloudflare, Neon DB, 도메인, CDN, 환경변수, 네트워크, 운영 환경 구성을 최종 확정한다.

## 2. 인프라 최종 구성

| 영역          | 구성                                  | 목적                                  |
| ------------- | ------------------------------------- | ------------------------------------- |
| DNS           | Cloudflare DNS                        | 도메인 관리, TLS, routing             |
| CDN/WAF       | Cloudflare                            | 정적 자원 캐싱, 공격 차단, rate limit |
| API           | Cloudflare Workers                    | 엣지 API 실행                         |
| Admin Web     | Cloudflare Pages                      | 운영자 웹 UI 배포                     |
| File Storage  | Cloudflare R2                         | 프로필/첨부/배너 이미지 저장          |
| Database      | Neon DB PostgreSQL                    | 영속 데이터 저장                      |
| Scheduler     | Cloudflare Workers Cron               | 알림/월말/통계 작업                   |
| Repository    | GitHub                                | 소스코드, Actions, 환경별 배포        |
| Observability | Cloudflare Logs, Sentry, Neon Metrics | 장애 탐지/분석                        |

## 3. 도메인 구조

| 도메인                        | 용도               | 대상                      |
| ----------------------------- | ------------------ | ------------------------- |
| `salary-hijacking.app`        | 서비스 대표 도메인 | 랜딩/정책/앱 링크         |
| `api.salary-hijacking.app`    | 사용자 API         | Cloudflare Workers        |
| `admin.salary-hijacking.app`  | 관리자 웹          | Cloudflare Pages + Access |
| `cdn.salary-hijacking.app`    | 이미지/첨부 CDN    | R2 public/custom domain   |
| `status.salary-hijacking.app` | 상태 페이지        | 운영 상태 공개 또는 내부  |

## 4. 환경 분리

| 환경        | 목적        | 도메인                       | DB Branch       | 배포 방식             |
| ----------- | ----------- | ---------------------------- | --------------- | --------------------- |
| local       | 개발자 로컬 | localhost                    | Neon dev branch | 수동 실행             |
| development | 통합 개발   | dev-api.salary-hijacking.app | dev             | 자동 배포             |
| staging     | QA/검수     | stg-api.salary-hijacking.app | staging         | PR merge 후 자동 배포 |
| production  | 실제 서비스 | api.salary-hijacking.app     | main/prod       | 승인 후 배포          |

## 5. Cloudflare 구성

### 5.1 Zone 설정

| 항목                 | 설정                                |
| -------------------- | ----------------------------------- |
| SSL/TLS              | Full strict                         |
| Always Use HTTPS     | enabled                             |
| HSTS                 | production enabled                  |
| WAF                  | managed rules + custom rule         |
| Bot Fight/Rate Limit | 로그인/글쓰기/파일 업로드 우선 적용 |
| Cache                | 정적 파일 cache, API no-store 기본  |
| CORS                 | 앱/관리자 origin allowlist          |

### 5.2 Worker Routes

| Route                                     | Worker                | 설명        |
| ----------------------------------------- | --------------------- | ----------- |
| `api.salary-hijacking.app/api/v1/*`       | salary-api-prod       | 사용자 API  |
| `api.salary-hijacking.app/admin/api/v1/*` | salary-admin-api-prod | 관리자 API  |
| `api.salary-hijacking.app/health*`        | salary-api-prod       | 헬스체크    |
| `stg-api.salary-hijacking.app/*`          | salary-api-stg        | staging API |

### 5.3 R2 Buckets

| Bucket                | 공개 여부                               | 용도                     |
| --------------------- | --------------------------------------- | ------------------------ |
| salary-files-prod     | private origin + signed/public URL 정책 | 사용자 첨부, 프로필      |
| salary-banners-prod   | public CDN                              | 광고/공지 배너           |
| salary-files-stg      | staging only                            | QA 파일                  |
| salary-backup-exports | private                                 | 백업 내보내기, 감사 보관 |

## 6. Neon DB 구성

| 항목       | 최종 기준                                                |
| ---------- | -------------------------------------------------------- |
| Project    | salary-hijacking-prod                                    |
| Branch     | main/prod, staging, development, preview-per-PR          |
| Connection | pooled connection URL 기본 사용                          |
| SSL        | require                                                  |
| Migration  | GitHub Actions에서 실행                                  |
| Role       | app_user, migration_user, readonly_user, admin_user 분리 |
| Backup     | Neon PITR/백업 + 별도 export 정책 병행                   |

## 7. 네트워크 접근 정책

| 대상       | 접근 기준                                                                 |
| ---------- | ------------------------------------------------------------------------- |
| 사용자 API | 공개 HTTPS, WAF/rate limit 적용                                           |
| 관리자     | Cloudflare Access 인증 필수                                               |
| DB         | 공개 연결 문자열은 GitHub/Cloudflare secret에만 저장, 앱에 절대 포함 금지 |
| R2         | 업로드는 presigned URL, private object 직접 접근 금지                     |
| 내부 도구  | Cloudflare Tunnel/Access를 통해 보호                                      |

## 8. 환경변수 배치

| 환경               | 저장 위치                     | 설명                              |
| ------------------ | ----------------------------- | --------------------------------- |
| 로컬               | `.dev.vars` 또는 `.env.local` | 실제 production secret 금지       |
| Cloudflare Workers | Worker Secret/Variables       | API runtime 설정                  |
| GitHub Actions     | Environment Secrets           | 배포 및 마이그레이션 키           |
| Expo/EAS           | EAS Secret                    | 앱 빌드에 필요한 공개/비공개 설정 |
| Neon               | Project/Role Credentials      | DB role별 연결 정보               |

## 9. 인프라 보안 기준

| 영역        | 필수 기준                                              |
| ----------- | ------------------------------------------------------ |
| 관리자 접근 | Cloudflare Access + 2FA 계정 + RBAC                    |
| API 보호    | WAF, rate limit, request body limit, CORS allowlist    |
| Secret      | GitHub/Cloudflare/Expo 환경별 분리, source commit 금지 |
| DB 권한     | app_user 최소 권한, migration_user DDL 권한 분리       |
| 파일        | 업로드 확장자/MIME 검증, 공개 URL 만료/권한 정책 적용  |
| 로그        | 개인정보/토큰/급여 상세 금액 원문 로그 금지            |

## 10. 인프라 배포 절차

1. GitHub PR 생성.
2. preview DB branch 생성.
3. lint/test/build 실행.
4. staging Worker/Pages 배포.
5. DB migration staging 적용.
6. QA 승인.
7. production environment approval.
8. production DB migration 적용.
9. production Worker/Pages 배포.
10. health check, smoke test, monitoring 확인.

## 11. 최종 수용 기준

| 기준       | 완료 조건                                                        |
| ---------- | ---------------------------------------------------------------- |
| 환경 분리  | local/dev/staging/prod가 독립되어 있다.                          |
| Cloudflare | DNS, WAF, Workers, Pages, R2, Access 기준이 확정되어 있다.       |
| Neon       | branch, role, pooled connection, migration 기준이 확정되어 있다. |
| 배포       | GitHub Actions와 승인 절차가 정의되어 있다.                      |
| 보안       | 공개/비공개 리소스와 접근 통제 기준이 명확하다.                  |
