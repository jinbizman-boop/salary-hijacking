# 09. 환경변수/시크릿 관리 문서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 API 키, DB URL, OAuth secret, push key, Cloudflare token, GitHub secret, 앱 빌드 환경변수의 관리 기준을 최종 확정한다.

## 2. 분류 기준

| 분류             | 설명                           | 저장 위치                     |
| ---------------- | ------------------------------ | ----------------------------- |
| Public Config    | 앱에 포함되어도 되는 공개 설정 | app config, env public prefix |
| Runtime Variable | 민감하지 않은 서버 설정        | Cloudflare Workers variables  |
| Secret           | 외부 노출 금지 값              | Cloudflare/GitHub/EAS secrets |
| Credential       | DB/OAuth/Push 권한             | secret manager only           |
| Derived Secret   | 해시 salt, 서명키              | secret manager only           |

## 3. 금지 원칙

| 금지사항         | 기준                                                            |
| ---------------- | --------------------------------------------------------------- |
| Git commit       | `.env`, `.dev.vars`, key file, service account json commit 금지 |
| 앱 번들 포함     | DB URL, server secret, OAuth client secret 포함 금지            |
| 로그 출력        | secret 값 전체/일부 출력 금지                                   |
| 공유 문서 기재   | 실제 secret 값 문서화 금지, 이름만 문서화                       |
| 개인 메신저 전달 | 금지, 권한 있는 secret store만 사용                             |

## 4. 환경별 저장 위치

| 대상         | local                                  | staging                               | production                            |
| ------------ | -------------------------------------- | ------------------------------------- | ------------------------------------- |
| API env      | `.dev.vars.example` + 개인 로컬 secret | Cloudflare Worker Secret              | Cloudflare Worker Secret              |
| DB URL       | 개발자별 dev DB                        | GitHub env secret + Cloudflare secret | GitHub env secret + Cloudflare secret |
| OAuth secret | sandbox app                            | staging secret                        | production secret                     |
| FCM key      | test project                           | staging secret                        | production secret                     |
| Expo/EAS     | local profile                          | EAS secret preview                    | EAS secret production                 |
| Admin Access | 테스트 정책                            | Cloudflare Access staging             | Cloudflare Access prod                |

## 5. 필수 환경변수 목록

### 5.1 API 서버

| 이름                   | 분류            | 환경       | 설명                                 |
| ---------------------- | --------------- | ---------- | ------------------------------------ |
| `APP_ENV`              | variable        | all        | local/development/staging/production |
| `API_BASE_URL`         | variable        | all        | API public base URL                  |
| `DATABASE_URL`         | secret          | server     | Neon pooled connection URL           |
| `DATABASE_DIRECT_URL`  | secret          | migration  | Neon direct migration URL            |
| `JWT_ACCESS_SECRET`    | secret          | server     | access token 서명키                  |
| `JWT_REFRESH_SECRET`   | secret          | server     | refresh token 서명키 또는 pepper     |
| `PASSWORD_HASH_PEPPER` | secret          | server     | 비밀번호 해시 보강 secret            |
| `NAVER_CLIENT_ID`      | variable/secret | server     | provider client id                   |
| `NAVER_CLIENT_SECRET`  | secret          | server     | provider client secret               |
| `KAKAO_CLIENT_ID`      | variable/secret | server     | provider client id                   |
| `KAKAO_CLIENT_SECRET`  | secret          | server     | provider client secret               |
| `GOOGLE_CLIENT_ID`     | variable/secret | server     | provider client id                   |
| `GOOGLE_CLIENT_SECRET` | secret          | server     | provider client secret               |
| `FCM_PROJECT_ID`       | variable        | server     | FCM project id                       |
| `FCM_SERVICE_ACCOUNT`  | secret          | server     | FCM service account JSON/base64      |
| `R2_BUCKET_NAME`       | variable        | server     | R2 bucket                            |
| `R2_ACCESS_KEY_ID`     | secret          | server     | R2 key if binding 외부 사용 시       |
| `R2_SECRET_ACCESS_KEY` | secret          | server     | R2 secret if binding 외부 사용 시    |
| `SENTRY_DSN`           | secret/variable | app/server | 오류 수집 DSN                        |
| `ADMIN_ALLOWED_DOMAIN` | variable        | admin      | 관리자 이메일 도메인 제한            |

### 5.2 모바일 앱

## 6. Secret Rotation 정책

| Secret               |     주기 | 즉시 교체 조건                    |
| -------------------- | -------: | --------------------------------- |
| JWT signing secret   |    180일 | 유출 의심, repo 노출, 관리자 퇴사 |
| Refresh token pepper |    180일 | 토큰 재사용 공격 감지             |
| DB password          | 90~180일 | 접근권한 변경, 유출 의심          |
| OAuth client secret  |    180일 | provider 보안 이벤트              |
| FCM service account  |    180일 | 발송 오남용 감지                  |
| Cloudflare API token | 90~180일 | 배포 권한 변경                    |
| GitHub deploy key    |    180일 | repository 권한 변경              |

## 7. 권한 원칙

| 역할           | 접근 가능                                     |
| -------------- | --------------------------------------------- |
| Developer      | local/dev secret, staging 일부                |
| Backend Lead   | staging/prod API secret 관리 권한             |
| DevOps Lead    | Cloudflare/GitHub/Neon production secret 관리 |
| Security/Admin | audit/rotation 승인                           |
| 일반 운영자    | secret 접근 불가                              |

## 9. 유출 사고 대응

1. 유출 범위 확인: repository, log, local PC, CI output, 배포 환경.
2. 해당 secret 즉시 폐기/rotation.
3. 관련 access token/session 무효화.
4. DB/API 접근 로그 점검.
5. 영향 사용자와 데이터 범위 판단.
6. 법무/개인정보 책임자 보고 기준 적용.
7. 재발 방지: secret scanning, 권한 축소, PR rule 강화.
