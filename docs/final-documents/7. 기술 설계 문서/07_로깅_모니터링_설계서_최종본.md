# 07. 로깅/모니터링 설계서 최종본

## 1. 문서 목적

본 문서는 급여납치 플랫폼의 에러 로그, 사용자 행동 로그, API 실패 로그, 알림 실패 로그, DB 성능 로그, 관리자 감사 로그를 최종 확정한다.

## 2. 관측성 목표

| 목표             | 기준                                           |
| ---------------- | ---------------------------------------------- |
| 장애 탐지        | 5분 이내 주요 장애 감지                        |
| 원인 추적        | requestId/sessionId/userId hash 기반 추적      |
| 개인정보 보호    | 급여/대출/저축/토큰 원문 로그 금지             |
| 운영 감사        | 관리자 행위 전부 audit log 저장                |
| 성능 분석        | endpoint, DB query, external call latency 측정 |
| 사용자 행동 분석 | 핵심 funnel 이벤트 수집                        |

## 3. 로그 유형

| 유형              | 대상                     | 저장/전송               |
| ----------------- | ------------------------ | ----------------------- |
| API Access Log    | 모든 API 요청            | Cloudflare Logs/Logpush |
| Application Error | 서버 예외                | Sentry + structured log |
| App Crash         | 모바일 앱 오류           | Sentry Mobile           |
| DB Slow Query     | 느린 쿼리                | Neon metrics + app log  |
| Auth Security Log | 로그인 실패, 토큰 재사용 | 보안 로그 저장          |
| Notification Log  | 푸시 생성/발송/실패      | DB notification_events  |
| Admin Audit Log   | 관리자 행위              | DB admin_audit_logs     |
| User Event Log    | 핵심 행동                | analytics_events        |

## 4. 공통 로그 필드

| 필드        | 설명                        | 개인정보 여부 |
| ----------- | --------------------------- | ------------- |
| timestamp   | 발생 시각 UTC               | 아님          |
| level       | DEBUG/INFO/WARN/ERROR/FATAL | 아님          |
| environment | dev/staging/prod            | 아님          |
| service     | api/admin/scheduler/mobile  | 아님          |
| requestId   | 요청 추적 ID                | 아님          |
| sessionId   | 세션 ID, 필요 시 hash       | 준식별        |
| userIdHash  | user_id hash                | 준식별        |
| endpoint    | API 경로 template           | 아님          |
| method      | HTTP method                 | 아님          |
| statusCode  | 응답 코드                   | 아님          |
| latencyMs   | 처리 시간                   | 아님          |
| errorCode   | 표준 오류 코드              | 아님          |
| appVersion  | 앱 버전                     | 아님          |
| deviceOs    | iOS/Android                 | 아님          |

## 5. 금지 로그 항목

| 금지 항목                   | 처리 기준                        |
| --------------------------- | -------------------------------- |
| accessToken/refreshToken    | 절대 로그 금지                   |
| OAuth authorization code    | 절대 로그 금지                   |
| 비밀번호/인증번호           | 절대 로그 금지                   |
| DB URL/API key/secret       | 절대 로그 금지                   |
| 급여 상세 금액 원문         | 기본 금지, 필요 시 범주화/마스킹 |
| 대출/저축 상세명            | 운영 로그 금지, DB에만 저장      |
| 파일 원본 URL private token | 절대 로그 금지                   |

## 6. API 로그 예시

```json
{
  "timestamp": "2026-06-15T03:00:00Z",
  "level": "INFO",
  "environment": "production",
  "service": "api",
  "requestId": "req_01JZ00001",
  "userIdHash": "usr_h_8f2a",
  "method": "POST",
  "endpoint": "/api/v1/variable-expenses",
  "statusCode": 201,
  "latencyMs": 182,
  "errorCode": null,
  "appVersion": "1.0.0"
}
```

## 7. 알림 실패 로그 기준

| 필드           | 설명                                     |
| -------------- | ---------------------------------------- |
| notificationId | 알림 ID                                  |
| userIdHash     | 사용자 hash                              |
| pushTokenHash  | 푸시 토큰 hash                           |
| provider       | FCM/APNs                                 |
| templateCode   | 알림 템플릿                              |
| status         | SENT, FAILED_RETRYABLE, FAILED_PERMANENT |
| errorCode      | provider 오류 코드                       |
| retryCount     | 재시도 횟수                              |
| nextRetryAt    | 다음 재시도 시각                         |

## 8. 핵심 사용자 행동 이벤트

| 이벤트명                 | 발생 시점        | 속성                   |
| ------------------------ | ---------------- | ---------------------- |
| app_opened               | 앱 실행          | appVersion, os         |
| login_succeeded          | 로그인 성공      | provider               |
| payroll_plan_created     | 급여 계획 생성   | yearMonth, paydayRange |
| fixed_expense_created    | 고정지출 생성    | category               |
| daily_budget_saved       | 일일 예산 저장   | budgetRange            |
| variable_expense_created | 변동지출 추가    | category, amountRange  |
| hijack_amount_viewed     | 납치금액 확인    | screen                 |
| growth_task_completed    | 레벨업 미션 완료 | taskType               |
| community_post_created   | 게시글 등록      | boardType, isAnonymous |
| notification_opened      | 알림 클릭        | notificationType       |
| ad_clicked               | 광고 클릭        | campaignId             |

## 9. 모니터링 대시보드

| 대시보드      | 핵심 지표                                       |
| ------------- | ----------------------------------------------- |
| API Health    | RPS, p95 latency, 5xx rate, 4xx rate            |
| DB Health     | connection count, slow query, CPU/storage, lock |
| Auth Security | 로그인 실패, 토큰 재사용 감지, rate limit hit   |
| Payroll Core  | 급여 계획 생성 수, 지출 추가 수, 계산 오류      |
| Community     | 게시글/댓글/신고 수, 숨김 처리 수               |
| Notification  | 생성/발송/실패/재시도 수                        |
| Mobile App    | crash-free session, screen load, API failure    |
| Admin         | 관리자 로그인, 제재, 배너/공지 변경             |

## 10. 알림/경보 정책

| 경보               |                   조건 | 심각도 | 조치                  |
| ------------------ | ---------------------: | ------ | --------------------- |
| API 5xx 증가       |          5분 0.5% 초과 | P1     | 배포/DB/외부연동 확인 |
| 로그인 실패 급증   |     10분 평균 대비 3배 | P2     | 보안 이벤트 확인      |
| DB connection 포화 |   5분 연속 임계치 초과 | P1     | rate limit/batch 중지 |
| 푸시 실패율 증가   |           10분 5% 초과 | P2     | FCM/APNs 상태 확인    |
| 배치 실패          | 단일 critical job 실패 | P1     | 재실행/rollback 검토  |
| 앱 crash 증가      |      release별 2% 초과 | P1     | hotfix 검토           |

## 11. 보존 기간

| 로그                | 보존 기간                             |
| ------------------- | ------------------------------------- |
| API access log      | 30~90일                               |
| error log           | 180일                                 |
| security log        | 1년                                   |
| admin audit log     | 5년 또는 법무 정책 기준               |
| user behavior event | 1년, 개인 식별 제거 후 통계 보존 가능 |
| notification event  | 180일                                 |
| DB backup log       | 백업 보존 기간 + 30일                 |

## 12. 최종 수용 기준

| 기준      | 완료 조건                                               |
| --------- | ------------------------------------------------------- |
| 추적성    | requestId 기반으로 앱/API/DB/외부연동을 연결할 수 있다. |
| 개인정보  | 민감 정보 로그 금지 기준이 있다.                        |
| 장애 탐지 | API, DB, 푸시, 앱 crash 경보가 정의되어 있다.           |
| 운영 감사 | 관리자 행위가 추적 가능하다.                            |
| 분석      | 핵심 사용자 행동 이벤트가 정의되어 있다.                |
