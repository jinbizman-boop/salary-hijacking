# 07. 급여납치 롤백 계획서 최종본

> 문서 상태: 최종본  
> 적용 범위: 급여납치 모바일 애플리케이션 iOS/Android 정식 출시  
> 작성 기준일: 2026-06-15  
> 기준 시간대: Asia/Seoul  
> 문서 원칙: 본 문서는 문서상·이론상 추가 기획 작업이 필요 없도록 출시 준비, 검수, 등록, 마케팅, 버전, 롤백 기준을 완결형으로 정의한다.

---

## 1. 문서 목적

본 문서는 급여납치 앱 출시 또는 업데이트 이후 장애, 보안 문제, 데이터 오류, 스토어 심사 문제, 성능 저하가 발생했을 때 이전 안정 상태로 복구하기 위한 롤백 기준과 절차를 정의한다.

## 2. 롤백 원칙

| 원칙               | 설명                                                                  |
| ------------------ | --------------------------------------------------------------------- |
| 사용자 피해 최소화 | 급여/지출/저축/커뮤니티 데이터 손상 방지를 최우선으로 한다            |
| 데이터 보존 우선   | 기능 롤백보다 데이터 정합성과 백업 복구 가능성을 우선한다             |
| 단계적 차단        | 전체 롤백 전 기능 플래그/트래픽 제한/단계적 배포 중단을 우선 검토한다 |
| 신속한 공지        | 사용자 영향이 있으면 앱 내 공지와 CS 응대 문구를 즉시 제공한다        |
| 원인 추적          | 롤백 후에도 로그, 배포 태그, DB 변경 이력을 보존한다                  |

## 3. 롤백 대상 유형

| 유형               | 예시                                     | 우선 조치                                    |
| ------------------ | ---------------------------------------- | -------------------------------------------- |
| 앱 클라이언트 오류 | 앱 실행 불가, 특정 화면 크래시           | 스토어 단계적 배포 중단, 이전 빌드 유지      |
| API 오류           | 로그인/API 5xx 급증                      | API 이전 버전 배포, 트래픽 우회              |
| DB 오류            | 마이그레이션 실패, 금액 계산 데이터 손상 | DB 백업 복구 또는 보정 스크립트              |
| 계산 오류          | 납치금액/잔액/달성률 오계산              | 계산 API/클라이언트 패치, 영향 데이터 재계산 |
| 보안 문제          | 인증 우회, 타 사용자 데이터 노출         | 즉시 서비스 일부 차단, 토큰 폐기, 긴급 패치  |
| 푸시 알림 오류     | 중복/오발송                              | 푸시 작업 중지, 배치 비활성화                |
| 커뮤니티 문제      | 신고/차단 오작동, 유해 게시글 노출       | 게시글 작성/노출 제한, 관리자 검수 전환      |
| 광고/제휴 문제     | 잘못된 광고 랜딩, 민감 데이터 기반 광고  | 배너 비활성화, 광고 로그 분리 확인           |
| 스토어 문제        | 심사 거절, 메타데이터 정책 위반          | 메타데이터 수정, 빌드 재제출                 |

| 심각도 | 기준                                                    | 롤백 의사결정 시간 | 조치                           |
| ------ | ------------------------------------------------------- | -----------------: | ------------------------------ |

| S4     | 개선 요청, 불편사항                                     |          정기 검토 | backlog 반영                   |

## 6. 앱 클라이언트 롤백 절차

### 6.1 Android

1. Google Play Console에서 단계적 배포 비율 증가 중단.
2. 신규 릴리즈가 100% 배포 전이면 staged rollout halt.
3. 문제가 있는 버전의 배포 범위 확인.
4. 이전 안정 버전이 사용 가능한 경우 신규 hotfix AAB를 생성해 상위 versionCode로 배포.
5. 심각도 S0/S1이면 인앱 강제 업데이트 정책 또는 서버 기능 차단 적용.
6. 릴리즈 노트에 안정성 개선 문구 반영.

### 6.2 iOS

1. App Store Connect에서 신규 버전 배포 상태 확인.
2. 이미 승인/출시된 앱은 동일 버전으로 직접 되돌릴 수 없으므로 hotfix 빌드를 새 버전으로 제출한다.
3. 서버 기능 플래그로 문제 기능을 우선 비활성화한다.
4. 심각한 오류는 Apple 심사 메모에 긴급성 설명 후 신속 제출한다.
5. 사용자 공지와 FAQ를 업데이트한다.

## 7. API 롤백 절차

1. 장애 발생 버전 태그 확인.
2. 현재 배포된 commit SHA 기록.
3. 이전 안정 태그 `api-vX.Y.Z` 확인.
4. Cloudflare/API 배포 파이프라인에서 이전 이미지 또는 이전 commit 재배포.
5. health check 통과 여부 확인.
6. 오류율, p95 응답시간, 로그인 성공률 확인.
7. API 문서와 릴리즈 노트에 롤백 사유 기록.

## 8. DB 롤백 절차

### 8.1 사전 원칙

- 운영 DB 변경 전 백업 스냅샷을 생성한다.
- destructive migration은 단독 배포하지 않는다.
- 컬럼 삭제/타입 변경은 최소 1개 버전 이상 deprecation 후 진행한다.

### 8.2 롤백 절차

1. 실패한 migration ID 확인.
2. 데이터 손상 범위 확인.
3. 읽기 전용 모드 또는 관련 쓰기 API 차단.
4. 백업 복구가 필요한지, 보정 스크립트로 충분한지 판단.
5. 보정 스크립트는 staging에서 검증 후 production 적용.
6. 금액 데이터는 재계산 로그와 전후값을 별도 기록.
7. 복구 후 샘플 사용자와 영향 사용자 데이터를 검증.

## 10. 사용자 공지 문안

### 10.1 일반 장애

안녕하세요. 급여납치 운영팀입니다.  
현재 일부 기능 이용이 원활하지 않아 확인 중입니다. 급여/지출 데이터 보호를 우선으로 조치하고 있으며, 복구가 완료되는 즉시 다시 안내드리겠습니다.

### 10.2 업데이트 오류

안녕하세요. 급여납치 운영팀입니다.  
최신 버전에서 일부 오류가 확인되어 안정화 작업을 진행하고 있습니다. 이용에 불편을 드려 죄송합니다. 안정적인 사용을 위해 수정 버전 배포 후 업데이트를 부탁드립니다.

### 10.3 데이터 정합성 오류

안녕하세요. 급여납치 운영팀입니다.  
일부 금액 표시 오류가 확인되어 재계산 및 검증을 진행했습니다. 실제 사용자가 입력한 원본 데이터 보호를 우선으로 조치했으며, 이상이 지속되면 1:1 문의로 알려주세요.

### 10.4 복구 완료

안녕하세요. 급여납치 운영팀입니다.  
확인된 오류 조치가 완료되었습니다. 현재 앱 이용이 정상화되었으며, 같은 문제가 반복되지 않도록 모니터링을 강화하겠습니다.

## 13. 롤백 완료 기준

롤백은 다음 조건이 모두 충족될 때 완료로 본다.

- 사용자 영향 기능이 정상화되었다.
- 오류율과 성능 지표가 안정 기준으로 회복되었다.
- 데이터 정합성 검증이 완료되었다.
- 사용자 공지와 CS 대응 문구가 배포되었다.
- 원인 분석과 재발 방지 항목이 기록되었다.
- hotfix 또는 후속 배포 계획이 확정되었다.
