# 04. 급여납치 스크린샷 기획서 최종본

> 문서 상태: 최종본  
> 적용 범위: 급여납치 모바일 애플리케이션 iOS/Android 정식 출시  
> 작성 기준일: 2026-06-15  
> 기준 시간대: Asia/Seoul  
> 문서 원칙: 본 문서는 문서상·이론상 추가 기획 작업이 필요 없도록 출시 준비, 검수, 등록, 마케팅, 버전, 롤백 기준을 완결형으로 정의한다.

---

## 1. 문서 목적

본 문서는 App Store와 Google Play에 등록할 급여납치 스토어 스크린샷의 화면 구성, 카피, 촬영 기준, 해상도 대응, 검수 기준을 최종 확정한다. 모든 스크린샷은 실제 앱 UI를 기반으로 하며 허위 기능, 과장된 수치, 제공하지 않는 혜택을 포함하지 않는다.

## 2. 스크린샷 전략

| 전략             | 설명                                                                                            |
| ---------------- | ----------------------------------------------------------------------------------------------- |
| 첫 3장 집중      | 검색 결과와 스토어 상단에서 핵심 가치가 바로 보이도록 급여 홈, 일일 예산, 계획 화면을 우선 배치 |
| 숫자 중심        | 수령금액, 지출금액, 납치금액, 남은금액을 명확히 노출                                            |
| 습관화 강조      | LV UP과 커뮤니티를 후반부에 배치해 단순 가계부가 아닌 자기관리 플랫폼임을 전달                  |
| 실제 UI 기반     | 앱 내부 화면 캡처를 사용하고 목업은 기기 프레임/배경/카피 보조 용도로만 사용                    |
| 스토어 정책 준수 | 가격, 순위, 타사명, 보장성 문구, 허위 이벤트 문구 제외                                          |

## 3. 최종 스크린샷 세트

| 순서 | 화면                | 대표 카피                            | 보조 카피                           | 핵심 UI                                      |
| ---: | ------------------- | ------------------------------------ | ----------------------------------- | -------------------------------------------- |
|    1 | 급여 홈             | 월급이 사라지기 전에 먼저 붙잡기     | 수령금액·지출금액·납치금액을 한눈에 | 누적 납치금액, 급여 카드, 고정지출           |
|    2 | 일일 예산           | 오늘 쓸 수 있는 돈만 정확히 보기     | 사용금액과 남은금액을 즉시 계산     | 일일 설정금액, 사용금액, 남은금액, 지출 추가 |
|    3 | 계획/설정           | 급여일마다 반복되는 지출을 미리 정리 | 고정지출·저축·생활비 계획 관리      | 목표 달성률, 급여 계획, 고정지출표, 저축표   |
|    4 | 알림                | 예산 초과와 결제 예정 놓치지 않기    | 목표 달성·미션·이벤트 알림 제공     | 알림 리스트, 새 알림 강조                    |

|    6 | 독서/뉴스/영어/건강 | 매일 하나씩 성장하는 루틴            | 콘텐츠와 미션으로 자기관리 지속     | 콘텐츠 카드, 추천 도서/뉴스/영어/홈트        |
|    7 | 커뮤니티            | 직장인 자기관리 인증 커뮤니티        | 소비관리와 레벨업 경험 공유         | 게시판 탭, 인기글, 게시글 카드               |
|    8 | 마이페이지          | 내가 지켜낸 월급과 성장 기록         | 누적 납치금액·레벨·성과 확인        | 프로필, 누적 금액, 레벨, 관리 메뉴           |

## 4. 필수 5장 축약 세트

스토어 등록 초기에는 최소 다음 5장을 필수로 사용한다.

1. 급여 홈
2. 일일 예산
3. 계획/설정
4. LV UP
5. 커뮤니티 또는 마이페이지

## 5. 각 스크린샷 상세 기획

### 5.1 1번 — 급여 홈

| 항목        | 내용                                                                                     |
| ----------- | ---------------------------------------------------------------------------------------- |
| 목적        | 급여납치의 핵심 개념을 즉시 이해시키는 첫 화면                                           |
| 화면 상태   | 로그인 완료, 급여 계획 등록 완료, 고정지출/변동지출 데이터 존재                          |
| 노출 데이터 | 수령금액 2,700,000원, 지출금액 73,000원, 납치금액 2,627,000원, 누적 납치금액 5,780,000원 |
| 카피        | 월급이 사라지기 전에 먼저 붙잡기                                                         |
| 보조 카피   | 수령금액·지출금액·납치금액을 한눈에                                                      |
| 검수        | 실제 계산식과 수치 일치, 광고 배너가 과도하게 중심을 차지하지 않음                       |

### 5.2 2번 — 일일 예산

| 항목      | 내용                                                               |
| --------- | ------------------------------------------------------------------ |
| 목적      | 사용자가 매일 앱을 켜야 하는 이유 전달                             |
| 화면 상태 | 오늘 예산 20,000원, 사용금액 13,000원, 남은금액 7,000원            |
| 카피      | 오늘 쓸 수 있는 돈만 정확히 보기                                   |
| 보조 카피 | 사용금액과 남은금액을 즉시 계산                                    |
| 핵심 장면 | 변동지출 추가 입력창 또는 지출 리스트                              |
| 검수      | 남은금액 음수 예시는 초기 스크린샷에서 제외, 긍정적 관리 경험 강조 |

### 5.3 3번 — 계획/설정

| 항목      | 내용                                                    |
| --------- | ------------------------------------------------------- |
| 목적      | 급여 전 계획형 관리 차별점 전달                         |
| 화면 상태 | 목표 달성률 87%, 급여 계획/고정지출/저축/생활비 표 노출 |
| 카피      | 급여일마다 반복되는 지출을 미리 정리                    |
| 보조 카피 | 고정지출·저축·생활비 계획 관리                          |
| 검수      | 표 텍스트가 모바일 스토어에서 읽을 수 있어야 함         |

### 5.4 4번 — 알림

| 카피      | 예산 초과와 결제 예정 놓치지 않기                |
| 보조 카피 | 목표 달성·미션·이벤트 알림 제공                  |
| 검수      | 현금성 보상 표현은 실제 운영 정책과 일치해야 함  |

## 10. 최종 완료 기준

스크린샷 기획은 다음 조건이 충족되면 완료로 본다.

- 최소 5장, 권장 8장 구성이 확정되어 있다.
- 모든 카피와 화면 목적이 정의되어 있다.
- 실제 앱 UI와 기능이 일치한다.
- 개인정보, 허위/과장, 금융/건강 보장성 표현이 제거되어 있다.
- App Store와 Google Play 업로드 규격에 맞게 추출 가능하다.
