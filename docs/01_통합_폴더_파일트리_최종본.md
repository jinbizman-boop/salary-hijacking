# 급여납치 플랫폼 통합 완성형 폴더 및 파일트리 최종본

- 기준일: 2026-06-15
- 저장소 루트: `salary-hijacking-platform/`
- 구조 목적: 모바일 앱, 관리자 웹, API, 스케줄러, 푸시, DB, 인프라, 문서, QA, 운영, 출시, 협업을 하나의 완성형 모노레포 기준으로 통합한다.

## 최종 구조 원칙

1. 사용자용 앱과 관리자용 앱은 `apps/` 아래에 분리한다.
2. API 서버, 스케줄러, 푸시 알림 서비스는 `services/` 아래에 분리한다.
3. 타입, API 계약, DB, UI, 보안, 유틸은 `packages/` 아래에 공통 패키지로 분리한다.
4. Neon DB 관련 자료는 `database/` 아래에서 단일 관리한다.
5. Cloudflare, Neon, GitHub, 도메인, 환경변수 기준은 `infra/` 아래에서 관리한다.
6. 최종 기획·설계 문서는 `docs/final-documents/` 아래에 통합 보관한다.
7. QA, 운영, 출시 산출물은 각각 `qa/`, `ops/`, `release/` 아래에서 실행 기준으로 관리한다.
8. 보안과 개인정보는 `security/`와 `docs/final-documents/8. 보안·개인정보·정책 문서/`에서 이중 관리한다.
9. 모든 스크립트는 `scripts/`, 내부 점검 도구는 `tools/`에 배치한다.
10. 폴더 변경은 `docs/governance/structure-change-policy.md` 승인 절차를 따른다.
